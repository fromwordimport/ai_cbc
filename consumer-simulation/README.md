## 快速开始

### 第一步：生成消费者画像
打开 [`05-Prompt模板库.md`](./05-Prompt模板库.md) 中的**模板一**，将 `{数量}` 替换为需要生成的人数（建议 5-10 人），粘贴到任意支持长上下文的 LLM（Claude / GPT-4 / Kimi 等）中执行。

### 第二步：准备模拟输入
对照 [`08-Agent输入信息规范.md`](./08-Agent输入信息规范.md)，确认已提供：消费者画像 + 当前情境 + 交互任务。按需补充可选输入。

### 第三步：执行情境模拟
从生成的画像中选择目标人物，打开 **模板二**，填入具体情境参数，让模型进入角色扮演模式。

### 第四步：验证真实性
对照 [`04-阶段三四-交互与验证.md`](./04-阶段三四-交互与验证.md) 中的「校验清单」和「对抗性测试」，检查模拟输出是否符合真实人类特征。

## 设计原则速查

| 原则 | 含义 | 反例 |
|------|------|------|
| **张力优先** | 主动组合矛盾标签，制造内在冲突 | 所有标签方向一致，生成"完美均值人" |
| **动机可溯** | 每个异常行为都有心理层面的解释 | 高收入者表现出极端节俭，却无历史成因说明 |
| **认知有限** | 模型会疲劳、会敷衍、会走捷径 | 无论多累都保持完美理性和热情 |
| **情境敏感** | 同一人在不同场景下行为不同 | 情绪高涨和焦虑时决策模式完全相同 |
| **时间延续** | 有记忆、有惯性、有状态转移 | 每次交互都是独立随机事件 |
