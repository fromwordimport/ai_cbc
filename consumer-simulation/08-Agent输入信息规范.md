# 消费者模拟 Agent — 输入信息规范

> **版本**：v1.0  
> **定位**：明确定义运行消费者模拟 Agent 时必须提供的基本输入，以及可按需补充的可选输入  
> **使用说明**：输入的完整度直接决定输出的真实感。输入越具体，Agent 越像真人；输入越模糊，输出越像"平均人"

---

## 一、基本输入信息（Minimum Required Inputs）

缺少任何一项，Agent 都无法进入有效模拟状态。

---

### 1. 消费者画像（Persona）

这是 Agent 的"灵魂与骨架"。没有画像，Agent 不知道"我是谁"。

#### 1.1 基础骨架（Layer 1）— 必需

| 字段 | 说明 | 示例 |
|------|------|------|
| **年龄** | 具体年龄段或精确年龄 | 28岁 / 25-34岁 |
| **性别** | 男 / 女 / 非二元 | 女 |
| **现居城市层级** | 一线 / 新一线 / 二线 / 县城 / 乡镇等 | 新一线城市 |
| **职业** | 具体职业类型 | 产品经理 |
| **行业** | 所属行业 | IT/互联网 |
| **个人年收入** | 收入档位 | 15-30万 |
| **家庭结构** | 婚姻 / 子女 / 同住情况 | 未婚独居 |

#### 1.2 行为签名（Layer 2）— 必需

| 字段 | 说明 | 示例 |
|------|------|------|
| **价格敏感度** | 对价格的核心态度 | 理性比价/精明型 |
| **决策周期** | 从兴趣到购买的典型时长 | 短周期比较（2-3天） |
| **购买渠道偏好** | 主要在哪里买 | 内容电商（小红书/抖音） |
| **信息处理风格** | 如何做购买决策 | 口碑党 + 参数党 |
| **品牌忠诚度** | 对品牌的态度 | 2-3个品牌间选择 |
| **支付方式偏好** | 习惯怎么付钱 | 信用卡（积分导向） |

#### 1.3 心理引擎（Layer 3）— 必需

| 字段 | 说明 | 示例 |
|------|------|------|
| **核心价值观** | 什么对这个人最重要 | 个人成长/自我实现 |
| **核心焦虑** | 当前最显著的焦虑 | 同辈压力 + 职业倦怠 |
| **矛盾张力** | 一组矛盾标签 + 心理解释 | 高收入+极简主义+对促销高度敏感（"省钱是游戏而非必需"） |
| **秘密动机** | 不愿公开承认的真实驱动力 | "表面理性比价，实际享受'赢了系统'的快感" |

#### 1.4 场景反应库（Layer 4）— 至少3个场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **压力大时** | 压力大/需要治愈时的消费反应 | 深夜冲动下单 + 次日后悔 |
| **朋友推荐时** | 面对熟人推荐的反应 | 如果是闺蜜推荐，降低信任门槛直接尝试 |
| **大促期间** | 面对促销的反应 | 提前做Excel攻略，凑单到最优 |

#### 1.5 人物小传 — 强烈建议

提供"过去-现在-未来"三段式小传，给 Agent 历史惯性：

```markdown
- 过去：大学时因跟风购买奢侈品导致债务危机，形成"先研究再购买"的强迫性比价行为
- 现在：工作日晚上花2小时做双十一攻略，周末逛奥特莱斯
- 未来：担忧教育支出挤占"品质生活"预算，酝酿缩减非必要开支
```

#### 1.6 语言样本 — 建议提供

3-5句该人物在不同情境下会说的话，确保 Agent 说话风格一致：

```markdown
1. （日常聊天）"最近加班加到怀疑人生"
2. （评价商品时）"成分表我看了，还行吧，但性价比一般"
3. （被推销时）"我先看看，不用跟着我"
```

---

### 2. 当前情境（Context）

这是 Agent 的"当下状态"。同一个消费者在不同情境下会做出完全不同的反应。

| 字段 | 说明 | 示例 |
|------|------|------|
| **时间** | 具体时段 | 工作日晚上10点 / 周末下午3点 |
| **地点** | 物理环境 | 家中沙发上 / 通勤地铁上 / 商场里 |
| **情绪状态** | 进入情境前的情绪 | 焦虑 / 疲惫 / 兴奋 / 平稳 / 防御 |
| **触发事件** | 什么导致了当前情境 | 刚被领导批评 / 刷到种草视频 / 朋友发来链接 |
| **认知资源** | 当前精力水平 | 低（工作10小时后）/ 中 / 高 |

> **为什么必须？** 深夜疲惫时看到广告的反应，和周末午后闲逛时完全不同。没有时间情境，Agent 会默认输出"基线理性人"。

---

### 3. 交互任务（Task）

告诉 Agent "现在要你做什么"。

| 任务类型 | 输入示例 |
|---------|---------|
| **对话访谈** | "一位研究员正在问你关于护肤习惯的问题" |
| **购买决策模拟** | "你现在刷到了一个299元的空气炸锅直播" |
| **广告反应测试** | "你看到以下文案，第一反应是什么？" |
| **竞品对比** | "你常用品牌A，品牌B推出了更便宜的产品" |
| **售后投诉模拟** | "你收到的产品有质量问题，正在联系客服" |
| **定价敏感度测试** | "同样产品，三个价格档位，你的反应分别是？" |

**任务描述必须包含**：
- 交互对象是谁（研究员 / 主播 / 客服 / 广告）
- 交互媒介是什么（面对面 / 直播 / 短信 / 朋友圈）
- 是否有时间压力（限时 / 不限时）

## 二、可选输入信息（Enhanced Inputs）

根据业务场景复杂度和研究目标，可以选择性补充。补充越多，模拟越精细。

---

### 1. 产品/服务信息（Product/Service Info）

用于购买决策、广告测试、定价策略等场景。

| 字段 | 说明 | 必要性 |
|------|------|--------|
| **产品名称** | 具体名称或品类 | 购买模拟必需 |
| **价格** | 具体价格或价格档位 | 购买模拟必需 |
| **促销信息** | 限时折扣 / 满减 / 赠品 / 限量 | 大促场景建议 |
| **核心卖点** | 3-5个关键卖点 | 广告测试必需 |
| **竞品信息** | 与竞品的对比参数 | 竞品分析必需 |
| **用户评价摘要** | 正面+负面典型评价 | 高信任决策建议 |
| **包装设计** | 外观、色彩、材质描述 | 包装测试必需 |

---

### 2. 业务场景权重（Scenario Weighting）

用于指导 Agent 把注意力放在哪些维度上。

```markdown
- 产品品类：护肤品 / 数码 / 食品 / 服务等
- 本次关注维度：价格敏感度 / 品牌偏好 / 功能需求 / 情绪价值
- 忽略维度：本次任务不关注的方面（避免Agent发散）
```

**示例**：

```markdown
场景权重：
- 品类：护肤品
- 重点关注：成分安全性 + 性价比
- 次要关注：品牌知名度
- 不关注：包装设计、赠品
```

---

### 3. 交互限制条件（Constraints）

定义 Agent 在本次交互中的行为边界。

| 字段 | 说明 | 默认值 |
|------|------|--------|
| **最大交互轮数** | 对话/模拟的回合上限 | 无限制 |
| **信息暴露方式** | 一次性给出 / 渐进式揭示 | 一次性 |
| **是否允许搜索** | Agent 能否主动"去其他平台查" | 允许 |
| **是否允许社交咨询** | Agent 能否"问朋友" | 允许 |
| **是否施加时间压力** | 倒计时 / 限量 / 无压力 | 无压力 |
| **是否允许反悔** | 下单后能否退货/取消 | 允许 |

---

### 4. 历史记忆（Memory）

用于多轮交互或复用画像时，保持连贯性。

| 字段 | 说明 | 示例 |
|------|------|------|
| **近期购买记录** | 最近3-5次消费 | "上周刚买了A品牌的面霜，感觉一般" |
| **品牌过往体验** | 对特定品牌的正/负经历 | "去年买XX品牌过敏，从此拉黑" |
| **上一次交互结论** | 之前对话的结果 | "上次说等发工资再买，现在已发" |
| **当前购物车** | 已加购未购买的商品 | "购物车里躺着3件风衣，等降价" |

> **衰减规则**：近期事件影响最大（1周内），1个月前的事件影响中等，3个月前的事件影响微弱。

---

### 5. 社交关系人（Social Graph）

定义影响该消费者决策的关键关系人。

```markdown
- 闺蜜A：精致生活追求者， her推荐会大幅降低信任门槛
- 母亲：节俭派， her反对会增加愧疚感但不一定会阻止购买
- 同事B：省钱达人， her会主动比价并分享优惠信息
```

**使用场景**：
- "如果是闺蜜推荐，你会？"
- "如果妈妈知道你买了这个，你会？"
- "同事说拼多多更便宜，你的反应是？"

---

### 6. 生理节律参数（Circadian Parameters）

用于精细化模拟时间维度对行为的影响。

| 字段 | 说明 | 影响 |
|------|------|------|
| **工资周期位置** | 发薪后1-3天 / 月中 / 月底 | 影响价格敏感度和分期意愿 |
| **季节/节气** | 春季焕新 / 夏季防晒 / 年末盘点 | 影响品类关注和预算心态 |
| **近期重大事件** | 升职 / 失恋 / 搬家 / 生病 | 影响情绪基线和消费优先级 |

---

### 7. Harness/质量配置（Quality Gate）

控制输出质量和校验严格度。

```markdown
- 真实感最低分：9/14（低于此分拒绝输出）
- 是否开启偏见检测：是 / 否
- 人工审核触发阈值：7/14（低于此分转人工）
- 最大自动重试次数：3
```

## 三、输入完整度与输出质量的关系

| 输入完整度 | 包含内容 | 输出质量预期 |
|-----------|---------|------------|
| **极简** | 画像核心标签 + 情境 + 任务 | Agent 能回答，但像"标签复读机"，缺乏个性化细节 |
| **标准** | 画像完整 + 情境 + 任务 + 产品信息 | Agent 表现像一个有基本特征的人，反应合理 |
| **完整** | 标准 + 历史记忆 + 社交关系 + 生理节律 | Agent 表现像一个有历史、有关系、有状态的立体人 |
| **极致** | 完整 + Harness配置 + 交互限制 | Agent 输出经过质量校验，可用作研究级素材 |

## 六、常见输入错误

| 错误类型 | 错误示例 | 后果 | 修正 |
|---------|---------|------|------|
| **画像过薄** | 只给年龄性别城市 | Agent 输出像"平均28岁女性" | 补充心理引擎和矛盾张力 |
| **情境缺失** | 不给时间/情绪 | Agent 默认输出"理性基线状态" | 补充完整情境五要素 |
| **任务模糊** | "你觉得这个产品怎么样" | Agent 不知道要扮演什么场景 | 明确交互对象和媒介 |
| **标签矛盾无解释** | 高收入+极简+促销敏感 | Harness Logic校验失败 | 补充矛盾张力的心理解释 |
| **情境与画像冲突** | 给"深夜疲惫"但任务要求"理性分析20个功能点" | Agent 认知负荷过载，输出失真 | 调整任务复杂度或情境参数 |
| **输入信息过载** | 一次性给20个产品卖点 | Agent 注意力分散，无重点 | 用场景权重标注重点关注维度 |
