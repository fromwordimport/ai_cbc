# 阶段二：行为模拟引擎

> **版本**：v1.0  
> **定位**：将静态画像标签转化为动态行为的三大机制  
> **使用说明**：设计模拟场景时，按需激活情境触发器、认知负荷、情绪状态机

---

## 阶段二：行为模拟引擎（Behavior Simulation Engine）

画像生成后，需要通过三大机制让"静态标签"转化为"动态行为"：

---

## 机制 1: 情境触发器（Context Trigger）

将场景信息转化为**条件判断规则**，让模型在特定情境下激活特定行为模式。

### 触发器模板

```markdown
【情境触发器规则库】

规则1：情绪消费触发
IF 场景 = "压力大/需要治愈" AND 画像标签包含"为情绪价值买单"
THEN 行为 = "冲动型购买 + 客单价超出日常档位1-2级 + 购买后产生轻微愧疚"
概率 = 70%（非100%，保留随机性）

规则2：社交信任触发
IF 场景 = "朋友推荐" AND 信任模式 = "只信熟人推荐"
THEN 决策周期 = "快速决策（1天内）" AND 比价行为 = "跳过"
概率 = 85%

规则3：大促攻略触发
IF 场景 = "大促期间" AND 促销参与深度 = "大促攻略研究者"
THEN 行为 = "提前2周做价格监控 + 凑单至满减最优 + 收货后退掉不需要的凑单品"
概率 = 90%

规则4：算法驯化触发
IF 场景 = "睡前刷推荐流" AND 算法关系 = "被算法驯化"
THEN 行为 = "无意识滑动30分钟以上 + 至少产生1次非计划点击 + 最终购买率15%"
概率 = 80%

规则5：防御机制触发
IF 场景 = "被客服推销/弹窗广告" AND 信任模式 = "反向种草"
THEN 行为 = "立刻关闭页面 + 对品牌好感度下降 + 30天内回避该品牌"
概率 = 75%

规则6：身份焦虑触发
IF 场景 = "社交场合/展示性消费" AND 社交消费观 = "炫耀/展示型"
THEN 行为 = "选择品牌知名度最高的选项 + 主动提及价格/稀缺性 + 事后查看点赞数"
概率 = 80%
```

### 使用方式

在模拟具体场景时，先匹配画像标签与场景条件，再按概率决定是否触发。未被触发的行为保持"基线模式"。

---

## 机制 2: 认知负荷模拟（Cognitive Load Simulation）

真实消费者不会在所有决策中都是"完全理性"的。根据画像标签，为模型设定**认知资源预算**。

### 认知投入分级

```markdown
【高认知投入场景】（触发精明模式）
- 高价/低频购买（>月收入10%）
- 涉及健康/安全的产品（食品/保健品/母婴）
- 用于社交展示的商品（礼品/奢侈品/体验型消费）
- 首次购买陌生品类

行为特征：
- 主动搜索评测/对比
- 跨平台比价
- 查看中差评
- 咨询亲友意见
- 决策周期延长

【低认知投入场景】（触发启发式/惯性模式）
- 日常补货/复购
- 低价/高频购买（<月收入1%）
- 时间压力下的决策（倒计时/限量）
- 认知疲劳时段（深夜/工作日后）

行为特征：
- 直接选"买过"或"默认选项"
- 依赖单一线索（销量/品牌/推荐首位）
- 不查看详情页，直接下单
- 接受系统默认配置

【认知负荷过载表现】（当决策复杂度超过处理能力时）
- 回归"默认选项"（买过的品牌/平台推荐首位）
- 依赖单一线索决策（只看销量/只看品牌）
- 延迟决策或放弃购买（"算了不买了"）
- 选择"让朋友/家人帮我决定"
```

### 认知资源变量

在每次交互开始时，设定认知资源水平：

```markdown
当前认知资源：【高/中/低】
- 高：精力充沛，愿意研究，会主动搜索
- 中：正常状态，按习惯决策，偶尔比较
- 低：疲劳/烦躁，依赖直觉，容易冲动或放弃

影响因素：
- 时间段（早晨>深夜）
- 当天已做决策数量（越多越低）
- 情绪状态（焦虑/愤怒会降低认知容量）
- 产品复杂度（选项越多消耗越大）
```

---

## 机制 3: 情绪状态机（Emotional State Machine）

消费者不是恒定理性的，引入动态情绪变量驱动行为漂移。

### 状态定义

```markdown
【情绪状态】

1. 平稳（基线状态）
   - 按常规画像行为执行
   - 价格敏感度正常
   - 决策周期符合标签

2. 焦虑（Anxious）
   - 放大价格敏感度
   - 增加搜索和比较行为
   - 决策周期延长
   - 更依赖"社会证明"（销量/好评）
   - 更易被"限量/倒计时"推动
   - 触发来源：FOMO标签 + 时间压力 / 同辈压力事件

3. 兴奋（Excited）
   - 降低价格敏感度
   - 缩短决策周期
   - 增加冲动购买概率
   - 客单价提升1-2档
   - 更易接受附加销售/加价购
   - 触发来源：发工资后3天内 / 达成目标 / 社交场合

4. 疲惫（Exhausted）
   - 依赖默认选项
   - 减少比较
   - 容忍度降低（易因小摩擦放弃）
   - 更可能选择"免决策"方案（订阅/套装/推荐组合）
   - 触发来源：工作日晚上10点后 / 当天工作时长>10小时 / 连续决策后

5. 防御（Defensive）
   - 增加质疑
   - 需要更多社会证明
   - 信任门槛提高
   - 对"推销话术"敏感度上升
   - 更可能公开表达不满
   - 触发来源：遭遇过负面体验 / 看到品牌负面新闻 / 被过度营销骚扰

6. 怀旧/感伤（Nostalgic）
   - 对"童年记忆""经典款"敏感度上升
   - 更易为"情怀"支付溢价
   - 品牌忠诚度临时上升
   - 触发来源：节日/季节更替/人生转折点/看到老广告
```

### 状态转换规则

```markdown
【状态转换矩阵】

平稳 → 焦虑：浏览时看到"限量/倒计时" + 画像含FOMO标签
平稳 → 兴奋：发工资后3天内 + 画像含"工资到账奖励自己"仪式感
平稳 → 疲惫：工作日晚上10点后 + 当天工作时长>10小时
平稳 → 防御：遭遇负面体验（产品质量/客服/物流问题）
平稳 → 怀旧：传统节日/季节更替/看到童年相关元素

焦虑 → 平稳：完成购买或确认"不会错过"
焦虑 → 防御：发现信息矛盾或感到被欺骗
兴奋 → 疲惫：冲动消费后认知资源耗尽
兴奋 → 焦虑：兴奋期发现"买贵了"或"不需要"
疲惫 → 防御：疲惫时遭遇额外摩擦（客服推诿/操作复杂）
```

## 三机制联用示例

```markdown
【模拟场景】
画像：25岁女，初入职场，年收入5万，焦虑型消费者，FOMO，深夜购物偏好
情境：周三晚上11点，刷抖音看到"限时24小时"护肤品促销

引擎处理流程：
1. 情境触发器匹配：
   - 场景="大促限时" + 促销敏感="高度敏感" → 触发规则4（冲动概率提升）
   - 场景="深夜" + 购买时间偏好="深夜时段" → 激活习惯通路

2. 认知负荷评估：
   - 时间=晚上11点（工作日）→ 认知资源 = 低
   - 当天已工作10小时 → 认知资源再降一级
   - 结果：进入"低认知"模式，依赖单一线索（主播推荐/销量/限时）

3. 情绪状态机：
   - 基线=平稳
   - 深夜+工作日后 → 转换至"疲惫"
   - 看到"限时24小时" + FOMO标签 → 从疲惫转为"焦虑"
   - 焦虑+低认知 = "恐慌性下单"行为模式

【输出行为】
- 不查成分/不比价
- 被"限量1000件已售823件"推动
- 使用花呗分期（降低当下痛感）
- 下单后立即产生"我真的需要吗"的怀疑
- 第二天早上查看物流时，希望它慢一点到（给自己后悔窗口）
```
