# 测试规范

> **版本**：v1.0  
> **定位**：定义消费者模拟系统的完整测试体系，确保生成质量、系统稳定性和工程可靠性  
> **使用说明**：按单元测试 → 集成测试 → 对抗测试 → 性能测试 → 人工盲测的顺序执行

---

## 一、测试金字塔

```
         ▲
        /│\      人工盲测（Golden Standard）
       / │ \     — 研究人员对输出质量做最终评判
      /  │  \
     /───────\   对抗测试（Adversarial Testing）

  /                \ — 端到端流程验证

```

### 2.2 SeedGenerator 测试用例示例

```python
class TestSeedGenerator:
    def test_valid_tension_combination(self):
        """测试正常张力组合"""
        seed = generator.generate_seed(
            scenario="skincare",
            constraints={"life_stage": "初入职场单身"}
        )
        assert 0.3 <= seed.tension_score <= 0.9
    
    def test_low_tension_retry(self):
        """测试低张力时自动重采样"""
        # 强制低张力约束
        with patch.object(generator, 'sample_life_stage', return_value="退休生活"):
            seed = generator.generate_seed(scenario="general")
            # 应该引入矛盾标签提升张力
            assert seed.tension_score >= 0.3
    
    def test_extreme_tension_capped(self):
        """测试过高张力被限制"""
        # 学生 + 年收入100万 这种不现实组合应该被降低概率
        seed = generator.generate_seed(
            constraints={"life_stage": "学生", "income": "100万元以上"}
        )
        assert seed.tension_score <= 0.9
        assert "收入来源" in seed.required_explanations
    
    def test_max_attempts_exceeded(self):
        """测试超过最大尝试次数抛出异常"""
        with patch.object(generator, 'calculate_tension', return_value=0.1):
            with pytest.raises(SeedGenerationError):
                generator.generate_seed(scenario="general")
```

```python
class TestEndToEndProfileGeneration:
    def test_complete_generation_pipeline(self):
        """测试完整画像生成流水线"""
        task = GenerationTask(
            type="BATCH_PROFILE",
            count=5,
            scenario="skincare-new-product",
            quality_gate={"min_authenticity_score": 9}
        )
        
        result = orchestrator.run_batch_generation(task)
        
        assert len(result.profiles) == 5
        assert result.report.summary["passed"] >= 4  # 至少4个通过
        assert result.report.summary["average_score"] >= 9
    
    def test_feedback_loop_convergence(self):
        """测试Feedback Loop能收敛到通过"""
        # 使用有缺陷的种子，强制触发修正
        task = GenerationTask(
            type="BATCH_PROFILE",
            count=1,
            scenario="forced_defect"
        )
        
        result = orchestrator.run_batch_generation(task)
        
        # 要么通过，要么转人工（不是无限循环）
        profile = result.profiles[0]
        assert profile.status in ["PASSED", "HUMAN_REVIEW"]
        assert len(profile.fix_history) <= 3
```

## 四、对抗测试（Adversarial Testing）

### 4.1 偏见防御测试

| 测试ID | 输入 | 预期系统反应 | 通过标准 |
|--------|------|-------------|---------|
| ADV-001 | 种子：退休+节俭+不会用手机+被保健品骗 | BiasDetector触发警告 | 相似度 > 0.7 |
| ADV-002 | 种子：Z世代+冲动+月光+沉迷短视频 | BiasDetector触发警告 | 相似度 > 0.7 |
| ADV-003 | 种子：小镇青年+土味+拼多多+向往都市 | BiasDetector触发警告 | 相似度 > 0.7 |
| ADV-004 | 种子：女性+只看颜值+情绪化+爱购物 | BiasDetector触发警告 | 相似度 > 0.7 |
| ADV-005 | 同一画像库中"退休+节俭"组合出现 > 30% | 多样性检查触发 | 标记高风险 |

### 4.2 逻辑陷阱测试

| 测试ID | 输入 | 预期系统反应 | 通过标准 |
|--------|------|-------------|---------|
| LOG-001 | 学生 + 年收入50万 + 极简主义 | LogicValidator捕获收入-行为不匹配 | RULE-002失败 |
| LOG-002 | 退休 + 育儿焦虑（无解释） | LogicValidator捕获人生阶段-焦虑不匹配 | RULE-003失败 |
| LOG-003 | 深度忠诚 + 转换原因 | LogicValidator捕获品牌关系不一致 | RULE-004失败 |
| LOG-004 | 高收入 + 极简主义 + 促销敏感（无解释） | LogicValidator捕获矛盾无解释 | RULE-001失败 |
| LOG-005 | 场景反应库 < 5 个 | SchemaValidator捕获结构不完整 | 结构校验失败 |

### 4.3 真实感边界测试

| 测试ID | 测试内容 | 预期Agent反应 | 危险信号 |
|--------|---------|--------------|---------|
| REAL-001 | 问"你去年网购花了多少钱？" | "不知道"/"没算过"/"不想说" | 精确报出数字 |
| REAL-002 | 连续问10个产品后问第11个 | 决策质量下降，依赖默认选项 | 保持同等热情 |

| REAL-005 | 最喜欢的品牌曝出丑闻 | 挣扎、渐进疏远 | 立刻发誓永不买 |

### 4.4 压力测试

| 测试ID | 压力条件 | 通过标准 |

## 六、人工盲测（Golden Standard）

### 6.1 测试设计

**目标**：验证 AI 生成的模拟输出是否达到人类研究者认可的真实感水平。

**实验设计**：
- **对照组 A**：带 Harness 的 AI 生成输出（10个画像 × 3个场景 = 30条记录）
- **对照组 B**：不带 Harness 的纯 Prompt 输出（同 30 条记录）
- **对照组 C**：真人访谈记录（脱敏后，同主题，10-15 条）

**评审员**：5 位有 3 年以上经验的消费者研究员，不知道各组来源。

### 6.2 评分维度

与 Authenticity Scorer 的 7 维度一致：

```markdown
1. 内在一致性（0-2分）
2. 情境敏感性（0-2分）
3. 认知有限性（0-2分）
4. 社会摩擦感（0-2分）
5. 时间延续性（0-2分）
6. 语言自然度（0-2分）
7. 知识边界感（0-2分）
```

额外增加：**"你是否相信这是一个真实人类？"**（是/否/不确定）

### 6.3 通过标准

| 指标 | 通过标准 | 优秀标准 |
|------|---------|---------|
| A组平均分 vs B组 | A > B，p < 0.05 | A 比 B 高 ≥ 2分 |
| A组平均分 vs C组 | A 与 C 无显著差异（p > 0.05） | A ≥ C - 1分 |
| A组"相信是真人"比例 | >= 60% | >= 80% |
| 评分者一致性（Krippendorff's α） | >= 0.6 | >= 0.8 |

### 6.4 结果分析模板

```markdown
## 人工盲测报告

### 样本分布
- A组（带Harness）：30条
- B组（纯Prompt）：30条
- C组（真人）：15条

### 评分结果
| 组别 | 平均分 | 标准差 | 中位数 |
|------|--------|--------|--------|
| A组 | 10.8 | 1.2 | 11.0 |
| B组 | 8.3  | 1.8 | 8.0  |
| C组 | 11.5 | 0.9 | 12.0 |

### 统计检验
- A vs B: t=4.52, p<0.001 **显著优于B组**
- A vs C: t=-1.23, p=0.22 **与C组无显著差异**

### "相信是真人"比例
- A组: 73%
- B组: 40%
- C组: 87%

### 主要问题
- A组在"社会摩擦感"维度仍有提升空间（平均分1.3）
- 建议优化防御情绪下的面子保护机制表达
```

## 七、自动化测试流水线

### 7.1 CI/CD 集成

```yaml
# .github/workflows/test.yml 示例
name: Test Pipeline

on: [push, pull_request]

jobs:
  unit-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Unit Tests
        run: pytest tests/unit -v --cov=src --cov-report=xml
      - name: Upload Coverage
        uses: codecov/codecov-action@v3

  integration-test:
    runs-on: ubuntu-latest
    services:
      mongodb:
        image: mongo:6
        ports: ["27017:27017"]
      redis:
        image: redis:7
        ports: ["6379:6379"]
    steps:
      - uses: actions/checkout@v3
      - name: Run Integration Tests
        run: pytest tests/integration -v

  adversarial-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Adversarial Tests
        run: pytest tests/adversarial -v --tb=short

  performance-test:
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v3
      - name: Run Performance Tests
        run: pytest tests/performance -v --benchmark-only
```
