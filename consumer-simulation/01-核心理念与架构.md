# 核心理念与架构

> **版本**：v1.0  
> **定位**：方案的设计哲学与四层驱动模型  
> **使用说明**：理解本方案"为什么这样设计"，再进入具体操作

---

## 一、核心设计理念：从「标签堆叠」到「人格涌现」

传统方案容易犯的错误是让模型"平均随机选标签"，生成的是**统计学意义上的均值人**，而非**有血有肉的真实个体**。

真实消费者的本质特征是：

> **内在矛盾性** + **情境驱动性** + **叙事一致性**

因此本方案要求模型在生成时遵循**"张力优先"原则**——主动选择 2-3 组看似矛盾的标签，并为其构建合理的心理叙事。

---

## 二、方案架构：四层驱动模型

```
┌─────────────────────────────────────────────────────────┐
│  Layer 4: 场景叙事层（Contextual Narrative）              │
│  → 在具体场景中"演出"消费者行为，而非罗列属性              │
├─────────────────────────────────────────────────────────┤
│  Layer 3: 心理动机层（Psychological Engine）              │
│  → 价值观冲突、焦虑来源、防御机制、深层需求                │
├─────────────────────────────────────────────────────────┤
│  Layer 2: 行为模式层（Behavioral Signature）              │
│  → 购买决策路径、价格敏感度、信息处理风格、品牌关系         │
├─────────────────────────────────────────────────────────┤
│  Layer 1: 基础画像层（Demographic Skeleton）              │
│  → 人口统计、社会经济、家庭结构、生命周期阶段               │
└─────────────────────────────────────────────────────────┘
```

### 层级间的解释义务

**关键规则**：上层必须能解释下层的"异常"。

| 层级组合示例 | 需要 Layer 3 提供的解释 |
|-------------|----------------------|
| Layer 1：年收入 30 万 + Layer 2：凑单退单高手 | "成长于节俭家庭，对'被宰'有强烈恐惧，高收入未改变行为惯性" |
| Layer 1：一线城市高管 + Layer 2：拼多多重度用户 | "时间碎片化，对'省小钱'有游戏化快感，非真缺钱" |
| Layer 1：退休老人 + Layer 2：直播购物狂热 | "独居空虚，直播间提供了陪伴感和被重视感" |
| Layer 1：Z世代学生 + Layer 2：买保险型基金 | "目睹父母投资失败，对'确定性'有执念，风险偏好极低" |

如果无法为矛盾组合提供合理解释，则该组合不成立，需要重新采样。

## 三、种子设定策略（Seed Configuration）

不随机均匀抽样，而是先确定**「人生阶段」+「核心焦虑」+「消费能力」**三要素作为种子。这三者的组合天然产生张力：

| 种子组合示例 | 天然张力 | 典型消费行为预判 |
|-------------|---------|----------------|
| 初入职场单身 + 同辈压力 + 年收入 3-8 万 | 欲望 > 能力 | 信贷消费、为社交货币买单、冲动与后悔循环 |
| 养育幼儿 + 育儿焦虑 + 家庭年收入 15-25 万 | 精细化需求 vs 预算约束 | 母婴产品研究狂、母婴社群活跃、对"智商税"高度警惕 |
| 退休生活 + 健康焦虑 + 年收入 3-8 万 | 信息脆弱性 vs 健康诉求 | 易被保健品话术打动、子女劝阻无效、混合着怀疑与渴望 |
| 中年空巢 + 职业倦怠 + 年收入 30-50 万 | 存在性空虚 + 购买力充沛 | 补偿性消费、突然沉迷新爱好、为"重启人生"买单 |
| 数字游民 + 自由独立 + 收入波动大 | 不确定感 vs 身份坚持 | 对"效率工具"极度慷慨、对"稳定资产"极度吝啬 |

### 种子设定的 Prompt 写法

```markdown
请先确定以下三要素：
1. 人生阶段：（从标签体系中选择1个）
2. 核心焦虑：（从标签体系中选择1-2个）
3. 消费能力：（从标签体系中选择1个收入档位）

要求：
- 三者组合必须产生可见张力
- 如果组合过于"和谐"（如：退休+无焦虑+高收入），需主动引入一个矛盾标签
```
