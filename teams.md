## 🎯 系统构建与协作流程（融合安全、合规、成本与业务验收）

### 阶段1：设定安全与业务基准（前置门禁）
1. **小验**主导联合**小联**，撰写《系统成功标准书》，明确业务验收指标与可接受偏差
2. **小伦**制定《虚拟消费者公平性规范》，明确禁止的偏见类型与人口基准数据源
3. **小安**起草《系统安全架构纲要》，划定工具权限与过滤规则
4. **小控**根据预估规模给出单次模拟成本上限，设定预算红线
5. 以上四项文件由**小P**批准后，方可进入下一阶段

### 阶段2：领域设计与行为仿真（融合安全与伦理审查）
1. **小联**输出属性水平与实验设计
2. **小示**与**小伦**共同将方案转化为嵌入公平性硬规则的提示模板
3. **小应**搭建Agent原型，进行小批量消费者模拟
4. **小安**并行开展红队测试，对生成接口进行注入攻击验证
5. **小伦**对仿真样本进行偏见审计，通过后方可推进

### 阶段3：建模管线与工具集成（成本感知）
1. **小数**封装联合分析模型API，**小端**设计工具调用协议
2. **小控**与**小端**协作嵌入计费跟踪与熔断逻辑
3. **小安**审计所有工具API权限与输入校验
4. **小测**在集成测试中加入安全与成本异常测试用例

### 阶段4：全链路串联与业务验收（业务卡口）
1. **小应**、**小端**、**小前**完成消费者生成→作答→建模→看板的全链路串联
2. **小验**组织业务用户UAT，依照《成功标准书》逐项验收
3. 不达标项由**小P**协调回溯，直至满足业务门禁
4. **小培**（如有）同步准备培训材料，对早期用户进行预热

### 阶段5：验证闭环与持续监控（上线与守护）
1. **小联**对比合成结果与先验认知，**小伦**核验全量样本公平性
2. 上线后，**小控**持续监控API费用，**小安**监控异常行为，**小维**保障整体稳定性
3. **小验**追踪实际业务KPI，定期输出《系统价值与健康度报告》
4. **小培**持续收集反馈、开展培训，推动深度采纳

### ⚠️ 关键决策门禁
- **小伦**与**小验**在每阶段结束时拥有一票否决权：若公平性不达标或业务价值存疑，项目不得进入下一阶段
- **小控**的成本熔断机制为自动执行，触发后立即通知**小P**处置
