## 一、系统总览：5分钟快速认知

### 1.1 系统能做什么

AI_CBC 是一个**虚拟消费者联合分析平台**，帮助你在数小时内完成传统调研需要 4-6 周的工作。

```
你的业务问题 ──→ 设定产品属性 ──→ 生成虚拟消费者 ──→ 生成CBC问卷
                                                          ↓
决策建议 ←── 解读报告 ←── 运行分析 ←── 消费者模拟作答 ←──┘
```

### 1.2 核心概念速查

| 术语 | 通俗解释 | 类比 |
|------|---------|------|
| **虚拟消费者** | AI "扮演" 的消费者，有完整画像 | 演员按照剧本表演 |
| **CBC（选择实验）** | 让消费者在多个产品选项中选一个 | 相亲时从几个人里选一个 |
| **效用值** | 对某个属性水平的"好感度分数" | 给餐厅打分 |
| **属性重要性** | 这个属性在决策中的"话语权" | 相亲时你最看重什么 |
| **WTP（支付意愿）** | 愿意为某项升级掏多少钱 | 加钱换更大房子 |
| **市场份额模拟** | 如果市场上有这些产品，大家会怎么选 | 预测选举结果 |

## 二、前端看板操作指南

### 2.1 导航结构

左侧导航栏包含 11 个功能模块：

```
┌─────────────────────────────────────┐
│  AI_CBC 虚拟消费者联合分析          │
├─────────────────────────────────────┤
│  总览              ← 研究项目列表   │
│  创建研究          ← 新建研究入口   │
│  画像管理          ← 查看/生成消费者│
│  问卷配置          ← 查看CBC问卷   │
│  对话实验室        ← 单消费者深度访谈│
│  属性重要性看板    ← 分析结果可视化 │
│  细分群体比较      ← 不同群体对比   │
│  市场份额模拟器    ← What-if 模拟   │
│  分析任务状态      ← 查看分析进度   │
│  作答模拟          ← 触发模拟填写  │
│  系统设置          ← 配置参数      │
└─────────────────────────────────────┘
```

### 2.2 标准操作流程（SOP）

#### 步骤 1：创建研究（创建研究页面）

1. 点击左侧 **"创建研究"**
2. 填写研究ID（如 `dishwasher-2024q3`）
3. 填写产品类别（如 `洗碗机`）
4. 填写研究目标（如：评估消费者对洗碗机各属性水平的偏好）
5. 选择目标人群标签（可多选）
6. 选择属性配置方式：
   - **使用默认属性**：系统预置洗碗机 6 属性（价格、容量、安装方式、功能配置、品牌、能效）
   - **自定义属性**：自行定义属性和水平
7. 点击 **"创建研究并生成问卷"**

> **预期结果**：研究创建成功，问卷自动生成，页面自动跳转到属性设计页

#### 步骤 2：查看/调整属性设计（属性设计页面）

1. 在左侧导航点击 **"问卷配置"**，展开研究列表
2. 点击 **"配置属性"** 进入属性设计页
3. 检查各属性的水平设置是否符合业务需求
4. 如需调整，修改后保存

#### 步骤 3：生成虚拟消费者（画像管理页面）

1. 点击左侧 **"画像管理"**
2. 点击右上角 **"批量生成"**
3. 输入研究ID和生成数量（建议 100-200）
4. 点击 **"开始生成"**

> **检查点**：生成后查看画像列表，确认：
> - 细分群体分布合理
> - 真实性评分 ≥ 9 分（绿色标签）
> - 偏见审计状态为 PASS（绿色标签）

#### 步骤 4：查看问卷（问卷配置页面）

1. 点击左侧 **"问卷配置"**
2. 展开研究行，查看选择集设计矩阵
3. 检查 D-efficiency 值：
   - ≥ 0.85：优秀（绿色）
   - 0.70-0.85：可接受（橙色）
   - < 0.70：需优化（红色）

#### 步骤 5：触发模拟作答（作答模拟页面）

1. 点击左侧 **"作答模拟"**
2. 选择要模拟的虚拟消费者（可多选）
3. 选择模拟模式：
   - **rule 模式**：基于规则快速模拟
   - **LLM 模式**：AI 深度推理模拟（更真实但更慢）
4. 点击 **"开始模拟作答"**

#### 步骤 6：运行分析（分析任务状态页面）

1. 点击左侧 **"分析任务状态"**
2. 选择研究项目和模型类型（默认 HB）
3. 点击 **"启动分析"**
4. 等待任务完成（通常 5-15 分钟）

> **检查点**：任务完成后展开行查看收敛诊断：
> - R-hat Max ≤ 1.1：已收敛（绿色）
> - R-hat Max > 1.1：未收敛（红色），结果不可靠

#### 步骤 7：解读结果（属性重要性看板）

1. 点击左侧 **"属性重要性看板"**
2. 选择研究项目和分析结果
3. 查看三个标签页：
   - **柱状图**：属性重要性排名可视化
   - **数据表格**：精确数值和置信区间
   - **WTP**：支付意愿分析

## 三、结果解读深度指南

### 3.1 效用值（Utility）解读

#### 什么是效用值？

效用值 = **"好感度分数"**

- **正数** = 喜欢（数值越大越喜欢）
- **负数** = 不喜欢（数值越小越不喜欢）
- **零附近** = 无所谓

#### 示例解读

假设"品牌"属性的效用值如下：

| 品牌 | 效用值 | 解读 |
|------|--------|------|
| 美的 | +0.8 | 最受欢迎 |
| 西门子 | +0.5 | 较受欢迎 |
| 海尔 | +0.2 | 略受欢迎 |
| 方太 | -0.1 | 略不受欢迎 |
| 老板 | -0.4 | 最不受欢迎 |

**业务含义**：在价格、功能等其他条件相同时，消费者倾向于选择美的，其次是西门子。

#### 重要注意事项

> **不同属性间的效用值不能直接比较大小！**
>
> 价格的效用值是 -0.05，品牌的效用值是 +0.8，**不代表品牌比价格重要 16 倍**。
> 重要性需要通过"属性重要性"指标来判断。

### 3.2 属性重要性（Attribute Importance）解读

#### 什么是属性重要性？

属性重要性 = **这个属性在消费者决策中的"话语权"**

所有属性的重要性加起来 = **100%**

#### 示例解读

| 属性 | 重要性 | 解读 |
|------|--------|------|
| 价格 | 45% | 最重要的决策因素，近一半决策由价格驱动 |
| 品牌 | 25% | 第二重要，品牌认知影响选择 |
| 容量 | 15% | 中等重要 |
| 安装方式 | 10% | 较不重要 |
| 功能配置 | 4% | 不太重要 |
| 能效 | 1% | 几乎不影响决策 |

#### 业务决策应用

```
如果价格重要性 > 40%：
  → 产品策略应聚焦性价比
  → 定价是核心竞争手段

如果品牌重要性 > 30%：
  → 品牌建设和营销投入回报高
  → 新品牌进入难度大

如果某属性重要性 < 5%：
  → 该属性不是差异化重点
  → 可考虑标准化以降低成本
```

### 3.3 置信区间（Confidence Interval）解读

#### 什么是置信区间？

置信区间 = **"真实值可能所在的范围"**

- 95% CI 表示：如果我们重复做 100 次研究，有 95 次真实值会落在这个区间内
- **区间越窄** = 估计越精确
- **区间越宽** = 估计越不确定

#### 如何看置信区间

在看板的 **"数据表格"** 标签页中，每个属性重要性都有：
- 均值（Mean）
- 95% CI 下限
- 95% CI 上限

#### 判断标准

| 情况 | 判断 | 行动 |
|------|------|------|
| 两个属性的 CI 不重叠 | 差异显著 | 可以据此做决策 |
| 两个属性的 CI 重叠 | 差异不显著 | 不能确定哪个更重要 |
| CI 很宽（> 20%） | 估计不精确 | 增加样本量重新分析 |

#### 示例

```
价格重要性: 45% [40%, 50%]  ← 区间窄，估计精确
品牌重要性: 25% [15%, 35%]  ← 区间较宽，有一定不确定性

结论：价格比品牌重要（CI 不重叠），但品牌重要性的精确度较低
```

### 3.4 WTP（支付意愿）解读

#### 什么是 WTP？

WTP = **消费者愿意为某项升级掏多少钱**

#### 示例解读

| 升级项 | WTP 均值 | 解读 |
|--------|----------|------|
| 容量 6套→8套 | ¥280 | 愿意多花 280 元升级容量 |
| 容量 8套→10套 | ¥350 | 愿意多花 350 元继续升级 |
| 功能 基础→智能 | ¥420 | 智能功能溢价 420 元 |
| 能效 三级→一级 | ¥150 | 愿意为一级能效多付 150 元 |

#### 业务应用

```
产品定价检查：
  如果实际价差 ≤ WTP → 消费者会买账 ✓
  如果实际价差 > WTP → 消费者会犹豫 ✗

示例：
  基础款 2999 元，智能款 3599 元（价差 600 元）
  WTP（基础→智能）= 420 元
  结论：价差 600 > WTP 420，消费者可能觉得不划算
  建议：智能款定价不超过 3419 元（2999 + 420）
```

#### 异常识别

| 异常 | 含义 | 行动 |
|------|------|------|
| WTP 为负数 | 模型异常 | 联系技术支持 |
| WTP 极大（> 产品价 50%） | 可能过度估计 | 谨慎使用，交叉验证 |
| 95% CI 包含 0 | 升级价值不确定 | 不能据此定价 |

### 3.5 市场份额模拟结果解读

#### 什么是市场份额模拟？

模拟不同产品配置在市场上的**相对吸引力**。

> **重要**：这不是真实市场份额预测！它假设消费者只在问卷中的选项中选择，忽略了渠道、促销、口碑等现实因素。

#### 示例解读

假设配置 3 个产品场景：

| 产品 | 配置 | 预测份额 | 95% CI |
|------|------|----------|--------|
| 产品 A | 美的/2999/10套/智能 | 42% | [38%, 46%] |
| 产品 B | 西门子/3499/13套/旗舰 | 35% | [31%, 39%] |
| 产品 C | 海尔/2599/8套/基础 | 23% | [19%, 27%] |

#### 业务应用

```
What-if 分析示例：

场景 1：产品 A 降价到 2799
  → 份额从 42% 提升到 48%（+6%）
  → 结论：降价有效，但需计算利润影响

场景 2：产品 B 增加智能功能（3499→3699）
  → 份额从 35% 降到 31%（-4%）
  → 结论：涨价幅度超过消费者接受度

场景 3：推出新产品 D（方太/3299/13套/智能）
  → 份额 15%，主要从 B 抢夺
  → 结论：市场有空间，但需评估产能
```

#### 使用原则

1. **看方向，不看绝对数值**：份额变化方向比具体数字更重要
2. **相对比较**：产品 A vs 产品 B 的相对优势
3. **不要用于销量预测**：未考虑渠道、营销、品牌力等
4. **CI 宽度判断可靠性**：区间窄的结果更可靠

## 四、常见问题与排查指南

### 4.1 问题速查表

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| **"R-hat > 1.1" 警告** | 模型未收敛 | 增加样本量或检查数据质量 |
| **价格效用值为正** | 模型异常 | 检查价格属性是否正确标记为 price 类型 |
| **WTP 为负数** | 计算异常 | 联系技术支持，可能是价格系数为正 |
| **某属性重要性 100%** | 其他属性可能设计有问题 | 检查属性水平是否足够差异化 |
| **D-efficiency < 0.7** | 实验设计不佳 | 增加选择集数量或调整属性水平 |
| **分析任务失败** | 数据不足或参数错误 | 检查是否有足够作答数据（≥100条） |
| **消费者画像不真实** | 生成参数不当 | 检查目标人群设定是否合理 |
| **偏见审计 FAIL** | 存在系统性偏见 | 联系小伦（伦理审核） |
| **页面加载失败** | 后端服务未启动 | 检查网络连接，刷新页面 |
| **找不到分析结果** | 分析尚未完成 | 在"分析任务状态"页面查看进度 |
