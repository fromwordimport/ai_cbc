## 二、关键指标速查表

### 2.1 模型质量指标

| 指标 | 正常值 | 异常处置 |
|------|--------|---------|
| R-hat | < 1.1 | > 1.1 → 结果不可信，需重新运行 |
| 价格系数 | 负数 | 正数 → 数据异常，排查模型 |
| 偏见审计 | PASSED | FAILED → 联系小伦 |
| ESS | ≥ 100 | < 100 → 估计不稳定 |
| McFadden R² | > 0.1 | < 0.1 → 解释力弱 |

### 2.2 属性重要性解读

| 重要性 | 业务含义 | 行动建议 |
|--------|---------|---------|
| > 40% | 决定性因素 | 优先优化，资源倾斜 |
| 20-40% | 重要因素 | 持续投入，不可忽视 |
| 10-20% | 次要因素 | 有差异化空间 |
| < 10% | 边缘因素 | 谨慎投入 |

### 2.3 效用值解读

| 观察 | 结论 |
|------|------|
| 某水平效用最高 | 消费者最喜欢 |
| 某水平效用最低 | 消费者最不喜欢 |
| 水平间差异大 | 偏好强烈（该属性重要） |
| 水平间差异小 | 不太敏感（该属性不太重要） |
| 价格系数为负 | 正常 |
| 价格系数为正 | 异常，检查模型 |

### 2.4 WTP 决策速查

| WTP vs 成本 | 决策 |
|-------------|------|
| WTP > 成本 | 升级有利可图，建议推出 |
| WTP ≈ 成本 | 盈亏平衡 |
| WTP < 成本 | 升级亏损，不建议 |
| WTP < 0 | 消费者排斥该升级 |

### 2.5 实验设计参数速查

| 参数 | 推荐值 | 最小值 | 说明 |
|------|--------|--------|------|
| 属性数量 | 4-7个 | 3个 | 包含价格 |
| 每属性水平 | 3-4个 | 2个 | 不超过6个 |
| 每人题目数 | 12题 | 8题 | 不超过16题 |
| 每群体样本 | ≥ 80人 | 50人 | 群体级分析 |
| 总样本量 | ≥ 240 | 100人 | 越多样越稳 |

## 三、常见问题一句话解答

| # | 问题 | 一句话答案 |
|---|------|-----------|
| 1 | 虚拟消费者像真人吗？ | 盲测相似度 75-80%，定位是真实调研的前置验证和补充。 |
| 2 | 效用值正负代表什么？ | 正数=消费者喜欢，负数=不喜欢，绝对值越大影响越强。 |
| 3 | 属性重要性什么意思？ | 消费者做选择时最看重什么，百分比越高越重要。 |
| 4 | WTP 能直接用来定价吗？ | 不能，是消费者意愿参考，还需考虑成本、竞争、品牌定位。 |
| 5 | 模型不收敛怎么办？ | 增加样本量、增加题目数、减少属性水平，还不收敛联系小数。 |
| 6 | 结果与预期不符怎么办？ | 先检查 R-hat，再和团队讨论，可能是惊喜发现而非错误。 |
| 7 | 一次研究多少钱？ | 约 ¥300-500（240人×12题），单消费者成本约 ¥2-4。 |
| 8 | 需要哪些属性？ | 加上价格，建议 4-7 个消费者真正关心的决策维度。 |
| 9 | 什么是禁止组合？ | 现实中不可能出现的属性搭配，如"高端品牌+最低价格"。 |
| 10 | 报告能导出吗？ | 支持 PDF / Markdown / Excel 导出，图表可直接放入 PPT。 |
| 11 | 系统支持哪些产品？ | 家电、数码、软件服务、虚拟商品等，几乎任意实体或虚拟产品。 |
| 12 | 虚拟消费者有偏见吗？ | 有自动偏见审计，CRITICAL 级偏见会自动拒绝并重新生成。 |
| 13 | 结果多久能出来？ | 全流程约 1-2 小时（批量操作有进度条可监控）。 |
| 14 | 发现 Bug 怎么办？ | 点击右下角"报告问题"，系统自动截屏，72小时内回复。 |
| 15 | 如何开始第一次使用？ | 联系小培预约 30 分钟带教，之后可独立操作。 |

## 四、结果解读四步法

```
  Step 1: 看质量 → R-hat < 1.1？ 价格系数为负？ 偏见审计 PASSED？
              ↓ 三项都通过
  Step 2: 看重要性 → 哪个属性排第一？ 哪些属性占比超过 20%？
              ↓
  Step 3: 看效用值 → 每个属性下哪个水平最受欢迎？ 差异大吗？
              ↓
  Step 4: 看 WTP → 哪些升级划算？ 哪些不值得做？
```

## 五、异常情况处置速查

| 现象 | 可能原因 | 处置 |
|------|---------|------|
| R-hat > 1.1 | 样本不足/属性过多 | 增加样本或减少水平数 |
| 价格系数为正 | 编码错误 | 联系小数排查 |
| "都不选" > 50% | 产品配置过于极端 | 调整属性水平 |
| WTP 异常大 | 价格系数接近 0 | 检查模型收敛，扩大价格区间 |
| 重要性全部相近 | 属性水平差异不够大 | 拉大水平间差距 |
| 群体差异很小 | 群体定义太相似 | 加强画像差异 |
| 成本超预算 | 样本量过大 | 先从小样本开始 |

## 七、核心概念速记

```
属性(Attribute) = 产品的特征维度（如价格、品牌、容量）
水平(Level)     = 属性下的选项（如 ¥2999、西门子、10套）
选择集(Choice Set) = 一道选择题（3个产品+都不选）

效用值(Utility)     = 吸引力分数  → 正=喜欢 负=不喜欢
重要性(Importance)  = 决策权重    → 百分比 总和100%
WTP                 = 愿意多付的钱 → 成本对比 → 决策依据

Effects Coding = 编码规则（参数和为0，最后一个水平由负和恢复）
Convergence    = 模型可信度    → R-hat < 1.1 才可信
```
