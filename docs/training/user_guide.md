## 一、系统简介

### 1.1 AI_CBC 是什么

AI_CBC 是一个"虚拟消费者联合分析平台"。

**通俗理解**：
> 传统的联合分析研究，需要找200-500个真人消费者一个个做问卷，耗时数周、花费数万元。AI_CBC 用大语言模型（LLM）生成"虚拟消费者"——他们有姓名、有性格、有偏好、有故事，像真人一样回答产品选择问卷。系统自动统计分析他们的选择，生成完整的市场洞察报告。

### 1.2 它解决什么问题

| 你面临的困难 | AI_CBC 如何帮你 |
|-------------|----------------|
| 想做消费者调研但预算有限 | 单次研究成本控制在 ¥500 以内，全流程 2 小时内完成 |
| 新产品还没上市，找不到真实用户 | 按需生成目标人群的虚拟消费者，分钟级就绪 |
| 想知道"如果改变配置，市占率会怎么变" | 市场模拟功能，实时调整参数看份额变化 |
| 分析报告太专业，看不懂统计术语 | LLM 自动撰写通俗解读，直接可用于内部汇报 |
| 想探索小众群体但招募不到 | 生成真实世界中难以招募的细分人群 |

### 1.3 系统不是万能的

AI_CBC 是**决策辅助工具**，不是决策替代工具。

- **可以用的场景**：产品概念快速筛选、定价策略探索、属性重要性排序、小众人群需求挖掘、假设快速验证
- **不能用的场景**：最终上市定价的唯一依据、精确销量预测、法律合规决策、涉及健康安全的关键决策

**推荐用法**：AI_CBC 结果 + 专家判断 +（如有条件）真实消费者研究交叉验证 = 最终决策

### 1.4 系统怎么工作的（一张图看懂）

```
你的需求（例如："我想知道消费者对洗碗机价格和容量的偏好"）
          │
          ▼
  ① 系统创建研究，定义属性和水平
          │
          ▼
  ② 自动生成虚拟消费者（具名、有性格、有故事）
          │
          ▼
  ③ 虚拟消费者逐题做出产品选择（就像真人填问卷）
          │
          ▼
  ④ 统计建模：分析"为什么选这个不选那个"
          │
          ▼
  ⑤ 生成报告：属性重要性排名、消费者愿意为哪些特性多花钱、市场模拟
```

### 1.5 首个验证场景：洗碗机产品定价研究

本手册以洗碗机为例（系统的首个验证场景），但系统支持各种产品类型——家电、数码产品、软件服务、虚拟商品等。

洗碗机案例中的六个属性：

| 属性 | 具体含义 | 可选值举例 |

## 二、核心概念（用通俗语言）

这一章帮你理解报告中会出现的术语。即使不完全理解背后的统计学，也足以看懂报告结论。

### 2.1 属性和水平

| 概念 | 通俗解释 | 类比 |
|------|---------|------|
| **属性（Attribute）** | 产品的某个特征维度 | 就像买手机时要考虑的"屏幕大小""内存""颜色" |
| **水平（Level）** | 某个属性下的具体选项 | "屏幕大小"这个属性下的"6.1寸""6.7寸" |
| **选择集（Choice Set）** | 一道选择题 | 给你三个不同配置的洗碗机，让你挑一个最喜欢的 |

**举例**：一道典型的选择题

```
如果市场上只有这三款洗碗机，你会选哪个？

┌──────────────────┬────────────┬────────────┬────────────┐
│                  │   选项 A   │   选项 B   │   选项 C   │
├──────────────────┼────────────┼────────────┼────────────┤
│ 价格             │  ¥3,999    │  ¥4,999    │  ¥2,999    │
│ 容量             │  10套      │  13套      │  6套       │
│ 安装方式         │  嵌入式     │  水槽式     │  台式       │
│ 核心功能         │  智能洗    │  全能      │  基础       │
│ 品牌             │  方太      │  西门子     │  美的       │
│ 能耗             │  一级      │  超一级     │  二级       │
└──────────────────┴────────────┴────────────┴────────────┘

□ 选项 A    □ 选项 B    □ 选项 C    □ 都不选
```

每个虚拟消费者要做12道这样的题，每道题三个选项不同（由系统自动组合生成）。

### 2.2 效用值（Part-Worth Utility）

**一句话定义**：某个属性水平对消费者的"吸引力分数"。

- **正数** = 消费者喜欢这个水平
- **负数** = 消费者不喜欢这个水平
- **绝对值越大** = 这个水平对决策影响越大

**通俗类比**：就像给每个产品特征"打分"。分数越高，消费者越喜欢。

```
举个例子：
  价格 ¥2,999 → +1.2分（便宜，喜欢）
  价格 ¥3,999 → +0.3分（还行）
  价格 ¥4,999 → -0.8分（贵了，不喜欢）
  
  解读：价格越低，消费者越喜欢（符合常识）。
```

**关键提醒**：
- 效用值只有相对比较才有意义，+2 不一定是 +1 的"两倍好"
- 看符号（正负）和排序比看具体数值更重要
- 价格效用必须是负数——如果出现正数，说明数据有问题

### 2.3 属性重要性（Attribute Importance）

**一句话定义**：做选择时，消费者最看重哪个属性。

属性重要性是一个百分比（所有属性加起来等于100%）。数值越大的属性，对消费者的购买决策影响越大。

```
示例输出：
  价格: 42%  ████████████████████
  容量: 25%  ████████████
  品牌: 17%  ████████
  功能: 16%  ████████
  
  解读：价格是最重要的决策因素，占42%的权重。容量排第二。
```

**业务意义**：

| 重要性 | 业务含义 | 你应该怎么做 |
|--------|---------|-------------|
| > 40% | 决定性因素 | 优先优化这个属性，资源向其倾斜 |
| 20-40% | 重要因素 | 不可忽视，保持关注 |
| 10-20% | 次要因素 | 有差异化空间，选择性投入 |
| < 10% | 边缘因素 | 不必投入太多资源 |

### 2.4 支付意愿（WTP）

**一句话定义**：消费者愿意为某项属性升级多付多少钱。

**通俗理解**：如果从"6套"升级到"10套"容量的 WTP = ¥800，意思是——消费者愿意为了大一倍的容量多花800元。

**业务应用**：

| 如果你的情况是... | 业务含义 |
|------------------|---------|
| WTP > 升级成本 | 升级有利可图，建议做 |
| WTP ≈ 升级成本 | 刚好盈亏平衡 |
| WTP < 升级成本 | 升级亏钱，不建议做 |
| WTP 是负数 | 消费者不仅不愿付费，还需要降价才接受 |

### 2.5 模型收敛（Convergence）

**一句话定义**：统计模型的"计算结果可信度检查"。

用 R-hat 指标衡量（类比：让多个人独立数同一堆钱，看他们数的结果是否一致）：

| R-hat 值 | 含义 | 结果能不能用 |
|-----------|-----|-------------|
| < 1.05 | 非常好，结果高度可信 | 放心使用 |
| 1.05-1.10 | 可以，结果基本可信 | 可以使用 |
| > 1.10 | 未通过，结果不可信 | 需要重新运行 |

### 2.6 关键概念一句话速记

| 概念 | 一句话 |
|------|--------|
| 属性 | 产品的某个特征维度（如价格、容量） |
| 水平 | 属性下的具体选项（如 ¥2999、10套） |
| 效用值 | 吸引力分数，正=喜欢，负=不喜欢 |
| 属性重要性 | 决策权重，这个属性在消费者心里有多重要 |
| WTP | 愿意多花多少钱换更好的属性 |
| 模型收敛 | 计算结果靠不靠谱的体检报告 |

## 三、操作流程

### 3.1 整体流程图

```
第1步              第2步              第3步              第4步
创建研究     →     配置属性     →      启动分析     →    查看报告
（10分钟）         （15分钟）           （系统自动）        （5分钟）

                               ┌─ 虚拟消费者生成
                               ├─ 问卷作答
        系统自动执行            ├─ 统计建模
                               ├─ 质量检查
                               └─ 报告生成
```

### 3.2 步骤1：创建研究

**你要做的事**：定义研究的"基本参数"

1. **填写研究名称**：例如"2026年6月洗碗机定价研究"
2. **选择产品类别**：从下拉列表选择（洗碗机、智能手表、扫地机等）或输入自定义
3. **填写研究目标**：一句话描述你想知道什么，例如"确定洗碗机最优定价区间，了解容量和品牌哪个更重要"
4. **设定目标人群**：选择你要研究的目标细分群体（可多选）

   | 可配置项 | 说明 | 示例 |
   |---------|------|------|
   | 年龄范围 | 目标消费者的年龄 | 25-35岁 |
   | 城市层级 | 一二线城市/下沉市场 | 一线/新一线 |
   | 收入区间 | 家庭/个人年收入 | 15-50万 |
   | 家庭结构 | 独居/已婚/有娃 | 已婚有子女 |
   | 生活价值观 | 消费观念取向 | 精致品质/实用主义 |

5. **设定样本量**：建议每群体 80-100 人，总样本量 240-300 人
6. **点击"创建"**，系统生成一个研究编号。

### 3.3 步骤2：配置属性与水平

**你要做的事**：告诉系统你的产品由哪些"可选项"组成

系统支持"属性发现"模式（推荐）和"手动配置"模式：

**模式A：属性发现（推荐给首次使用者）**

1. 输入产品的自然语言描述，例如："一款面向年轻家庭的洗碗机"
2. 系统自动分析并建议属性/水平列表
3. 你确认或调整建议的属性/水平
4. 系统自动生成实验设计方案

**模式B：手动配置（高级用户）**

1. 逐个添加属性（点击"添加属性"）
2. 为每个属性添加水平（每个属性2-6个水平，推荐3-4个）
3. 指定属性类型：
   - **价格型**：系统自动作为连续变量处理
   - **分类型**：如品牌、颜色、安装方式
4. 可选：设置禁止组合

   某些属性组合在现实中不可能出现，需要手动排除：

   | 禁止组合示例 | 原因 |
   |-------------|------|
   | 高端品牌 + 最低价格 | 品牌定位不符 |
   | 大容量 + 非嵌入式 | 物理限制 |
   | 旗舰功能 + 最低价格 | 成本不支持 |

5. 确认属性配置，点击"生成实验设计"

**关键提醒**：
- 属性数量建议 4-7 个，太少失去分析价值，太多消费者认知过载
- 每个属性的水平数建议 2-4 个
- 价格一定是其中一个属性，系统必须知道价格才能计算 WTP

### 3.4 步骤3：启动分析

**你要做的事**：确认配置，点击"启动"

系统会自动执行以下全流程（全程约 1-2 小时）：

1. **画像生成**（约 10 分钟）：根据你设定的人群参数，生成具名虚拟消费者
2. **偏见检测**（约 2 分钟）：自动检查虚拟消费者是否存在偏见（匿名进行）
3. **问卷作答**（约 30-60 分钟）：每个虚拟消费者依次回答 12 道选择题
4. **数据清洗**（约 2 分钟）：检查数据完整性、排除异常
5. **统计建模**（约 10 分钟）：分层贝叶斯模型拟合
6. **质量诊断**（约 1 分钟）：检查模型是否收敛、价格系数是否正常
7. **报告生成**（约 3 分钟）：自动撰写解读和图表

**期间你可以做的事**：
- 在"进度监控"页面查看实时状态
- 收到异常通知（如某个画像生成失败、成本超出预警）
- 中途可以暂停或取消（不推荐，可能导致数据不完整）

### 3.5 步骤4：查看与使用报告

分析完成后，系统会通知你。报告包含以下部分：

**报告章节导航**：

| 章节 | 内容 | 你要关注什么 |
|------|------|-------------|
| 执行摘要 | 一页纸核心结论 | 快速了解最重要的3-5个发现 |
| 属性重要性分析 | 各属性的决策权重排名 | 消费者最看重什么 |
| 效用值分析 | 每个属性水平的吸引力 | 哪个具体水平最受欢迎 |
| 群体细分对比 | 不同人群的偏好差异 | 是否需要差异化产品策略 |
| 支付意愿（WTP） | 属性升级的货币价值 | 定价和配置优化的依据 |
| 市场份额模拟 | 不同产品组合的市场份额预测 | What-if 分析 |
| 数据质量报告 | 模型收敛、样本量等 | 结果可信度确认 |

**如何使用报告**：
- 执行摘要可直接放入你的内部汇报PPT
- 属性重要性图可作为产品优先级排布的依据
- 市场模拟器支持"拖拉拽"调整产品配置，实时看份额变化
- 报告支持导出 PDF / Markdown / 数据表（Excel）

### 3.6 "What-If" 市场模拟（高级功能）

报告页面提供一个交互式市场模拟器：

1. 拖拽调整各属性水平，组合成 2-4 款"候选产品"
2. 系统实时计算每款产品的预测市场份额
3. 可以看到"如果推出一款 ¥3,999 + 10套 + 嵌入式 + 西门子"的产品，市占率是多少
4. 支持添加"竞品"进行竞争模拟

## 四、如何解读结果

### 4.1 第一步：先看报告"可信不可信"

拿到报告后，先检查三个关键指标（通常在报告开头的数据质量部分）：

| 检查项 | 正常值 | 如果异常... |
|--------|--------|------------|
| 模型收敛性（R-hat） | < 1.1 | 结果不可信，需要重新运行（联系技术支持） |
| 价格系数方向 | 负数 | 结果不可信（极少出现，如出现需排查数据） |
| 偏见审计状态 | PASSED | 结果不可用，联系小伦（伦理审计） |

**如果三个指标都正常**，继续往下看。

### 4.2 第二步：看属性重要性排名

这是报告中最重要的部分。它直接告诉你：**消费者在做购买决策时，最看重什么**。

```
示例解读：
  价格: 41.7%  ████████████████████  ← 最重要
  容量: 25.0%  ████████████          ← 第二重要
  品牌: 16.7%  ████████              ← 并列第三
  功能: 16.7%  ████████              ← 并列第三

  业务结论：
  - 定价策略是你最强的竞争杠杆（占 42% 的权重）
  - 提供合适的容量是第二重要的差异化方向
  - 品牌和功能的影响力有限
```

**注意看群体差异**：

不同消费者群体的重要性排名通常不同。如果某个属性在不同群体间差异超过 15 个百分点，说明存在明显的细分机会——不要"一刀切"。

```
示例：
              价格重要性    品牌重要性
  精致白领     35%           25%        ← 品牌敏感，价格相对不敏感
  Z世代租客    55%           10%        ← 极度价格敏感

  业务含义：精致白领愿意为品牌溢价买单，Z世代租客只认性价比
  → 可能需要两款差异化的产品
```

### 4.3 第三步：看效用值——哪个具体水平最受欢迎

在了解了"什么属性最重要"之后，再看看"每个属性下哪个具体选项最好"。

**效用值解读口诀**：
- 正数 = 喜欢，负数 = 不喜欢
- 同属性内比较：看哪个水平效用最高（最受欢迎）
- 同属性内看差距：差距大说明消费者对这一属性的偏好很强烈

```
示例：
  品牌效用值：
    西门子 → +0.9  ← 最受欢迎
    方太   → +0.4
    美的   → -0.1
    小米   → -0.3  ← 最不受欢迎

  解读：在洗碗机这个品类，西门子的品牌吸引力最强，小米相对较弱。
```

### 4.4 第四步：看支付意愿（WTP）——指导定价

WTP 告诉你：**消费者愿意为"更好的配置"多掏多少钱**。

```
示例：
  容量升级 WTP（6套→10套）= ¥800
  → 如果升级成本是 ¥500，那升级有利可图
  → 如果升级成本是 ¥1000，那升级亏损，不如保持 6 套

  品牌升级 WTP（美的→西门子）= ¥350
  → 如果西门子的采购/授权成本比美的贵不到 ¥350，可以考虑
  → 但如果要多花 ¥500，就不值得
```

**WTP 与成本的比较决策表**：

| 升级项 | WTP | 预估成本 | 利润空间 | 决策 |
|--------|-----|---------|---------|------|
| 容量 6→10套 | ¥800 | ¥500 | +¥300 | 建议推出 |
| 品牌 美的→西门子 | ¥350 | ¥600 | -¥250 | 不建议 |
| 功能 基础→智能 | ¥200 | ¥150 | +¥50 | 可考虑 |

### 4.5 第五步：市场模拟——产品组合优化

市场模拟是报告中最"好玩"的部分。你可以：

1. 选出你觉得最好的 2-3 款产品配置
2. 系统告诉你每款的预测市场份额
3. 调整价格或配置，看份额如何变化
4. 找到"既好卖又赚钱"的最优配置

```
示例输出：
  场景：市面同时有这三款产品

  产品A：¥3,999 + 10套 + 嵌入式 + 智能 + 西门子 + 一级 → 市占率 38%
  产品B：¥2,999 + 6套  + 台式   + 基础 + 美的   + 二级 → 市占率 35%
  产品C：¥5,999 + 13套 + 嵌入式 + 全能 + 方太   + 超一级 → 市占率 27%

  如果你调整产品A的价格从 ¥3,999 降到 ¥3,499：
  产品A 份额从 38% 上升到 46%，从产品B和产品C各抢走一部分
```

### 4.6 常见结果解读误区

| 误区 | 纠正 |
|------|------|
| "效用值最高的属性水平就应该做" | 效用值只反映消费者偏好，不反映成本和可行性 |
| "WTP = 定价建议" | WTP 是消费者意愿的上限，定价还要考虑成本、竞争、品牌定位 |
| "属性重要性排序就是优先级排序" | 重要性高代表消费者在意，但还要结合"改进空间"和"改进成本" |
| "市场份额模拟就是真实销量预测" | 模拟基于消费者偏好，未考虑渠道、营销、竞争反应等因素 |

## 五、常见问题

### 5.1 关于虚拟消费者

**Q: 虚拟消费者真的能像真人一样做选择吗？**

虚拟消费者不是真人的"复制品"，而是基于大量真实消费者行为数据（LLM 训练语料中的消费行为模式）生成的"行为模拟器"。在盲测中，其选择模式与真人消费者的相似度达到 75-80% 以上。

更重要的是理解它的定位：**虚拟消费者研究是真实调研的"前置验证"和"补充探索"**，而非替代。

**Q: 虚拟消费者会不会有"AI 味"？**

系统设计了七维度真实性评价体系，包括内在一致性、认知有限性、社会摩擦感等。每个虚拟消费者都经过多轮审校，确保：
- 有矛盾的偏好（如"想买好的但舍不得花钱"），并给出心理层面的解释
- 会有认知局限（不会对所有产品都表现专业）
- 语言自然（不使用营销术语或学术词汇）

如果发现"AI 味"过重的情况，可以通过反馈渠道报告。

**Q: 虚拟消费者会不会有偏见？**

系统有内置的偏见审计管线，每批虚拟消费者生成后自动运行七项公平性检查，包括：
- 偏好是否与受保护属性（性别、年龄等）产生系统性关联
- 群体的多样性是否充足

如果检测到 CRITICAL 级别的偏见，系统会自动拒绝该批次并要求重新生成。

**Q: 如果我对分析结果有疑虑，怎么验证？**

推荐以下几个验证步骤：
1. **直觉检查**：结果的总体方向是否符合你的业务直觉？（如价格越高偏好越低）
2. **专家评审**：找团队里的联合分析专家（小联）或建模科学家（小数）复核
3. **交叉验证**：如果条件允许，用一小批真实消费者数据做对比
4. **假设检验**：尝试修改一个参数（如调整价格区间），看结果是否发生合理的变化

### 5.2 关于实验设计

**Q: 我应该设几个属性？几个水平？**

| 参数 | 建议范围 | 说明 |
|------|---------|------|
| 属性数量 | 4-7 个 | 太少分析无意义，太多消费者认知过载 |
| 每属性水平数 | 2-4 个 | 2 个太简单，5+ 个会让问卷变长 |
| 每人题目数 | 8-16 题 | 系统默认 12 题，足够且不会疲劳 |
| 总样本量 | 每群体 ≥ 80 人 | 如需群体级分析，每群体至少 80 人 |

**Q: 什么是"禁止组合"？必须设置吗？**

禁止组合是指现实中不可能出现的属性搭配。例如：
- 西门子（高端品牌）搭配 ¥2,999（入门价格）—— 现实中不存在
- 13 套大容量 + 台式安装 —— 物理限制不允许

不设置禁止组合，虚拟消费者可能面临"不合逻辑的选择题"，影响数据质量。**强烈建议设置**。

**Q: 我的产品属性在不同市场可能不同，怎么办？**

系统支持分市场（或分群体）设定部分属性的水平。例如：
- 在一线城市，价格区间设为 ¥3,999-5,999
- 在下沉市场，价格区间设为 ¥2,499-3,999

然后在分析时做市场间的交叉对比。

### 5.3 关于模型结果

**Q: 报告显示"模型未收敛"怎么办？**

如果 R-hat > 1.1，说明模型计算未收敛，结果不可信。可能的原因和解决：

| 原因 | 解决方案 |
|------|---------|
| 样本量太小 | 增加虚拟消费者数量（建议每群体 ≥ 80 人） |
| 选择集太少 | 增加每人题目数（建议 12-16 题） |
| 属性水平过多 | 减少部分属性的水平数 |
| "都不选"比例过高 | 检查产品配置是否过于极端 |
| 价格区间过窄 | 拉大价格跨度 |

如果以上调整后仍然不收敛，联系小数（数据/建模科学家）。

**Q: 属性重要性排名和我的预期不一样，说明什么？**

不一定是坏事。可能有几种情况：

1. **可能是惊喜发现**：消费者实际偏好和我们的假设不同（这正是研究价值所在）
2. **可能是设计原因**：你的属性水平设定范围可能影响了重要性计算。例如，如果价格区间设得太宽，价格的重要性会被放大
3. **可能是样本原因**：生成的虚拟消费者群体可能需要调整

建议先和团队一起讨论，看差异是否有合理的业务解释。

**Q: WTP 计算出来的金额太高/太低怎么办？**

WTP 的合理性取决于两个因素：
- **价格系数是否正常**：如果价格系数接近 0，WTP 会异常大。检查模型是否收敛
- **业务基准是否合理**：将 WTP 与行业数据、竞品定价、内部成本做交叉比对

如果 WTP 偏差超过 30%，建议在报告中使用时标注"估算值"，不作为精确定价依据。

### 5.4 关于成本

**Q: 每次研究大概花多少钱？**

- 一次标准研究（240 个虚拟消费者，12 道题）的 LLM API 调用成本约 ¥300-500
- 每位虚拟消费者的平均成本约 ¥2-4
- 系统设有成本熔断机制，超出预算自动暂停

**Q: 如何控制成本？**

- 不要设置过大的样本量（先从小样本开始验证方向）
- 利用"属性发现"功能减少人工反复调试
- 对于探索性研究，先用 50-100 人的小样本，确认方向后再扩量

### 6.1 我怎么报告问题？

系统的持续改进依赖于你的反馈。如果你在使用中遇到任何问题，请通过以下方式反馈：

- 在报告页面点击"请求复查"，系统会重新运行模型并生成对比报告
