## 一、检查清单使用说明

### 1.1 何时使用

- **实验设计初稿完成后**：自检设计是否合理
- **实验上线前**：最终确认无遗漏
- **跨团队协作时**：作为沟通标准
- **项目复盘时**：对照检查改进点

### 1.2 检查流程

```
阶段1：属性设计检查 → 阶段2：实验设计检查 → 阶段3：样本设计检查
        ↓                      ↓                      ↓
   属性筛选原则            设计参数合理性          样本量充足性
   水平定义完整性          禁止组合检查            群体覆盖度
   编码方案确认            D-efficiency目标        与画像系统联动
```

## 三、阶段2：实验设计检查

| 参数 | 设定值 | 标准范围 | 是否通过 | 备注 |

| 位置均衡 | 是 | 必须 | [ ] | |
| 同集不重复 | 是 | 必须 | [ ] | |

### 3.2 样本量计算验证

```
总参数数 = 价格(1) + 容量(2) + 安装(2) + 功能(2) + 品牌(3) + 能耗(2) = 12个参数

最小样本量公式：
n_respondents × n_choice_sets ≥ n_params × 5
n × 12 ≥ 12 × 5
n ≥ 5人（理论最小）

实际推荐：
- 基础估计：n ≥ 100
- 细分分析：每群体 n ≥ 80
- 本方案：3群体 × 80人 = 240人
- 总选择观察数：240 × 12 = 2880个选择
```

| 检查项 | 计算结果 | 是否通过 |
|--------|---------|---------|
| 理论最小样本量 | 5人 | [ ] |
| 推荐基础样本量 | ≥100人 | [ ] |
| 细分群体样本量 | 每群体≥80人 | [ ] |
| 总选择观察数 | ≥参数×5 | [ ] |

### 3.3 编码方案检查

| 属性 | 编码方式 | 参数数 | 检查项 | 是否通过 |
|------|---------|--------|--------|---------|
| 价格 | 连续 | 1 | 价格作为连续变量 | [ ] |
| 容量 | Effects coding | 2 | 3水平→2参数 | [ ] |
| 安装方式 | Effects coding | 2 | 3水平→2参数 | [ ] |
| 核心功能 | Effects coding | 2 | 3水平→2参数 | [ ] |
| 品牌 | Effects coding | 3 | 4水平→3参数 | [ ] |
| 能耗等级 | Effects coding | 2 | 3水平→2参数 | [ ] |

**总参数数验证**：1 + 2 + 2 + 2 + 3 + 2 = **12个参数**

**Effects coding检查要点**：
- [ ] 最后一个水平的效用 = -(其余参数之和)
- [ ] 所有参数之和为0
- [ ] 命名规范：`{attribute_id}_{level_index}`，level_index范围0到n_levels-2
