# 虚拟消费者验证逻辑白皮书

> **版本**：v1.0
> **定位**：面向项目团队与业务方的培训材料，系统讲解"如何判断虚拟消费者是否足够真实"的验证逻辑
> **编写者**：小培（变更管理与培训专家）
> **依据文档**：《虚拟消费者真实性验收标准》、《伦理与偏见审计规范》
> **目标读者**：小联、小数、小示、小应、小验，以及首次接触本系统的业务方

---

## 一、为什么需要这份白皮书

虚拟消费者是 AI_CBC 系统的"原材料"。如果原材料不真实，后续所有分析结果都不可信。

本白皮书回答三个核心问题：

1. **什么是"真实"** —— 我们用什么标准衡量虚拟消费者像不像真人
2. **如何验证** —— 从自动化检验到人工盲测的完整验证链路
3. **发现问题怎么办** —— 质量门禁与处置流程

---

## 二、真实性定义：七维度模型

### 2.1 核心框架

虚拟消费者的真实性不是"感觉像"，而是**七个可独立评估维度的综合得分**。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    虚拟消费者真实性七维度模型                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐             │
│  │ ① 内在一致性     │  │ ② 情境敏感性     │  │ ③ 认知有限性     │             │
│  │  矛盾有解释      │  │  不同场景反应不同 │  │  会疲劳、会敷衍  │             │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘             │
│                                                                             │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐             │
│  │ ④ 社会摩擦感     │  │ ⑤ 时间延续性     │  │ ⑥ 语言自然度     │             │
│  │  犹豫、面子、矛盾 │  │  过去影响现在    │  │  语言符合身份    │             │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘             │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ ⑦ 知识边界感                                                         │   │
│  │  对超出认知领域表现真实无知或漠不关心                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 评分规则

每个维度按 **0-2分** 评分：

| 分数 | 等级 | 含义 |
|------|------|------|
| **0分** | 不合格 | 明显违反该维度核心原则，不可接受 |
| **1分** | 部分合格 | 基本符合，但存在可改进的瑕疵 |
| **2分** | 优秀 | 完全符合，甚至超出预期 |

**总分与处置**：

| 总分范围 | 评级 | 处置 |
|---------|------|------|
| 12-14分 | 优秀 | 可直接入库使用 |
| 9-11分 | 合格 | 可通过，建议优化 |
| 7-8分 | 需审查 | 需人工复核，决定是否可用 |
| < 7分 | 不合格 | 拒绝入库，需重新生成或修正 |

## 三、各维度详解与检查清单

### 3.1 维度1：内在一致性（Internal Consistency）

**核心问题**：这个消费者的画像逻辑自洽吗？矛盾组合有解释吗？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 矛盾标签解释 | 所有 tension_combination 必须附 ≥30字解释 | 规则引擎 |
| 收入-消费匹配 | 低收入高消费必须有不寻常的合理解释 | 规则引擎 |
| 人生阶段-焦虑匹配 | 退休+育儿焦虑需特殊说明 | 规则引擎 |
| 品牌忠诚度一致性 | 深度忠诚者不应有品牌转换原因 | 规则引擎 |
| 时间线因果链 | present行为至少有一条可由past解释 | LLM辅助 |

**常见失败案例**：

- ❌ "月收入5000元，但每年买3个最新款iPhone" —— 无解释
- ✅ "月收入5000元，但每年买3个最新款iPhone，因为她靠二手交易平台倒卖手机赚取差价" —— 有解释

**自动评分规则**：
- 5项全部通过 → 2分
- 3-4项通过 → 1分
- 少于3项通过 → 0分

---

### 3.2 维度2：情境敏感性（Contextual Sensitivity）

**核心问题**：同一消费者在不同情境下反应是否差异化、非模板化？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 场景反应差异化 | 至少5个场景反应不重复 | 规则引擎 |
| 情境驱动行为 | 情境变化导致行为变化 | LLM辅助 |
| 非模板化 | 场景反应不能直接从标签推导 | LLM辅助 |

**常见失败案例**：

- ❌ "促销场景：我买；压力场景：我买；日常场景：我买" —— 无差异
- ✅ "促销场景：会囤货但只囤消耗品；压力场景：完全不想购物；日常场景：理性比价" —— 有差异

---

### 3.3 维度3：认知有限性（Cognitive Boundedness）

**核心问题**：消费者是否表现出"人"的认知局限，而非超理性？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 决策深度合理 | 疲惫/低认知资源状态下不过度分析 | 规则引擎 |
| 信息边界 | 对超出日常经验的产品表现出合理无知 | LLM辅助 |
| 决策瑕疵 | 偶尔做出事后看来"不完美"的决定 | LLM辅助 |

**关键原则**：真实人类会疲劳、会敷衍、会凭直觉做决定。虚拟消费者不应是"完美理性人"。

---

### 3.4 维度4：社会摩擦感（Social Friction）

**核心问题**：消费者在社会互动中是否表现出犹豫、面子、矛盾等真实人类特征？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 犹豫表现 | 决策过程中有真实犹豫 | LLM辅助 |
| 面子考虑 | 社交场景中有面子/形象考量 | LLM辅助 |
| 矛盾行为 | 行为与自我认知有合理矛盾 | LLM辅助 |

**常见失败案例**：

- ❌ "我毫不犹豫地选择了最便宜的选项" —— 缺乏犹豫
- ✅ "其实我想买贵的，但室友在旁边，不好意思显得太挥霍，最后选了中档的" —— 有面子考量

---

### 3.5 维度5：时间延续性（Temporal Continuity）

**核心问题**：消费者是否有记忆、有惯性，过去经历影响现在行为？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 过去-现在因果 | present行为可由past经历解释 | 规则引擎 |
| 品牌经验 | 对品牌的态度有历史成因 | 规则引擎 |
| 消费习惯惯性 | 购买行为体现习惯而非纯理性 | LLM辅助 |

---

### 3.6 维度6：语言自然度（Linguistic Naturalness）

**核心问题**：语言风格是否符合身份，有无营销/学术术语？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 无营销术语 | 不含"品牌价值""用户痛点""心智"等 | 关键词过滤 |
| 无学术术语 | 不含"效用函数""偏好结构""弹性"等 | 关键词过滤 |
| 身份匹配 | 语言风格与年龄/职业/教育匹配 | LLM辅助 |
| 个性化 | 有口头禅、语气词、个性化表达 | LLM辅助 |

**禁止词汇库（部分）**：

| 类别 | 示例词汇 |
|------|---------|
| 营销术语 | 品牌价值、用户痛点、心智、触达、转化、留存、私域、公域、流量、变现 |
| 学术术语 | 效用函数、偏好结构、价格弹性、边际效用、需求曲线、消费者剩余、无差异曲线、预算约束 |
| 通用套话 | 总的来说、综上所述、从专业的角度、根据数据分析 |

---

### 3.7 维度7：知识边界感（Knowledge Boundaries）

**核心问题**：对超出认知范围的事物，是否表现真实无知或漠不关心？

**检查项**：

| 检查项 | 通过标准 | 检测方式 |
|--------|---------|---------|
| 专业领域边界 | 非专业人士不表现专业判断 | LLM辅助 |
| 技术细节边界 | 非技术人士对参数不妄加评论 | LLM辅助 |
| 信息缺口承认 | 主动承认不了解某些方面 | LLM辅助 |

**常见失败案例**：

- ❌ "这款洗碗机的变频电机转速达到3200rpm，能效比行业领先" —— 普通消费者不会这样说话
- ✅ "我不知道这个电机具体是什么技术，但听起来挺安静的" —— 承认无知

## 四、统计验证方法

### 4.1 样本覆盖度检验

确保生成的消费者样本在关键维度上覆盖充分：

| 维度 | 最小覆盖要求 | 检验方法 |
|------|------------|---------|
| 年龄 | 至少覆盖4个年龄段 | 频次统计 |
| 性别 | 男女比例 40%-60% | 卡方检验 |
| 收入 | 至少覆盖3个收入层级 | 频次统计 |
| 城市 | 至少覆盖3个城市层级 | 频次统计 |
| 职业 | 至少覆盖5个职业类别 | 频次统计 |
| 价值观 | 至少覆盖3种核心价值观 | 频次统计 |

### 4.2 分布检验（与真实基准对比）

验证合成样本的分布是否与真实人口分布一致：

- **连续变量**（年龄、收入等）：KS检验
- **分类变量**（性别、城市层级等）：卡方检验
- **多样性**：Shannon熵指数

**告警阈值**：

| p值范围 | 风险等级 | 处置 |
|---------|---------|------|
| p < 0.01 | CRITICAL | 暂停生成，全面审查 |
| 0.01 ≤ p < 0.05 | HIGH | 标记问题批次，要求修正 |
| 0.05 ≤ p < 0.1 | MEDIUM | 记录偏差，下批次调整采样权重 |
| p ≥ 0.1 | LOW | 通过 |

### 4.3 模型可恢复性测试

验证统计模型能否从虚拟消费者的选择数据中准确恢复预设的偏好参数：

| 指标 | 目标值 | 可接受范围 |
|------|--------|-----------|
| 效用参数恢复误差（MAPE） | ≤ 10% | ≤ 15% |
| 属性重要性排序一致性 | 100% | ≥ 80% |
| 价格系数方向正确率 | 100% | 100% |
| 市场份额预测R² | ≥ 0.85 | ≥ 0.70 |

## 五、偏见与伦理验证

### 5.1 伦理四原则

```
┌─────────────────────────────────────────────────────────────┐
│                    虚拟消费者伦理四原则                        │
├─────────────────────────────────────────────────────────────┤
│  1. 非歧视原则 —— 偏好结构不得与受保护属性系统性相关           │
│  2. 代表性原则 —— 合成样本应反映真实人口分布特征               │
│  3. 透明性原则 —— 生成逻辑、决策过程必须可追溯、可审计         │
│  4. 无害原则 —— 不得被用于放大刻板印象或歧视性营销             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 受保护属性清单

以下属性的任何标签**不得**成为偏好的系统性预测因子：

| 类别 | 受保护属性 | 风险示例 |
|------|-----------|---------|
| 人口统计 | 性别、民族、籍贯 | "女性只看颜值"、"某民族节俭" |
| 社会经济 | 职业、教育程度 | "蓝领工人只看价格"、"高学历者理性" |
| 家庭结构 | 婚姻状况、子女数量 | "宝妈容易被种草"、"单身青年冲动消费" |
| 地域 | 现居城市层级 | "小镇青年土味"、"一线城市精英" |
| 年龄 | 年龄段 | "老年人不会用手机"、"Z世代月光" |

### 5.3 六条公平性硬规则

| 规则 | 描述 | 审计方法 | 阈值 | 违规处置 |
|------|------|---------|------|---------|
| 规则1 | 偏好与性别解耦 | 卡方检验 | p ≥ 0.05 | 拒绝并重新生成 |
| 规则2 | 民族标签不关联消费行为 | 卡方检验 | p ≥ 0.05 | 标记复核 |
| 规则3 | 地域不决定品牌偏好 | KS检验 | p ≥ 0.05 | 标记复核 |
| 规则4 | 年龄不决定技术接受度 | KS检验 | p ≥ 0.05 | 拒绝并重新生成 |
| 规则5 | 职业不决定消费档次 | ANOVA | p ≥ 0.05 | 标记复核 |
| 规则6 | 多样性保障（熵指数、性别均衡等） | 批次级约束 | 见规范 | 重采样 |

### 5.4 刻板印象模式库（关键模式）

| 模式名称 | 关键词示例 | 风险等级 |
|---------|-----------|---------|
| 老年人技术恐惧 | "不会用手机"、"看不懂"、"操作复杂" | HIGH |
| 女性颜值导向 | "只看颜值"、"外观控"、"少女心" | CRITICAL |
| Z世代冲动消费 | "月光"、"冲动消费"、"花呗透支" | HIGH |
| 小镇青年土味 | "土味"、"low"、"没见过世面" | CRITICAL |
| 蓝领只看价格 | "只看价格"、"便宜就行"、"不讲究" | HIGH |

## 六、人工盲测（金标准）

### 6.1 盲测设计

**目的**：通过人类专家判断，验证虚拟消费者的真实感。

**设计原则**：
- **双盲**：评估者不知道哪些是虚拟的，哪些是真实的
- **对照**：混合真实消费者画像（来自过往调研）和虚拟消费者画像
- **多维度**：按七维度模型逐一评估
- **统计显著性**：样本量足够进行显著性检验

### 6.2 盲测流程

```
准备阶段
  │
  ├── 收集真实消费者画像（n=20）
  ├── 生成虚拟消费者画像（n=20，与真实画像混排）
  ├── 匿名化处理（移除可识别信息）
  └── 随机编号（001-040）
       ↓
评估阶段
  │
  ├── 招募评估者（5-8位消费者研究员）
  ├── 每位评估者独立评估全部40个画像
  ├── 按七维度逐一评分（0-2分）
  └── 附加问题："你认为这是真实消费者还是AI生成的？"
       ↓
分析阶段
  │
  ├── 统计真实 vs 虚拟在七维度上的得分差异
  ├── 计算虚拟画像被误判为真实的比例
  ├── 分析哪些维度是"暴露"AI身份的关键
  └── 输出盲测报告
```

### 6.3 盲测通过标准

| 指标 | 目标值 | 可接受值 |
|------|--------|---------|
| 虚拟画像平均真实性评分 | ≥ 真实画像的90% | ≥ 真实画像的80% |
| 虚拟画像被误判为真实的比例 | ≥ 40% | ≥ 30% |
| 七维度评分无显著差异（t检验） | p > 0.05 | p > 0.01 |
| 无单一维度明显暴露AI身份 | 所有维度 p > 0.05 | — |

## 七、质量门禁与处置流程

### 7.1 批次级门禁

```
虚拟消费者批次生成完成
   │
   ▼
┌─────────────────────────────────┐
│ 自动质量检查                     │
│ • Schema校验: 100%通过          │
│ • Logic校验: 核心规则通过        │
│ • 真实性评分: 平均分 ≥ 9        │
│ • 覆盖度检查: 全部维度通过       │
│ • 偏见检测: 无CRITICAL级别       │
└─────────────────────────────────┘
   │
   ├── 全部通过 → 入库，可用于模拟
   │
   └── 部分未通过 → 标记问题画像
        │
        ├── 可自动修正 → FIX Agent修正 → 重检
        │
        └── 无法自动修正 → 人工复核
             │
             ├── 人工通过 → 入库
             └── 人工不通过 → 废弃
```

### 7.2 系统级质量门禁（每阶段结束时）

| 阶段 | 质量门禁 | 通过标准 | 负责人 |
|------|---------|---------|--------|
| 阶段2结束 | 消费者真实性 | 盲测虚拟评分 ≥ 真实的80% | 小测 |
| 阶段3结束 | 模型可恢复性 | 参数恢复MAPE ≤ 15% | 小测+小数 |
| 阶段4结束 | 端到端准确性 | 与历史调研属性排序一致性 ≥ 80% | 小测+小验 |
| 阶段5持续 | 持续质量监控 | 每批次真实性评分 ≥ 9 | 小测 |

## 八、常见问题速查

### Q1：真实性评分9分可以入库吗？

可以。9-11分属于"合格"级别，可通过但建议优化。入库后应在后续批次中关注该画像所属群体的评分趋势。

### Q2：偏见检测发现CRITICAL级别怎么办？

立即暂停该批次，通知小伦（伦理审计负责人）和小示（行为工程师）。CRITICAL意味着刻板印象命中数超过批次量的5%，或发现系统性歧视模式。

### Q3：盲测中虚拟画像被误判率低于30%怎么办？

说明当前生成策略存在明显缺陷。需要：
1. 分析哪些维度暴露了AI身份
2. 由小示优化Prompt模板
3. 增加对应维度的Few-shot示例
4. 重新生成并再次盲测

### Q4：价格系数为正怎么办？

价格系数应为负值（价格越高，效用越低）。如果为正：
1. 检查数据编码是否正确
2. 检查是否有异常样本干扰
3. 重新运行模型
4. 如持续为正，需审查虚拟消费者的价格敏感度设定

### Q5：多样性熵指数低于0.6怎么办？

说明样本在某维度上过度集中。处置：
1. 识别哪个维度多样性不足
2. 调整该维度的采样权重
3. 对代表性不足的群体进行重采样
4. 重新计算熵指数
