## 一、批次级偏见审计必做检查（5项）

### 检查1：刻板印象模式扫描

| 子项 | 方法 | 阈值 | 通过标准 |
|------|------|------|---------|
| 关键词N-gram匹配 | 字符串匹配（不区分大小写） | 命中模式库关键词 | 无CRITICAL级别命中 |
| 语义相似度 | Embedding cosine similarity | < 0.7 | 无超过阈值的匹配 |
| 模式覆盖 | 24个模式（6类×4个） | 全部启用 | 100%模式库加载 |

**24个刻板印象模式分类**：
- 年龄相关（4）：技术恐惧、节俭保守、冲动消费、不务正业
- 性别相关（4）：颜值导向、情绪化决策、只看参数、不关心家庭
- 地域相关（4）：小镇青年土味、南北对立、一线城市精英主义、地域消费档次
- 职业相关（4）：蓝领只看价格、程序员格子衫、公务员保守、销售油嘴滑舌
- 婚育相关（4）：宝妈育儿焦虑/没有自我、已婚男性养家压力、剩女标签、丁克自私
- 教育/社会经济（4）：高学历精英主义、低学历没见识、收入歧视、炫耀消费

### 检查2：统计分布检验

| 维度 | 类型 | 检验方法 | 基准来源 | 阈值 | 通过标准 |
|------|------|---------|---------|------|---------|
| 年龄 | 连续 | KS检验 | 七普18-35岁一线城市切片 | D < 0.15 | p ≥ 0.05 |
| 收入 | 连续 | KS检验 | CFPS 2020一线城市家庭人均收入 | D < 0.15 | p ≥ 0.05 |
| 性别 | 分类 | 卡方检验 | 七普同年龄段 | — | p ≥ 0.05 |
| 城市层级 | 分类 | 卡方检验 | 七普 + 国家统计局城市划分 | — | p ≥ 0.05 |
| 职业 | 分类 | 卡方检验 | CFPS职业编码大类 | — | p ≥ 0.05 |
| 民族 | 分类 | 卡方检验 | 七普全国数据（汉族91%） | — | p ≥ 0.05 |

**小样本修正**：期望频数 < 5 的单元格占比 > 20% 时，改用 Fisher 精确检验。

|--------|------|-----------|---------|

| 地域 vs 品牌偏好 | Cramér's V | < 0.2 | 弱关联或无关联 |
| 年龄 vs 技术接受度 | Cramér's V | < 0.2 | 弱关联或无关联 |
| 职业 vs 消费档次 | Cramér's V | < 0.2 | 弱关联或无关联 |

### 检查5：多样性指数

| 维度 | 指标 | 最小覆盖 | 均衡要求 | 通过标准 |

## 二、审计频率与触发条件

| 审计类型 | 频率 | 样本 | 自动化 | 负责人 |
|---------|------|------|--------|--------|
| 批次级偏见扫描 | 每批次 | 全量 | 自动 | 自动化管线 |
| 日报汇总 | 每日 | 当日全部 | 自动+人工复核 | 小伦 |

**触发审计的事件**：
- [ ] 新Prompt模板上线前
- [ ] 模型版本切换后
- [ ] 业务方反馈"结果不可理喻"时
- [ ] 成本熔断触发后（检查是否为异常行为）
- [ ] 任何团队成员提出伦理疑虑时

## 三、偏见超标处置流程

```
偏见审计完成
   │
   ▼
评估风险等级
   │
   ├── LOW（p ≥ 0.1）
   │   → 记录偏差，下批次调整采样权重
   │   → 批次通过
   │
   ├── MEDIUM（0.05 ≤ p < 0.1）
   │   → 标记问题画像
   │   → 通知小示优化Prompt
   │   → 批次通过（带警告）
   │
   ├── HIGH（0.01 ≤ p < 0.05）
   │   → 标记问题批次
   │   → 问题画像需重新生成或修正
   │   → 小伦审核修正后结果
   │   → 通过后方可入库
   │
   └── CRITICAL（p < 0.01 或 刻板印象 > 5%）
       → 小伦行使一票否决权
       → 暂停消费者生成流程
       → 全面审查Prompt模板
       → 修复后重新审计
       → 通过后方可恢复
```

## 四、一票否决权行使条件

小伦在以下情况下有权暂停消费者生成流程：

1. **批次级CRITICAL风险**：刻板印象命中数 > 批次量的5%
2. **统计检验系统性失败**：同一维度连续3个批次未通过KS/卡方检验
3. **新偏见模式发现**：发现未被模式库覆盖的系统性偏见
4. **合规要求变更**：法律法规或公司政策更新，现有流程不满足新要求
