## 二、通用系统设计原则

### 2.1 产品适配能力

系统必须支持以下产品类型的联合分析：

| 产品类别 | 示例 | 属性特点 | 水平设计要点 |
|---------|------|---------|-------------|
| **大家电** | 洗碗机、冰箱、洗衣机 | 功能属性多、价格区间宽、安装相关 | 强调容量、能耗、噪音、安装方式 |
| **小家电** | 智能手表、空气净化器、扫地机 | 技术迭代快、外观重要、场景细分 | 强调续航、智能化、外观设计 |
| **数码3C** | 手机、平板、耳机 | 参数透明、品牌效应强、更新周期短 | 强调性能、存储、品牌、价格 |
| **软件/服务** | SaaS产品、会员服务、在线教育 | 无形性、体验难量化、订阅模式 | 强调功能模块、价格模型、服务等级 |
| **虚拟商品** | 游戏道具、数字藏品、虚拟会员 | 情感价值高、稀缺性、社交属性 | 强调稀有度、外观、社交功能 |

### 2.2 人群适配能力

系统必须支持灵活的人群设定：

| 人群维度 | 可配置项 | 示例 |
|---------|---------|------|
| 人口统计 | 年龄、性别、收入、城市、教育 | 18-35岁 / 一线城市 / 本科及以上 |
| 家庭结构 | 婚姻、子女、同住情况 | 已婚有娃 / 独居 / 与父母同住 |
| 生活方式 | 价值观、消费观、时间分配 | 精致生活 / 实用主义 / 科技先锋 |
| 产品经验 | 使用历史、品牌认知、痛点 | 首次购买 / 升级换代 / 品牌忠诚者 |

## 三、首个验证场景：洗碗机定价研究

### 3.1 业务背景

公司计划推出一款面向年轻家庭的洗碗机产品，需要回答：
1. 目标消费者最看重洗碗机的哪些属性？
2. 不同价格区间的市场接受度如何？
3. 最优配置组合是什么？
4. 与竞品相比的差异化定位建议？

### 3.2 研究设计

#### 产品属性与水平（示例）

| 属性 | 水平1 | 水平2 | 水平3 | 水平4 |
|------|-------|-------|-------|-------|
| **价格** | ¥2,999 | ¥3,999 | ¥4,999 | ¥5,999 |
| **容量** | 6套（小家庭） | 10套（标准） | 13套（大家庭） | — |
| **安装方式** | 台式（免安装） | 嵌入式 | 水槽式 | — |
| **核心功能** | 基础洗+烘干 | 智能洗+烘干+消毒 | 全能洗+AI智能+除菌 | — |
| **品牌** | 美的 | 西门子 | 方太 | 小米 |
| **能耗等级** | 二级 | 一级 | 超一级 | — |

#### 目标消费者画像

| 细分群体 | 占比 | 核心特征 |
|---------|------|---------|
| **精致白领（25-30岁）** | 40% | 独居/情侣、注重品质、时间敏感、价格不敏感 |
| **新手宝妈（28-35岁）** | 35% | 有0-6岁娃、家务繁重、健康焦虑、口碑驱动 |
| **Z世代租客（22-28岁）** | 25% | 租房、空间有限、预算敏感、颜值党 |

## 四、验收标准

### 4.1 必须满足（Must Have）

| 验收项 | 验收标准 | 验证方法 | 通过阈值 |
|--------|---------|---------|---------|
| **属性重要性排序合理性** | TOP3属性符合业务直觉 | 专家评审 | ≥ 2/3专家认可 |
| **价格效用方向** | 价格系数为负 | 自动检查 | 100%正确 |
| **模型收敛性** | HB模型R-hat < 1.1 | 自动诊断 | 100%通过 |
| **消费者真实性** | 人工盲测评分 | 盲测实验 | 虚拟评分 ≥ 真实评分的75% |
| **系统可用性** | 业务用户独立完成全流程 | UAT测试 | 成功率 ≥ 80% |
| **偏见检测** | 无CRITICAL级别偏见 | 自动化审计 | 100%通过 |
| **成本可控** | 单次研究在预算内 | 成本追踪 | ≤ ¥500 |

### 4.2 期望满足（Should Have）

| 验收项 | 验收标准 | 验证方法 | 通过阈值 |
|--------|---------|---------|---------|
| **细分群体差异显著** | 不同群体的偏好差异可被解释 | 统计检验+专家评审 | p < 0.05 |
| **WTP合理性** | 支付意愿在业务合理范围内 | 专家评审 | 偏差 ≤ 30% |
| **市场份额预测方向正确** | 低价配置份额 > 高价配置 | 逻辑检查 | 符合直觉 |
| **报告可直接使用** | 非技术人员可直接用于汇报 | 用户测试 | 理解度 ≥ 80% |
| **发现反直觉洞察** | 至少1个超出专家预期的发现 | 专家评审 | 有1个即可 |

### 4.3 加分项（Nice to Have）

| 验收项 | 验收标准 | 验证方法 |
|--------|---------|---------|
| **What-if模拟** | 可实时调整配置看份额变化 | 功能测试 |
| **跨产品复用** | 同一套系统可快速切换到其他产品 | 功能测试 |
| **自我优化** | 根据使用数据持续改进消费者真实性 | 盲测对比（迭代前后） |

### 4.4 可接受偏差

| 指标 | 目标值 | 可接受上限 | 说明 |
|------|--------|-----------|------|
| 属性重要性数值偏差 | ≤ 10% | ≤ 15% | 与专家预期对比 |
| WTP估计偏差 | ≤ 25% | ≤ 30% | 与业务认知对比 |
| 市场份额预测偏差 | ≤ 10个百分点 | ≤ 15个百分点 | 方向必须正确 |
| 消费者真实性评分 | ≥ 真实评分的80% | ≥ 75% | 盲测结果 |

## 五、关键业务问题（必须回答）

### 5.1 洗碗机场景问题

1. **在3000-6000元价位段，消费者最愿意为哪些属性付费？**
   - 预期：容量和核心功能可能是关键

2. **台式、嵌入式、水槽式三种安装方式，各自的溢价空间有多大？**
   - 预期：嵌入式溢价最高，台式门槛最低

3. **品牌vs功能，哪个对购买决策的影响更大？**
   - 预期：不同细分群体可能有显著差异

4. **超一级能耗是否足以支撑500元以上的溢价？**
   - 预期：在部分群体中可能成立

5. **三个细分群体的偏好差异有多大？是否需要推出差异化产品？**
   - 预期：差异显著，建议差异化配置

### 5.2 通用能力验证问题

1. **系统能否在2小时内完成从需求输入到报告输出的全流程？**
2. **更换产品（如从洗碗机换成智能手表）后，系统能否在30分钟内重新配置？**
3. **系统生成的消费者画像是否支持导出和人工审核？**
4. **分析结果是否支持按任意画像维度进行交叉分析？**

## 七、使用限制与免责声明

### 7.1 适用范围

✅ **适用场景**：
- 产品概念快速筛选
- 定价策略探索
- 属性重要性排序
- 小众人群需求挖掘
- 假设快速验证

❌ **不适用场景**：
- 最终上市定价的唯一依据
- 精确销量预测
- 法律法规相关的合规决策
- 涉及健康安全的关键决策

### 7.2 建议的使用方式

```
AI_CBC研究结果
      │
      ├──→ 与真实消费者研究交叉验证（推荐）
      │
      ├──→ 与行业数据/竞品分析对比
      │
      └──→ 作为内部讨论和假设筛选的输入
                │
                ▼
         结合专家判断 → 形成最终决策
```

### 7.3 自我优化声明

系统具备根据每次使用数据进行自我优化的能力：
- **偏好校准**：根据用户反馈调整消费者偏好分布
- **Prompt优化**：积累高质量画像样本，优化生成模板
- **模型迭代**：持续运行参数恢复测试，改进建模策略

但自我优化不改变以下原则：
- 公平性硬规则始终强制执行
- 偏见审计每批次必做
- 人工盲测定期执行（每迭代周期至少1次）
