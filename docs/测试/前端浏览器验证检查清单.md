# AI_CBC 前端浏览器手动验证检查清单

> 由前端工程师（小前）提供，供 QA（小测）在浏览器端执行 GUI 交互验证时参考。
> 环境：前端 http://localhost:3001，后端 http://localhost:8000

## 一、页面可访问性（12 个页面）

| # | 页面 | 访问路径 | 验证要点 |
|---|------|---------|---------|
| 1 | 总览（Dashboard） | `/` | 左侧菜单加载、研究列表显示、统计卡片（4个） |
| 2 | 创建研究 | `/studies/new` | 表单字段（study_id, product_category, research_goal, target_segments） |
| 3 | 画像管理 | `/personas` | 画像列表加载、分页、搜索 |
| 4 | 画像详情 | `/personas/:personaId` | 四层画像数据展示 |
| 5 | 问卷配置 | `/questionnaires` | CBC 实验设计总览、问卷展开详情 |
| 6 | 问卷预览 | `/studies/:studyId/questionnaire` | 选择集设计矩阵展示 |
| 7 | 作答模拟 | `/studies/:studyId/responses` | mode 选择器、消费者多选、模拟按钮 |
| 8 | 对话实验室 | `/interview` | 多轮对话、情绪标签、矛盾检测 |
| 9 | 属性重要性看板 | `/importance` | 图表渲染、属性排名、置信区间 |
| 10 | 细分群体比较 | `/segment-comparison` | 群体选择、对比分析结果表格 |
| 11 | 市场份额模拟器 | `/market-simulator` | 产品场景配置、份额预测图表 |
| 12 | 分析任务状态 | `/analysis-status` | model_type 选择器、任务队列、收敛诊断 |
| 13 | 系统设置 | `/settings` | LLM 配置、成本预算、保存/重置 |

## 二、任务 #3 新增入口验证

### 2.1 左侧菜单「作答模拟」

- [ ] 点击后页面标题变为「AI_CBC」或「总览」（因为 `/responses` 映射到 Dashboard）
- [ ] 不报错、不白屏

### 2.2 Dashboard 操作列按钮
- [ ] 研究列表加载成功（至少显示 demo-study-001、demo-study-002）
- [ ] 对 status ≠ READY/COMPLETED 的研究：显示「生成问卷」按钮
- [ ] 对 status = READY/COMPLETED 的研究：显示「查看问卷」按钮
- [ ] 所有研究行：显示「模拟作答」按钮
- [ ] 所有研究行：显示「查看分析」按钮
- [ ] 点击「生成问卷」→ 调用 API → 刷新列表 → 状态变为 READY
- [ ] 点击「查看问卷」→ 跳转到 `/studies/{studyId}/questionnaire`
- [ ] 点击「模拟作答」→ 跳转到 `/studies/{studyId}/responses`
- [ ] 点击「查看分析」→ 跳转到 `/importance?study={studyId}`

### 2.3 QuestionnaireConfig「生成问卷」按钮
- [ ] 进入「问卷配置」页面
- [ ] 对未生成问卷的研究行：显示「生成问卷」按钮
- [ ] 点击后调用 API → 刷新该行问卷状态 → 显示「已生成」标签

## 三、任务 #4 参数选择器验证

### 3.1 ResponseSimulator mode 选择器
- [ ] 进入 `/studies/demo-study-001/responses`
- [ ] 页面显示「选择要模拟的虚拟消费者」多选框（应有 5 个 mock 画像可选）
- [ ] 页面显示「选择模拟模式」下拉框
- [ ] 下拉框选项：随机模式 (stochastic)、确定性模式 (deterministic)、LLM 模式 (llm)
- [ ] 默认选中「随机模式」
- [ ] 选择消费者 + 切换 mode → 点击「开始模拟作答」
- [ ] 模拟过程中显示 Spin + 「正在模拟作答，请稍候...」文字（无假进度条）
- [ ] 模拟完成后显示成功 Alert
- [ ] 验证不同 mode 的 API 请求体正确：`{persona_ids: [...], mode: "deterministic"}`

### 3.2 AnalysisStatus model_type 选择器
- [ ] 进入「分析任务状态」页面
- [ ] 页面显示「选择研究项目运行分析」下拉框
- [ ] 页面显示「选择模型类型」下拉框
- [ ] 下拉框选项：HB (Hierarchical Bayes)、MNL (Multinomial Logit)、Latent Class
- [ ] 默认选中「HB」
- [ ] 选择研究 + 切换 model_type → 点击「启动分析」
- [ ] 按钮文字为「启动分析」（非「启动 HB 分析」）
- [ ] 任务加入队列，显示状态、进度、收敛诊断
- [ ] 验证不同 model_type 的 API 请求体正确：`{model_type: "mnl", ...}`

### 3.3 SegmentComparison 类型安全
- [ ] 进入「细分群体比较」页面
- [ ] 选择研究 → 输入分析 ID → 选择群体 A/B → 点击「运行对比分析」
- [ ] 结果表格正确渲染：属性、方法、t统计量、p值、显著性、效应量、均值
- [ ] 总体检验卡片正确渲染：方法、统计量、p值、显著性
- [ ] 无 TypeScript 类型错误（已在 build 时验证）

## 四、完整主链路验证（建议用 demo-study-001）

### 4.1 创建研究（可选，如 create_study 已修复）
- [ ] 点击「创建研究」
- [ ] 填写中文表单：study_id（如"洗碗机-测试-001"）、product_category（"洗碗机"）、research_goal（"测试中文"）、target_segments（["一线城市年轻家庭"]）
- [ ] 提交成功 → Dashboard 列表出现新研究

### 4.2 生成问卷
- [ ] 在 Dashboard 或 QuestionnaireConfig 点击「生成问卷」
- [ ] 等待 API 返回 → 显示 D-efficiency 值
- [ ] 点击「查看问卷」→ 问卷预览页面加载正确

### 4.3 生成/选择画像
- [ ] 进入「画像管理」→ 点击「生成画像」
- [ ] 或直接在 ResponseSimulator 页面选择已有画像（后端已预置 5 个）

### 4.4 模拟作答
- [ ] 进入「作答模拟」→ 选择 1-3 个画像
- [ ] 切换 mode（stochastic / deterministic / llm）
- [ ] 点击「开始模拟作答」→ 等待完成 → 显示成功提示

### 4.5 运行分析
- [ ] 进入「分析任务状态」→ 选择研究 → 选择 model_type（hb/mnl/latent_class）
- [ ] 点击「启动分析」→ 任务进入队列 → 状态变为 RUNNING → COMPLETED
- [ ] 展开收敛诊断：R-hat Max、ESS、发散次数

### 4.6 查看结果
- [ ] 进入「属性重要性看板」→ 查看属性排名图表
- [ ] 切换 WTP 标签页 → 查看支付意愿数据（后端已补充 mock 数据）
- [ ] 进入「市场份额模拟器」→ 配置产品场景 → 查看份额预测
- [ ] 进入「细分群体比较」→ 选择两个群体 → 查看对比结果

## 五、异常场景验证

- [ ] 空状态：无研究时 Dashboard 显示空提示
- [ ] 空状态：无画像时 ResponseSimulator 消费者选择器为空
- [ ] 网络断开：前端显示错误提示（非白屏）
- [ ] 后端 500：前端显示错误 Alert，可重试
- [ ] 未选择研究直接点击「启动分析」：前端提示「请选择研究项目」
- [ ] 未选择画像直接点击「开始模拟作答」：前端提示「请选择至少一个虚拟消费者」

## 六、已知限制（非缺陷）

1. **后端 mock 数据**：WTP 数据已补充，但可能只有 1-2 个属性有值
2. **画像生成**：后端预置 5 个 mock 画像，如需更多需手动生成
3. **分析任务**：mock 模式下分析立即完成（无真实计算），收敛诊断为固定值
