## 一、威胁模型（STRIDE分析）

### 1.3 关键威胁场景

| # | 威胁场景 | 攻击向量 | 影响 | 优先级 |
|---|---------|---------|------|--------|
| T1 | **提示注入攻击** | 通过画像描述、情境描述注入恶意指令，诱导Agent输出有害内容或泄露系统信息 | Agent行为失控、系统信息泄露 | P0 |
| T2 | **对抗性选择集污染** | 构造特殊的选择集组合，诱导虚拟消费者产生系统性偏差 | 分析结果失真、决策误导 | P0 |

| T4 | **API密钥泄露** | 密钥硬编码、日志泄露、配置不当导致API密钥暴露 | 直接经济损失、API滥用 | P0 |

| T6 | **供应链污染** | 第三方模型、库、插件含有恶意代码 | 数据窃取、后门植入 | P1 |
| T7 | **DoS攻击** | 大量请求耗尽API配额或系统资源 | 服务中断、成本激增 | P1 |
| T8 | **日志篡改** | 攻击者修改审计日志掩盖痕迹 | 无法追溯、合规失败 | P2 |

### 2.2 输入清洗与过滤

#### 2.2.1 输入校验规则

```python
class InputSanitizer:
    """输入清洗器 - 所有外部输入必须经过此层"""
    
    # 禁止字符集（可能导致注入）
    FORBIDDEN_CHARS = [
        '\x00',  # 空字符
        '\x1a',  # 替换字符
    ]
    
    # 危险关键词模式（提示注入检测）
    INJECTION_PATTERNS = [
        r"ignore\s+previous\s+instructions",
        r"disregard\s+all\s+prior",
        r"you\s+are\s+now\s+a",
        r"new\s+persona\s*:",
        r"system\s*:\s*you\s+are",
        r"\{\{\s*.*system.*\}\}",
        r"<\s*system\s*>",
        r"DAN\s*\(Do\s*Anything\s*Now\)",
        r"jailbreak",
        r"ignore\s+the\s+above",
    ]
    
    # 最大输入长度
    MAX_INPUT_LENGTH = {
        "persona_description": 10000,  # 字符
        "scenario_context": 5000,
        "question_text": 2000,
        "user_query": 1000,
    }
    
    @classmethod
    def sanitize(cls, text: str, context: str) -> tuple[str, list[str]]:
        """
        清洗输入文本
        
        Returns:
            (cleaned_text, warnings)
        """
        warnings = []
        
        # 1. 长度检查
        max_len = cls.MAX_INPUT_LENGTH.get(context, 10000)
        if len(text) > max_len:
            warnings.append(f"输入超过最大长度({max_len})，已截断")
            text = text[:max_len]
        
        # 2. 禁止字符过滤
        for char in cls.FORBIDDEN_CHARS:
            if char in text:
                warnings.append(f"发现禁止字符，已移除")
                text = text.replace(char, "")
        
        # 3. 注入模式检测
        for pattern in cls.INJECTION_PATTERNS:
            if re.search(pattern, text, re.IGNORECASE):
                warnings.append(f"检测到潜在提示注入模式: '{pattern}'")
                # 不直接拒绝，记录并标记，由后续层处理
        
        # 4. Unicode归一化（防止同形字符攻击）
        text = unicodedata.normalize('NFKC', text)
        
        return text, warnings
    
    @classmethod
    def validate_persona_input(cls, persona_data: dict) -> tuple[bool, list[str]]:
        """验证消费者画像输入的合法性"""
        warnings = []
        
        # 检查是否包含系统指令
        text_fields = [
            persona_data.get("layer1", {}).get("narrative", ""),
            persona_data.get("layer3", {}).get("psychology", ""),
            persona_data.get("layer4", {}).get("scene_reactions", ""),
        ]
        
        for field in text_fields:
            if field:
                _, field_warnings = cls.sanitize(field, "persona_description")
                warnings.extend(field_warnings)
        
        # 检查标签值是否在允许范围内
        # (由Schema Validator处理，此处仅做安全层面的检查)
        
        is_safe = not any("注入" in w or "禁止" in w for w in warnings)
        return is_safe, warnings
```

### 2.3 Agent沙箱与权限控制

#### 2.3.1 工具调用沙箱

```python
class ToolSandbox:
    """
    工具调用沙箱 - 限制Agent可执行的操作
    """
    
    # 允许的工具白名单
    ALLOWED_TOOLS = {
        "data_loader": {
            "allowed_operations": ["load_csv", "load_json", "load_parquet"],
            "forbidden_operations": ["load_executable", "download_url", "write_file"],
            "max_file_size_mb": 100,
        },
        "model_fitter": {
            "allowed_operations": ["fit_mnl", "fit_hb", "fit_latent_class"],
            "forbidden_operations": ["exec_code", "system_call", "network_request"],
            "max_runtime_seconds": 300,
        },
        "result_processor": {
            "allowed_operations": ["compute_importance", "compute_wtp", "simulate_market"],
            "forbidden_operations": ["write_file", "delete_file", "send_email"],
        },
        "visualizer": {
            "allowed_operations": ["plot_importance", "plot_utilities", "plot_market_sim"],
            "forbidden_operations": ["display_external_image", "open_url"],
        },
    }
    
    @classmethod
    def execute(cls, tool_name: str, operation: str, params: dict) -> dict:
        """在沙箱中执行工具调用"""
        
        # 1. 检查工具是否在白名单中
        if tool_name not in cls.ALLOWED_TOOLS:
            raise SecurityError(f"未授权的工具: {tool_name}")
        
        tool_config = cls.ALLOWED_TOOLS[tool_name]
        
        # 2. 检查操作是否被允许
        if operation in tool_config["forbidden_operations"]:
            raise SecurityError(
                f"操作 '{operation}' 在工具 '{tool_name}' 中被禁止"
            )
        
        if operation not in tool_config["allowed_operations"]:
            raise SecurityError(
                f"未知操作: '{operation}'，允许的操作: {tool_config['allowed_operations']}"
            )
        
        # 3. 参数校验
        sanitized_params = cls._sanitize_params(params)
        
        # 4. 执行（带超时）
        try:
            with timeout(tool_config.get("max_runtime_seconds", 60)):
                result = cls._do_execute(tool_name, operation, sanitized_params)
            return result
        except TimeoutError:
            raise SecurityError(f"工具调用超时: {tool_name}.{operation}")
    
    @classmethod
    def _sanitize_params(cls, params: dict) -> dict:
        """清理参数中的危险内容"""
        # 递归检查参数值
        # 禁止包含文件路径、URL、系统命令等
        sanitized = {}
        for key, value in params.items():
            if isinstance(value, str):
                # 检查路径遍历
                if ".." in value or value.startswith("/"):
                    raise SecurityError(f"参数包含潜在路径遍历: {key}={value}")
                sanitized[key] = value
            elif isinstance(value, (int, float, bool)):
                sanitized[key] = value
            elif isinstance(value, (list, dict)):
                sanitized[key] = cls._sanitize_params(value)
            else:
                raise SecurityError(f"不支持的参数类型: {type(value)}")
        return sanitized
```

### 2.4 输出内容审查

```python
class OutputContentFilter:
    """输出内容过滤器"""
    
    # 敏感内容类别
    SENSITIVE_CATEGORIES = {
        "hate_speech": ["种族歧视", "性别歧视", "地域歧视", "仇恨言论"],
        "violence": ["暴力", "恐怖主义", "自残", "伤害他人"],
        "illegal": ["毒品", "赌博", "诈骗", "洗钱"],
        "privacy": ["身份证号", "手机号", "银行卡", "住址"],
        "misinformation": ["虚假医疗", "伪科学", "阴谋论"],
    }
    
    # 系统信息泄露检测
    SYSTEM_INFO_PATTERNS = [
        r"system\s*prompt",
        r"instructions?\s*for\s*you",
        r"your\s*training\s*data",
        r"API\s*key",
        r"model\s*name\s*:",
    ]
    
    @classmethod
    def filter(cls, content: str) -> tuple[str, list[dict]]:
        """
        过滤输出内容
        
        Returns:
            (filtered_content, violations)
        """
        violations = []
        
        # 1. 敏感内容检测
        for category, keywords in cls.SENSITIVE_CATEGORIES.items():
            for keyword in keywords:
                if keyword in content:
                    violations.append({
                        "type": "sensitive_content",
                        "category": category,
                        "keyword": keyword,
                        "severity": "HIGH",
                        "action": "remove_or_replace"
                    })
        
        # 2. 系统信息泄露检测
        for pattern in cls.SYSTEM_INFO_PATTERNS:
            if re.search(pattern, content, re.IGNORECASE):
                violations.append({
                    "type": "information_disclosure",
                    "pattern": pattern,
                    "severity": "CRITICAL",
                    "action": "block_and_alert"
                })
        
        # 3. 处理违规内容
        filtered = content
        for v in violations:
            if v["action"] == "block_and_alert":
                # 阻断并告警
                raise SecurityError(f"检测到严重安全违规: {v['type']}")
            elif v["action"] == "remove_or_replace":
                # 替换敏感词
                filtered = filtered.replace(v.get("keyword", ""), "[已过滤]")
        
        return filtered, violations
```

## 三、红队测试规范

### 3.1 红队测试目标

验证系统在以下攻击场景下的防御能力：

1. **提示注入**：能否通过输入操控Agent行为
2. **数据污染**：能否通过异常输入影响分析结果
3. **越狱攻击**：能否绕过安全限制获取违规输出
4. **权限提升**：能否执行未授权操作
5. **信息提取**：能否获取系统内部信息

### 3.3 红队测试执行流程

```
Phase 1: 测试计划
───────────────
• 确定测试范围（系统组件、API端点、Agent类型）
• 选择测试用例（从用例库中选取相关用例）
• 设定通过标准（防御成功率 ≥ 95%）
       ↓
Phase 2: 测试执行
───────────────
• 在隔离环境中执行攻击用例
• 记录每次攻击的输入、系统响应、防御触发情况
• 收集防御绕过案例（如有）
       ↓
Phase 3: 结果分析
───────────────
• 计算防御成功率
• 分类漏洞类型（输入层/处理层/输出层/监控层）
• 评估漏洞严重程度
       ↓
Phase 4: 修复验证
───────────────
• 开发修复方案
• 回归测试验证修复效果
• 更新防御规则库
       ↓
Phase 5: 报告输出
───────────────
• 红队测试报告
• 漏洞修复记录
• 防御能力基线更新
```

### 4.2 异常检测规则

```yaml
anomaly_detection_rules:
  rule_001_api_spike:
    name: "API调用量激增"
    condition: "requests_per_minute > baseline * 5"
    duration: "5 minutes"
    severity: "HIGH"
    action: "rate_limit + alert"
    
  rule_002_injection_spike:
    name: "注入攻击频率异常"
    condition: "injection_attempts_per_hour > 10"
    severity: "CRITICAL"
    action: "block_ip + alert + incident_response"
    
  rule_003_cost_spike:
    name: "API费用激增"
    condition: "hourly_cost > budget * 0.2"
    severity: "HIGH"
    action: "cost_fuse + alert"
    
  rule_004_failed_auth:
    name: "认证失败激增"
    condition: "failed_auth_per_ip > 5 in 1 hour"
    severity: "MEDIUM"
    action: "temporary_block + alert"
    
  rule_005_unusual_tool_call:
    name: "异常工具调用"
    condition: "tool_call not in whitelist OR params contain suspicious patterns"
    severity: "CRITICAL"
    action: "block + alert + audit"
    
  rule_006_data_exfiltration:
    name: "数据外泄检测"
    condition: "response_size > threshold AND contains sensitive patterns"
    severity: "CRITICAL"
    action: "block + alert + forensic"
```

| 级别 | 定义 | 示例 | 响应团队 |
|------|------|------|---------|

| P1 | 核心功能受影响 | 大规模注入攻击成功、模型被篡改 | 小安 + 小P + 小维 |
| P2 | 局部功能受影响 | 单个端点被绕过、部分数据异常 | 小安 + 相关开发人员 |
| P3 | 轻微异常 | 个别告警、统计偏差 | 小安 |
