## 一、项目现状概览

### 1.1 系统架构

AI_CBC 是一个基于 LLM 的虚拟消费者联合分析（CBC）研究平台，由三大子系统构成：

| 子系统 | 目录 | 状态 |
|--------|------|------|
| 消费者模拟系统 | `consumer-simulation/` | 文档完整，代码实现中 |
| CBC 问卷系统 | `cbc-questionnaire-system/` | 文档完整，核心引擎实现中 |
| 数据分析系统 | `cbc-analysis-system/` | 文档完整，HB/MNL 引擎实现中 |

### 1.2 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | FastAPI + Uvicorn | 0.111.0 / 0.30.0 |
| 前端框架 | React + TypeScript + Vite | 18.3.1 / 5.5.4 |
| 数据存储 | MongoDB (motor/beanie) + Redis | 6 / 7-alpine |
| 任务队列 | Celery + Redis | 5.4.0 |
| 统计建模 | PyMC + ArviZ + SciPy | 5.15.0 / 0.18.0 |
| LLM 客户端 | Anthropic + OpenAI | 0.28.0 / 1.35.0 |
| 监控 | Prometheus + Grafana | 2.51 / 10.4 |
| 容器化 | Docker + Docker Compose | — |
| CI/CD | GitHub Actions | — |

### 1.3 项目阶段定位

根据《系统总览白皮书》，当前项目处于 **Phase 4 全链路串联与业务验收** 阶段，即将进入 Phase 5 持续监控。本次验收的核心任务是验证 MVP 是否满足上线标准。

### 2.2 通过/失败标准（基于《系统成功标准书》）

#### Must Have（必须满足）

| 验收项 | 标准 | 一票否决 |
|--------|------|---------|
| 属性重要性排序 TOP3 与业务直觉一致 | 专家评审 | 否 |
| 价格效用方向为负 | 100% 正确 | **是** |
| HB 模型收敛 | R-hat < 1.1 | 否 |
| 消费者真实性盲测 | ≥ 真实评分的 75% | 否 |
| 偏见审计 | 无 CRITICAL 级别 | **是** |
| 6 条公平性硬规则嵌入 Prompt | 100% 嵌入 | **是** |
| 系统可用性 | 独立操作成功率 ≥ 80% | 否 |
| 单次研究成本 | ≤ 500 元 | 否 |
| 红队防御成功率 | ≥ 95% | 否 |

#### Should Have（期望满足）

| 验收项 | 标准 |
|--------|------|
| 细分群体差异显著 | p < 0.05 |
| WTP 估计偏差 | ≤ 30% |
| 市场份额预测 | R² ≥ 0.5 且方向正确 |
| 报告可直接使用 | 理解度 ≥ 80% |
| 反直觉洞察 | ≥ 1 个 |
| 业务用户满意度 | ≥ 4.0/5 |
