## 六、数据集管理规范

### 6.1 存储与访问

| 项目 | 规范 |
|------|------|
| 目录结构 | `tests/data/{normal,abnormal,extreme,synthetic}/` |
| 版本控制 | 所有数据集纳入Git LFS或单独数据仓库 |
| 访问权限 | 正常/极限/合成数据：团队内开放；异常数据中的偏见数据：仅测试团队访问 |
| 更新频率 | 产品属性变更时同步更新；合成数据固定不变（seed固定） |

### 6.2 数据集命名规则

```
{类别前缀}-{序号}-{简要描述}.json

类别前缀：
  N  = Normal（正常数据）
  A  = Abnormal（异常数据）
  X  = Extreme（极限数据）
  S  = Synthetic（合成数据）

示例：
  N-001-dishwasher-attributes.json
  A-005-bias-critical-gender.json
```

### 6.3 数据脱敏要求

| 数据类型 | 脱敏要求 |
|---------|---------|
| 正常画像模板 | 使用虚构/合成描述，不含真实姓名、电话、地址 |
| 异常攻击载荷 | 确保攻击payload仅在测试环境执行，不对外发送 |
| 偏见测试数据 | 标注「TEST DATA - BIAS TRIGGER」，隔离存储 |
| 合成选择数据 | 纯数学生成，无真实个人信息 |
