### 2.2 功能详情

#### ① 消费者画像生成（Persona Generation）

| 子功能 | 描述 | 状态 |
|--------|------|------|
| 画像模板库 | 支持按人口统计、家庭结构、生活方式、产品经验四维度配置 | ✅ 可用 |
| 批量生成 | 单次最多生成 500 个虚拟消费者画像 | ✅ 可用 |
| LLM 驱动 | 基于 Claude-3.5-Sonnet / GPT-4o 生成自然语言画像描述 | ✅ 可用 |
| 画像审校 | 自动审校（REV Agent）检查画像一致性与完整性 | ✅ 可用 |
| 画像导出 | 支持 JSON / CSV 格式导出 | ✅ 可用 |

#### ② CBC 问卷设计（Questionnaire Design）

| 子功能 | 描述 | 状态 |
|--------|------|------|
| 属性/水平配置 | 可视化配置产品属性及各水平值 | ✅ 可用 |
| 实验设计算法 | 自动生成 D-efficient 选择集（D-efficiency ≥ 0.85） | ✅ 可用 |
| 问卷预览 | 实时预览 CBC 选择集，支持交互式验证 | ✅ 可用 |
| 属性数量 | 支持 3-10 个属性，每个属性 2-6 个水平 | ✅ 可用 |
| 任务数控制 | 支持 6-20 个选择任务，自动计算平衡性 | ✅ 可用 |

#### ③ 虚拟消费者模拟作答（Simulation）

| 子功能 | 描述 | 状态 |
|--------|------|------|
| 逐题模拟 | 每个消费者独立逐题完成选择任务 | ✅ 可用 |
| 选择理由 | 生成自然语言形式的选择理由（便于业务解读） | ✅ 可用 |
| 无选项逻辑 | 支持 "None of the above" 选项及其合理性 | ✅ 可用 |
| 批量模拟 | 支持 100+ 消费者并行模拟（Celery 异步队列） | ✅ 可用 |
| 响应模拟器 | 独立工具模拟单个消费者面对特定选择集的决策 | ✅ 可用 |

#### ④ HB 联合分析（Hierarchical Bayes Analysis）

| 子功能 | 描述 | 状态 |
|--------|------|------|
| HB 建模 | 基于 PyMC 实现分层贝叶斯模型 | ✅ 可用 |
| 模型收敛诊断 | 自动检查 R-hat（< 1.1）与 ESS（≥ 100） | ✅ 可用 |
| 属性重要性 | 自动计算并输出各属性相对重要性 | ✅ 可用 |
| 效用值分布 | 输出群体均值 + 个体-level 效用分布 | ✅ 可用 |
| WTP 估算 | 基于效用差计算支付意愿区间 | ✅ 可用 |
| 细分对比 | 按画像维度进行交叉分析（性别、年龄、城市等） | ✅ 可用 |

#### ⑤ 市场模拟（Market Simulator）

| 子功能 | 描述 | 状态 |
|--------|------|------|
| 产品配置 | 自定义竞品属性组合，构建假设产品 | ✅ 可用 |
| 份额预测 | 基于 Logit 模型预测市场份额 | ✅ 可用 |
| What-if 分析 | 实时调整价格/属性，观察份额变化 | ✅ 可用 |
| 竞品对标 | 支持最多 5 个产品同时对比 | ✅ 可用 |

#### ⑥ 可视化看板（Dashboard）

| 子功能 | 描述 | 状态 |
|--------|------|------|
| 研究总览 | 所有研究项目列表及状态 | ✅ 可用 |
| 属性重要性图表 | 柱状图 + 误差线展示 | ✅ 可用 |
| 效用分布图 | 小提琴图 / 箱线图展示个体效用分布 | ✅ 可用 |
| 市场模拟图 | 份额预测饼图 / 条形图 | ✅ 可用 |
| 细分对比图 | 分组柱状图对比不同人群 | ✅ 可用 |
| 结果导出 | 报告导出为 PDF / 数据导出为 CSV | ✅ 可用 |

## 四、已知限制（Known Limitations）

> ⚠️ **以下限制是 MVP 阶段的明确已知问题，上线前请务必评估影响。**

| 限制类别 | 具体限制 | 影响等级 | 说明 | 预计解决版本 |
|---------|---------|---------|------|------------|
| **数据持久化** | 数据为内存存储（MongoDB Docker 卷持久化，但无异地备份） | 🔴 高 | 宿主机故障或误删卷时数据不可恢复 | v0.2.0 |
| **安全认证** | 无用户登录/鉴权/权限机制 | 🔴 高 | 任何可访问网络者均可操作；不适合公网暴露 | v0.2.0 |
| **测试覆盖率** | 自动化覆盖率约 60%，部分异常路径和边界场景未覆盖 | 🟡 中 | 需依赖手动测试清单补充 | v0.2.0 |
| **国际化** | 界面仅支持中文；分析报告中部分统计术语为英文 | 🟡 中 | 中文业务用户可用，海外团队受限 | v0.3.0 |
| **多租户** | 无组织/团队/项目级隔离，数据全局可见 | 🟡 中 | 仅适合单项目单团队使用 | v0.3.0 |
| **LLM 稳定性** | 相同输入可能因温度参数产生不同结果 | 🟡 中 | 已设置温度=0.2，但仍存在随机性 | 持续优化 |
| **模型选择** | 暂不支持用户自选模型（固定路由） | 🟢 低 | 后台自动路由至 Claude-3.5-Sonnet | v0.2.0 |
| **数据导出格式** | 不支持 Excel 格式，仅 CSV / JSON | 🟢 低 | 可用第三方工具转换 | v0.3.0 |
| **实时协作** | 不支持多用户同时编辑同一研究 | 🟢 低 | 通过流程规避（串行操作） | v0.3.0 |

## 五、升级部署注意事项

### 5.1 从 v0.1.0 升级到未来版本的注意事项

| 注意事项 | 说明 |
|---------|------|
| **数据备份** | 升级前务必执行 `mongodump` 备份 MongoDB 数据；v0.2.0 可能引入数据库 Schema 变更 |
| **环境变量** | 新版本可能增加必需环境变量，请阅读对应版本发布说明；建议保留 `.env` 版本历史 |
| **LLM API Key** | 密钥存储在 `.env` 中，升级时确保新环境已配置；建议使用 Docker Secrets 或 Vault 管理 |
| **Redis 缓存** | 升级时 Redis 缓存会丢失，不影响核心数据，但首次请求可能稍慢 |
| **Docker 卷** | 升级时保留 `mongo_data` 和 `grafana_data` 卷；不要直接删除旧卷 |
| **Nginx 配置** | 如自定义了 nginx.conf，升级时需注意新版本配置变更，建议 diff 对比 |
| **Prometheus 指标** | 指标名称可能在新版本中变更，注意 Grafana 仪表盘是否需同步更新 |

### 5.2 首次部署检查清单

```markdown
[ ] 确认服务器满足最低配置：4核 CPU / 8GB 内存 / 50GB 磁盘
[ ] 安装 Docker ≥ 24.0 和 Docker Compose ≥ 2.20
[ ] 复制 `.env.example` 为 `.env` 并填写 API Key
[ ] 检查 8000/3000/9090/80 端口未被占用
[ ] 执行 docker-compose up -d 启动全部服务
[ ] 访问 http://localhost:8000/health 确认 API 健康
[ ] 访问 http://localhost:3000 确认 Grafana 可登录（默认 admin/admin）
[ ] 执行一次端到端测试：创建研究 → 生成画像 → 运行模拟 → 查看分析
[ ] 确认日志目录 ./logs 已创建并有写入权限
[ ] 确认备份目录 ./backups/mongo 已创建
```

### 5.3 回滚策略

如升级后发现问题，按以下步骤回滚：

```bash
# 1. 停止新版本容器
docker-compose down

# 2. 恢复数据备份（如已升级数据）
mongorestore --host localhost:27017 --db aicbc ./backups/mongo/aicbc-$(date +%Y%m%d)

# 3. 切回旧版本镜像标签（如使用版本标签）
docker-compose pull aicbc-api:v0.1.0  # 或本地构建的旧版本

# 4. 重新启动
docker-compose up -d
```

## 六、性能基准

| 指标 | 测试条件 | 基准值 | 备注 |
|------|---------|--------|------|
| 单次画像生成 | 100 个消费者 | ~60 秒 | 批量并发，Celery 2 Worker |

| 内存占用 | 全服务运行 | ~4GB | 含 MongoDB + Redis + API |
| 单次研究总成本 | 标准 100 人研究 | ¥300-500 | 含 LLM Token 费用 |
