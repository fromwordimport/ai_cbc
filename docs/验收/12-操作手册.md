## 二、分步骤操作指南

### Step 1：创建研究项目

**目标**：创建一个新的 CBC 研究，设定研究名称、目标产品类别和基本参数。

**操作步骤**：

1. 在首页点击「新建研究」按钮
2. 填写研究基本信息：
   - **研究名称**：如「洗碗机概念测试-Q3」
   - **产品类别**：从下拉框选择（大家电 / 小家电 / 数码 3C / 软件服务 / 虚拟商品）
   - **研究目标**：简要描述本次研究要回答的问题（如「确定最优定价区间」）
3. 点击「创建」按钮

> 📷 **截图位置**：
> - `tests/screenshots/02-study-create-form.png` — 新建研究表单（空状态）
> - `tests/screenshots/03-study-create-filled.png` — 表单已填写示例
> - `tests/screenshots/04-after-study-create.png` — 创建成功后的研究卡片

**常见问题**：
- **表单验证失败**：检查必填项（名称、类别）是否已填写。错误提示会高亮显示。
  > 📷 `tests/screenshots/05-empty-form-validation.png` — 表单验证错误提示示例

---

### Step 2：配置产品属性与水平

**目标**：定义研究中的产品属性（如价格、容量、品牌）及各属性下的水平值（如价格：3000元/4000元/5000元）。

**操作步骤**：

1. 在研究详情页，点击「问卷配置」标签
2. 点击「添加属性」按钮，逐个添加属性：
   - 属性名称（如「价格」）
   - 属性类型（价格 / 功能 / 品牌 / 外观 / 服务）
   - 是否含「None」选项（消费者可选择"都不选"）
3. 为每个属性添加水平值：
   - 输入水平值标签（如「2999元」）
   - 如有描述可填写水平描述（用于消费者理解）
4. 调整选择任务数量（推荐 8-12 个）
5. 点击「生成实验设计」

> 📷 **截图位置**：
> - `tests/screenshots/05-questionnaire-config.png` — 问卷配置页面（属性列表）
> - `tests/screenshots/06-questionnaire-config.png` — 展开的水平值配置
> - `tests/screenshots/07-questionnaire-expanded.png` — 完整的属性与水平配置

**最佳实践**：
- 属性数量建议 **3-6 个**，过多会导致消费者认知负荷增大
- 价格属性**务必包含**（用于计算支付意愿 WTP）
- 水平值之间应有 **明显差异**，避免消费者难以区分

---

### Step 3：设定目标人群

**目标**：定义本次研究的虚拟消费者人群画像维度，系统将据此生成消费者。

**操作步骤**：

1. 在研究详情页，点击「人群设定」标签
2. 配置人群维度：
   - **人口统计**：年龄范围（如 25-40 岁）、性别、城市级别、收入范围
   - **家庭结构**：婚姻状态、是否有子女、同住情况
   - **生活方式**：价值观标签（如「精致生活」「实用主义」）
   - **产品经验**：使用历史、品牌认知度、当前痛点
3. 设定生成数量（MVP 阶段建议 50-200 人）
4. 点击「保存人群设定」

> 📷 **截图位置**：
> - `tests/screenshots/deep-10-settings.png` — 人群设定与参数配置界面

**提示**：人群维度越具体，生成的画像越真实，但会提高单次成本。建议首次研究使用 100 人规模。

---

### Step 4：生成虚拟消费者画像

**目标**：基于 Step 3 的设定，由 LLM 生成具有完整背景故事和偏好特征的虚拟消费者。

**操作步骤**：

1. 点击「生成画像」按钮
2. 系统弹出确认窗口，显示预计生成数量与预估成本
3. 确认后，系统开始异步生成（进度条显示实时进度）
4. 生成完成后，进入「画像管理」页面查看

> 📷 **截图位置**：
> - `tests/screenshots/deep-01-persona-page.png` — 画像管理总览页面
> - `tests/screenshots/deep-02-persona-modal.png` — 生成画像确认弹窗
> - `tests/screenshots/deep-03-persona-form-filled.png` — 画像生成参数表单
> - `tests/screenshots/deep-04-persona-generated.png` — 生成完成的画像卡片列表
> - `tests/screenshots/07-persona-manager.png` / `08-persona-manager.png` — 画像管理器界面

**查看单个画像**：
- 点击画像卡片，可展开查看完整的自然语言描述、人口统计标签、产品偏好等
- 如画像不符合预期，可点击「重新生成」替换该个体

> 📷 **截图位置**：
> - `tests/screenshots/deep-02-after-generate.png` — 生成后查看单个画像详情

**提示**：生成 100 个画像通常需要 1-2 分钟。期间请勿关闭页面或刷新。

### Step 5：查看与验证问卷

**目标**：在启动模拟前，人工审校 CBC 问卷设计是否合理。

**操作步骤**：

1. 点击「问卷预览」标签
2. 逐页浏览选择任务：
   - 检查属性组合是否合理（无重复、无矛盾）
   - 检查选择集是否覆盖不同属性组合
   - 检查 "None" 选项是否适当
3. 如发现问题，返回 Step 2 修改属性/水平后重新生成实验设计

> 📷 **截图位置**：
> - `tests/screenshots/15-questionnaire-preview.png` — 问卷预览界面（选择任务展示）

**验证要点**：
- [ ] 每个选择任务中，所有产品都包含相同属性（仅水平值不同）
- [ ] 属性组合不存在明显占优（如一个产品在各方面都更好）
- [ ] 价格变化范围符合业务常识
- [ ] 总任务数不会让消费者疲劳（建议 ≤ 15 个）

---

### Step 6：启动模拟作答

**目标**：让虚拟消费者逐题完成 CBC 问卷选择，并记录选择理由。

**操作步骤**：

1. 确认画像和问卷已准备就绪
2. 点击「启动模拟」按钮
3. 系统弹出确认窗口，显示：
   - 模拟消费者数量
   - 总选择任务数
   - 预估 LLM 调用成本
   - 预计完成时间
4. 确认后，系统进入模拟执行状态
5. 可在「模拟实验室」实时观察进度和单个消费者的选择过程

> 📷 **截图位置**：
> - `tests/screenshots/deep-05-interview.png` — 模拟作答界面（选择任务）
> - `tests/screenshots/deep-06-interview-select-open.png` — 展开选项的选择界面
> - `tests/screenshots/deep-07-interview-response.png` — 消费者选择后展示理由
> - `tests/screenshots/deep-08-interview-round2.png` — 多轮任务连续作答
> - `tests/screenshots/12-interview-lab.png` / `14-interview-lab.png` — 模拟实验室总览

**模拟状态说明**：

| 状态 | 含义 | 操作 |
|------|------|------|
| 排队中 | 任务已提交，等待 Celery Worker | 无需操作，自动执行 |
| 进行中 | 正在逐题模拟 | 可实时查看进度 |
| 已完成 | 全部消费者已完成 | 进入 Step 7 查看分析 |
| 部分失败 | 个别消费者模拟失败 | 可重试失败批次 |

### Step 7：查看分析结果

**目标**：查看 HB 联合分析结果，包括属性重要性、效用值、WTP、细分对比等。

**操作步骤**：

1. 模拟完成后，点击「分析结果」标签
2. 系统首先执行 HB 分析（自动进行，通常 1-2 分钟）
3. 分析完成后，依次查看各模块：

#### 7.1 属性重要性看板

> 📷 **截图位置**：
> - `tests/screenshots/08-importance-dashboard.png` / `09-importance-dashboard.png` — 属性重要性柱状图

- 查看各属性的相对重要性百分比
- 检查价格属性是否排名合理（通常价格重要性较高）
- 与业务直觉交叉验证：TOP3 属性是否符合预期？

#### 7.2 效用值分布

> 📷 **截图位置**：
> - `tests/screenshots/10-after-hb-analysis.png` — HB 分析后的效用值展示

- 查看各属性水平的效用值（正向 = 偏好，负向 = 厌恶）
- **关键检查**：价格效用值是否全部为负？（如为正，说明模型异常，需联系技术团队）
- 查看效用值的个体分布离散程度（差异大 → 细分群体有意义）

#### 7.3 细分群体对比

> 📷 **截图位置**：
> - `tests/screenshots/10-segment-comparison.png` / `12-segment-comparison.png` — 细分对比图表

- 选择分组维度（如性别、城市级别、年龄组）
- 对比不同细分群体的属性重要性差异
- 寻找统计显著差异（p < 0.05）作为洞察线索

#### 7.4 分析状态追踪

> 📷 **截图位置**：
> - `tests/screenshots/11-analysis-status.png` / `13-analysis-status.png` — 分析状态与进度

**分析状态说明**：

| 状态 | 含义 |
|------|------|
| 排队中 | 等待分析队列 |
| 分析中 | HB 模型正在采样（MCMC 运行中） |
| 收敛验证 | 检查 R-hat 与 ESS |
| 完成 | 结果可查看 |
| 失败 | 模型未收敛或数据异常 |

### Step 8：市场模拟（What-if 分析）

**目标**：基于分析结果，模拟不同产品配置的市场份额，辅助定价和配置决策。

**操作步骤**：

1. 点击「市场模拟」标签
2. 添加产品配置（最多 5 个产品）：
   - 为每个产品设定属性组合（如「产品 A：2999元 + 13套 + 嵌入 + 品牌 X」）
   - 系统基于 Logit 模型预测每个产品的市场份额
3. 调整产品参数，实时观察份额变化
4. 对比不同场景（如提价 500 元对份额的影响）

> 📷 **截图位置**：
> - `tests/screenshots/09-market-simulator.png` / `11-market-simulator.png` — 市场模拟界面
> - `tests/screenshots/deep-09-market-sim.png` — 市场模拟详细配置

**使用场景示例**：

| 场景 | 操作 | 决策支持 |
|------|------|---------|
| 定价探索 | 固定其他属性，调整价格水平 | 找到最优价格点 |
| 配置优化 | 固定价格，调整功能属性 | 找到最具竞争力的配置 |
| 竞品对标 | 输入竞品参数，预测与竞品的市场格局 | 差异化定位 |
| 新品引入 | 添加新产品配置，观察对现有产品份额的蚕食 |  cannibalization 评估 |

## 三、响应模拟器（独立工具）

**用途**：不运行完整研究，快速测试单个消费者面对特定选择集的决策行为。

**操作步骤**：
1. 进入「响应模拟器」页面
2. 选择一个已有的消费者画像（或临时创建）
3. 输入选择任务（属性组合）
4. 点击「模拟决策」，查看该消费者的选择及理由

> 📷 **截图位置**：
> - `tests/screenshots/16-response-simulator.png` / `tests/screenshots/deep-11-response-sim.png` — 响应模拟器界面

**适用场景**：
- 验证单个画像的行为是否符合预期
- 快速测试假设场景（如"高端消费者会选哪个？"）
- 调试和优化画像质量

## 五、常见问题与故障排除（FAQ）

### Q1：系统加载很慢 / 页面空白

**可能原因**：
- 后端服务未启动或正在重启
- 网络连接不稳定

**解决方法**：
1. 检查浏览器控制台（F12 → Network）是否有 API 请求失败（红色）
2. 访问 `/health` 端点检查后端状态
3. 联系管理员确认服务状态

---

### Q2：生成画像时进度卡住

**可能原因**：
- LLM API 速率限制或暂时不可用
- Celery Worker 未运行或队列堆积

**解决方法**：
1. 等待 5-10 分钟，系统会自动重试
2. 如仍卡住，点击「取消」后重新启动
3. 检查「设置」页面中的成本监控，确认是否触发熔断
4. 联系管理员检查 Worker 状态

---

### Q3：分析结果中价格效用为正数

**问题严重性**：🔴 高 — 这是模型异常，结果不可信

**可能原因**：
- 价格水平设计不合理（如价格差异太小）
- 消费者样本量不足
- 模型未收敛

**解决方法**：
1. 检查价格属性是否至少包含 3 个明显差异的水平
2. 检查模拟作答数据量是否充足（建议 ≥ 50 人）
3. 重新运行分析，或增加采样迭代次数
4. 如问题持续，联系技术团队（小数 / 小端）

---

### Q4：市场模拟的份额预测不符合直觉

**可能原因**：
- 产品配置设置过于极端（如某产品各方面都最优）
- 未考虑 "None" 选项的影响

**解决方法**：
1. 确保产品配置之间有 realistic 的 trade-off
2. 检查是否启用了 "None" 选项及其影响
3. 与属性重要性结果交叉验证

---

### Q5：如何判断分析结果是否可信？

**自检清单**：
- [ ] 属性重要性 TOP3 与业务直觉基本一致
- [ ] 价格效用值为负（全部水平）
- [ ] HB 模型 R-hat < 1.1（系统已自动检查）
- [ ] 样本量 ≥ 50 人
- [ ] 细分群体差异可被统计检验确认（p < 0.05）

如以上全部满足，结果可信度 **高**；
如有 1-2 项不满足，结果 **需谨慎解读**；
如 ≥ 3 项不满足，建议 **重新运行研究或联系技术团队**。

---

### Q6：研究数据丢失了 / 找不到之前的研究

**MVP 已知问题**：数据持久化依赖 Docker 卷，如宿主机重启或容器重建可能丢失数据。

**解决方法**：
1. 检查研究是否保存在本地浏览器缓存中（尝试刷新页面）
2. 联系管理员检查 MongoDB 数据卷状态
3. 建议定期手动导出研究数据作为备份

---

### Q7：如何控制单次研究成本？

**成本控制建议**：
1. 在「设置」页面查看成本估算（`tests/screenshots/17-settings.png` / `13-settings.png`）
2. 减少消费者数量（50-100 人通常足够概念验证）
3. 减少选择任务数量（8-10 个即可）
4. 避免频繁重复生成（每次生成都会产生 LLM 调用成本）
5. 关注成本监控告警，如接近 ¥500 上限请暂停或调整参数

---

### Q8：界面出现英文术语 / 看不懂统计指标

**MVP 已知问题**：部分分析输出含英文统计术语（如 R-hat、ESS、Logit）。

**常用术语速查**：

| 术语 | 中文含义 | 说明 |
|------|---------|------|
| R-hat | 收敛诊断指标 | < 1.1 表示模型收敛 |
| ESS | 有效样本量 | > 100 表示采样充分 |
| MAPE | 平均绝对百分比误差 | 模型精度指标，越小越好 |
| WTP | 支付意愿 | 消费者愿意为某属性多付多少钱 |
| Logit | 逻辑特模型 | 市场份额预测的基础模型 |
| Utility | 效用值 | 数值越高表示越偏好 |
| Importance | 重要性 | 百分比，越高表示该属性越关键 |

## 六、安全与合规提醒

1. **数据保密**：研究数据可能包含产品策略信息，请勿截图外发至未授权渠道
2. **结果边界**：AI_CBC 产出仅用于内部假设检验，不可作为最终定价唯一依据
3. **公平性审查**：如研究涉及特定人群（如性别、年龄、地域），请确保人群设定无系统性偏见
4. **公网暴露**：MVP 无认证机制，请勿将系统直接暴露至公网
