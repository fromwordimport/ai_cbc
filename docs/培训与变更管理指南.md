## 二、培训体系设计

### 2.1 培训内容矩阵

| 模块 | 目标受众 | 时长 | 形式 | 核心内容 |
|------|---------|------|------|---------|
| **M1: 系统概览** | 全员 | 30分钟 | 演示 | 是什么、能做什么、不能做什么 |
| **M2: 虚拟消费者原理** | 使用者 | 45分钟 | 讲解+讨论 | LLM如何"演"消费者、验证逻辑、局限性 |
| **M3: 结果解读** | 使用者 | 60分钟 | 讲解+实操 | 效用值、重要性、WTP、市场份额 |
| **M4: 系统操作** | 使用者 | 90分钟 | 实操 | 端到端操作：输入→输出 |
| **M5: 高级应用** | 资深使用者 | 60分钟 | 案例研讨 | What-if模拟、细分分析、交叉验证 |
| **M6: 问题排查** | 使用者 | 30分钟 | 手册自学 | 常见错误、反馈渠道、升级路径 |

### 2.2 培训工作坊设计（M3+M4 合并，半天）

```markdown
# AI_CBC 实操工作坊（4小时）

## 开场（15分钟）
- 破冰："你最近一次做消费者调研花了多久？"
- 工作坊目标预告

## 第一部分：理解虚拟消费者（45分钟）

### 讲解（20分钟）
【类比引入】
"想象你在招聘演员来模拟焦点小组。每个演员（虚拟消费者）都有：
• 一份详细的人物小传（画像）
• 导演给的具体情境（问卷场景）
• 即兴发挥但有框架的表演（LLM决策）

关键问题：演员演得像不像？我们用7个维度来评估。"

【验证逻辑展示】
• 展示一个"真实消费者"vs"虚拟消费者"的盲测案例
• 解释7维度评分
• 强调"不是替代真实消费者，而是快速试错"

### 互动（15分钟）
【小组讨论】
"以下场景中，你会信任AI_CBC的结果吗？为什么？
A. 新品概念快速筛选（3个概念 → 1个）
B. 最终上市定价决策
C. 年度品牌健康度追踪
D. 小众人群需求探索"

### 答疑（10分钟）

## 休息（15分钟）

## 第二部分：解读分析结果（60分钟）

### 讲解（25分钟）
【效用值解读】
"效用值就像'好感度分数'：
• 正数 = 喜欢，负数 = 不喜欢
• 数值大小 = 喜欢/不喜欢的程度
• 不同属性间不能直接比较（单位不同）"

【属性重要性解读】
"重要性 = 这个属性在决策中的'话语权'：
• 价格重要性45% = 近一半的决策由价格驱动
• 品牌重要性25% = 品牌是第二重要因素
• 所有重要性加起来 = 100%"

【WTP解读】
"WTP = 消费者愿意为某项升级掏多少钱：
• 存储从128GB→256GB，WTP=¥300
• 意味着：如果升级价差 ≤ ¥300，消费者会买账"

【市场份额模拟解读】
"份额预测 = 如果市场上只有这几个选项，大家会怎么选：
• 不是真实市场份额（忽略了很多现实因素）
• 但在'相对吸引力'上是有参考价值的"

### 实操（25分钟）
【案例练习】
提供一份真实项目的分析报告，要求学员：
1. 指出最重要的3个发现
2. 给出一个定价建议
3. 识别一个需要警惕的偏差

### 讨论（10分钟）
各组分享解读，讲师点评

## 午餐（60分钟）

## 第三部分：系统操作（90分钟）

### 演示（20分钟）
讲师完整演示一次研究流程：
1. 输入产品属性和研究目标
2. 查看生成的消费者画像
3. 查看CBC问卷
4. 运行分析
5. 查看报告

### 实操（50分钟）
【分组任务】
每组使用系统完成一个真实业务问题：
"假设我们要推出一款新手机，价格在3000-5000区间，
请用AI_CBC分析消费者最看重的3个属性。"

任务分解：
1. 设定属性和水平（10分钟）
2. 生成消费者并快速检查画像（15分钟）
3. 运行分析（自动，等待5分钟）
4. 解读结果并准备汇报（20分钟）

### 汇报与反馈（20分钟）
每组5分钟汇报，讲师点评

## 结束（15分钟）
- 总结要点
- 发放证书/徽章
- 收集反馈
- 建立后续支持群
```

## 三、"感觉不合理"反馈转译流程

### 3.1 反馈收集与结构化

```
业务用户反馈
   │ "这个价格段消费者不可能这么看重品牌"
   │
   ▼
小培收集
   │
   ├── 记录原始表述
   ├── 确认具体场景（产品、价位、人群）
   ├── 获取用户预期（"你认为应该是什么结果？"）
   └── 了解结论影响（"这个结果会影响什么决策？"）
   │
   ▼
结构化分类
   │
   ├── 类型A：结果反直觉但可能正确
   │   → 转小联/小数验证
   │   → 如果是统计事实 → 准备教育材料
   │
   ├── 类型B：虚拟消费者行为不真实
   │   → 转小示改进
   │   → 调整提示模板或偏好注入
   │
   ├── 类型C：模型估计偏差
   │   → 转小数改进
   │   → 调整先验或模型选择
   │
   ├── 类型D：系统展示误导
   │   → 转小前改进
   │   → 优化可视化或增加置信区间
   │
   └── 类型E：用户误解
   │   → 小培跟进教育
   │   → 补充培训或FAQ
   │
   ▼
反馈闭环
   │
   ├── 问题修复后 → 通知原反馈者
   ├── 邀请验证修复效果
   └── 记录为案例（成功/教训）
```

### 3.2 反馈转译示例

| 原始反馈 | 结构化分析 | 转译结果 | 处理负责人 |
|---------|-----------|---------|-----------|
| "40岁消费者不可能选最便宜选项" | 画像收入与选择不匹配 | 画像价格敏感度与收入档位的匹配规则需调整 | 小示 |
| "价格重要性100%，其他为0" | 模型估计异常 | 可能是属性间相关性过高或样本量不足 | 小数 |
| "报告里WTP是负数，看不懂" | 计算错误展示 | WTP计算异常值未过滤，展示需优化 | 小数+小前 |
| "为什么华为比苹果效用高？不合理" | 用户预期与结果冲突 | 可能是真实偏好，需验证或解释 | 小联 |
| "操作界面太复杂，找不到分析按钮" | 用户体验问题 | 界面布局优化，增加引导 | 小前 |

## 五、FAQ

### 5.1 关于虚拟消费者

**Q: AI生成的消费者和真实消费者有什么区别？**
A: 虚拟消费者是LLM根据规则"扮演"的消费者。关键区别：
- ✅ 可控：可以精确设定偏好结构
- ✅ 透明：每个决策都有完整理由
- ✅ 快速：分钟级生成数百个
- ⚠️ 局限：可能无法完全捕捉真实世界的复杂性
- ⚠️ 适用：前置筛选、假设验证、小众人群探索
- ❌ 不适用：最终上市决策的唯一依据

**Q: 怎么知道虚拟消费者"像不像真人"？**
A: 我们用三重验证：
1. 自动评分：7维度真实性总分（满分14分）≥ 9分及格，≥ 12分优秀
2. 统计检验：与真实人口分布对比
3. 人工盲测：研究员判断，要求误判率 ≥ 30%

**Q: 虚拟消费者会不会有偏见？**
A: 我们在Prompt中嵌入了6条公平性硬规则，每批次都经过偏见审计。但如同真实调研也可能有样本偏差，建议关键决策前与真实消费者研究交叉验证。

### 5.2 关于分析结果

**Q: 效用值是什么意思？**
A: 效用值 = "好感度分数"。正数表示喜欢这个水平，负数表示不喜欢。数值越大表示越喜欢/越不喜欢。注意：不同属性间的效用值不能直接比较大小。

**Q: 属性重要性加起来为什么是100%？**
A: 重要性是"相对权重"。就像你买东西时，价格是决定性因素（45%），品牌其次（25%），其他因素各占一部分，加起来就是100%。

**Q: WTP（支付意愿）为负数是什么意思？**
A: 正常情况下WTP应为正数。如果出现负数，说明模型估计异常，需要检查数据或联系技术支持。

**Q: 市场份额预测可信吗？**
A: 市场份额预测反映的是"相对吸引力"，不是真实市场份额。它假设消费者只在问卷中的选项中选择，忽略了渠道、促销、口碑等现实因素。适合用于"哪个配置相对更吸引人"的判断，不适合预测真实销量。

### 5.3 关于系统使用

**Q: 一次研究需要多长时间？**
A: 标准配置（100消费者×12题）约需30-60分钟，其中大部分是系统自动运行。人工投入主要在前期设定属性和后期解读结果。

**Q: 最多可以生成多少个消费者？**
A: 技术上无硬性限制，但建议单批次不超过500人（成本和时间考虑）。如需更多，可分批次生成后合并分析。

**Q: 结果可以导出吗？**
A: 支持导出为PDF报告、Excel数据表、PPT图表。也可以直接复制报告中的内容。
