# 远程 feature 分支合并与 CI/CD 门禁实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 安全地处理 4 个远程 `feature/*` 分支：清理 2 个已合并分支，通过 PR + CI 逐个合并 2 个未合并分支。

**Architecture:** 采用保守串行策略。先清理本地工作区并同步远程 `master`；对未合并分支，每个分支独立创建 PR、等待 GitHub Actions CI 通过、使用 merge commit 合并、最后清理远程分支。任何冲突或 CI 失败都必须停止并请求授权。

**Tech Stack:** Git, GitHub CLI (`gh`), GitHub Actions

### Task 4: 合并第一个未合并分支（feature/cicd-pipeline-flag-scripts）

- [ ] **Step 1: 创建本地跟踪分支**

  Run:
  ```bash
  git checkout -b feature/cicd-pipeline-flag-scripts origin/feature/cicd-pipeline-flag-scripts
  ```

  Expected:
  - 输出 `Branch 'feature/cicd-pipeline-flag-scripts' set up to track 'origin/feature/cicd-pipeline-flag-scripts'.`

- [ ] **Step 2: 本地预检合并（无冲突测试）**

  Run:
  ```bash
  git merge --no-ff --no-commit origin/master
  ```

  Expected（无冲突）:
  - 输出 `Automatic merge went well; stopped before committing as requested`

  若出现冲突，立即停止，运行 `git diff --name-only --diff-filter=U` 列出冲突文件，向用户汇报并请求授权解决。

- [ ] **Step 3: 取消本地预检合并**

  Run:
  ```bash
  git merge --abort
  ```

  Expected:
  - 工作区恢复干净

- [ ] **Step 4: 创建 Pull Request**

  Run:
  ```bash
  gh pr create \
    --base master \
    --head feature/cicd-pipeline-flag-scripts \
    --title "ci: merge feature/cicd-pipeline-flag-scripts" \
    --body "合并远程分支 feature/cicd-pipeline-flag-scripts，需通过 CI 门禁。"
  ```

  Expected:
  - 输出 PR URL，例如 `https://github.com/fromwordimport/ai_cbc/pull/<number>`

- [ ] **Step 5: 获取 PR 编号并等待 CI 结果**

  Run:
  ```bash
  PR_NUM=$(gh pr list --head feature/cicd-pipeline-flag-scripts --json number --jq '.[0].number')
  gh pr checks "$PR_NUM"
  ```

  Expected（通过）:
  - 第一条命令输出 PR 编号
  - 所有 checks 状态为 `pass`

  若存在 `fail` 状态，运行 `gh run view --failed` 查看失败 job 日志，向用户汇报失败原因并请求授权修复。

- [ ] **Step 6: 合并 Pull Request（需 CI 通过授权）**

  Run:
  ```bash
  PR_NUM=$(gh pr list --head feature/cicd-pipeline-flag-scripts --json number --jq '.[0].number')
  gh pr merge "$PR_NUM" --merge --subject "ci: merge feature/cicd-pipeline-flag-scripts"
  ```

  Expected:
  - 输出 `✓ Merged pull request <number> ...`

- [ ] **Step 7: 更新本地 master**

  Run:
  ```bash
  git checkout master
  git pull origin master
  ```

  Expected:
  - `git log --oneline --decorate -1` 显示最新的 merge commit

- [ ] **Step 8: 请求删除远程功能分支授权并执行**

  Run:
  ```bash
  git push origin --delete feature/cicd-pipeline-flag-scripts
  git fetch --prune origin
  ```

  Expected:
  - 远程分支删除成功
  - `git branch -r` 不再显示该分支

## 回滚策略

若合并后发现异常，按以下顺序处理：

1. **尚未推送到 master 的本地合并**：
   ```bash
   git reset --hard origin/master
   ```

2. **已推送到 master 的合并**：
   ```bash
   git revert -m 1 <merge-commit-hash>
   git push origin master
   ```

3. **已删除的远程 feature 分支**：
   若需要恢复，从本地分支或引用日志重建：
   ```bash
   git reflog
   git push origin <local-branch-name>:feature/xxx
   ```
