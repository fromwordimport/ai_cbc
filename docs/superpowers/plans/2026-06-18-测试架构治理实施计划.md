# AI_CBC 测试架构治理实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 AI_CBC 当前混乱的测试模块重构为按测试金字塔分层的清晰架构，消除冗余与 CI/CD 冲突，建立可持续的测试质量门禁。

**Architecture:** 通过 `pytest.mark` 分层体系（unit/integration/e2e/security/redteam/performance）控制执行范围，配合目录重构与测试数据工厂，实现 CI 快速反馈、Nightly 全量回归、Security-scan 策略一致。

**Tech Stack:** pytest, uv, ruff, mypy, bandit, pip-audit, GitHub Actions, Vitest, npm audit, Locust, Trivy.
