## 第一部分：各层关键发现

### Layer 1 — 统计计算引擎层

**评级：C+**

#### P0-CRITICAL（3项）

- ✅ **已修复** **[P0-STAT-001]** `segment_comparison.py:208-212` — **多重比较未校正（复现问题）**。对每个属性独立运行 Welch's t-test，7 个属性下 FWER ≈ 30%。这是上一轮审查标记的 P1 问题，**至今未修复**。

- ✅ **已修复** **[P0-STAT-002]** `segment_comparison.py:27-28` — **Hotelling's T² 缺少样本量检查**。当 `n_a < p` 或 `n_b < p` 时协方差矩阵必然奇异，仅靠 `LinAlgError` 捕获而非前置检查。

- ✅ **已修复** **[P0-STAT-003]** `segment_comparison.py:158` — **置换检验 p 值缺少 +1 平滑**。使用 `np.mean(perm_stats >= obs_stat)` 而非标准公式 `(count+1)/(n_perm+1)`，导致 p=0 的可能性。

#### P1-MAJOR（4项）

- 无协方差矩阵条件数检查——接近奇异的矩阵可能不触发异常但结果不稳定
- `test_type="welch"` 命名误导——实际运行 Hotelling's T² 作为整体检验
- **Cohen's d 使用合并方差与 Welch 假设矛盾**——Welch 假设方差不齐，但效应量用合并标准差
- **LatentClass 模型静默降级为 HB**——API 接受 `latent_class` 但静默回退，用户不知情

### Layer 2 — API/数据模型层

**评级：C**

#### P0-CRITICAL（2项）

- ✅ **已修复** **[P0-API-001]** `store.py:131-136` — **Store key 碰撞**。`save_segment_comparison` 仅用 `analysis_id` 作为 key，同一分析下不同 segment pair 互相覆盖。

- ✅ **已修复** **[P0-API-002]** `routes.py:432` — **路由层调用未传 segment 参数**。调用签名与 store 方法不匹配。

#### P1-MAJOR（7项）

- 缺少缓存读取逻辑——每次请求重新计算，已存结果不被复用
- 缺少 `segment_a == segment_b` 校验
- 缺少 segment 在数据中不存在的校验（`n_a=0` 或 `n_b=0` 会静默产生 NaN）
- **interpretation 为规则模板生成，过于粗糙**——仅 2 种模板，缺少方向性描述和效应量
- **Agent 工具集缺少 `compare_segments` 工具**——LLM Agent 无法通过工具调用执行分段对比
- **AnalysisAgent 10 步管线缺少分段比较步骤**
- `overall_test.statistic` 语义在不同 test_type 下不一致（T² vs Euclidean distance）

## 第二部分：跨层断裂分析

### 断裂 #1：分段标签数据流不完整

```
画像生成(segment字段) → 问卷填写(choice_records.segment) → 分段对比(mask过滤)
                                                               ↑
                                          LatentClass自动发现 —— 完全缺失
```

当前完全依赖预定义 segment 标签。LatentClass 模型未实现意味着**系统无法基于数据自动发现新的人群细分**。

### 断裂 #2：前后端类型不兼容

```
后端 models.py              前端 api.ts
per_attribute: list[...]  vs  comparisons: SegmentComparisonItem[]
n_a, n_b, overall_test    vs  完全缺失
interpretation: str       vs  未定义
```

根因：前端开发使用 mock 数据独立开发，未与后端真实 API 联调验证。

### 断裂 #3：Agent/报告/看板管线缺失分段对比

```
AnalysisAgent 10-step pipeline: ✅ validate → preprocess → fit → convergence → importance → WTP → report
                                  ❌ 无 segment comparison 步骤
subsystem_tools: ✅ 6 tools (fit/importance/wtp/simulate/run_full/select)
                  ❌ 无 compare_segments 工具
Report:           ✅ 收敛评估 + 重要性 + WTP
                  ❌ 无分段洞察章节
```
