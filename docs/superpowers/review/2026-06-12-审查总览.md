## 二、已修复的关键问题（亮点）

本轮审查-修复迭代解决了 **46 个 P0 级后端缺陷**，其中最具影响力的修复：

| 修复项 | 影响范围 | 说明 |
|--------|---------|------|
| 价格标准化量纲恢复 | 重要性看板 + WTP | `importance.py` 和 `wtp.py` 增加 `price_std` 还原，重要性排序和 WTP 金额从"不可靠"变为"可靠" |
| A-efficiency 公式修正 | 问卷配置 | `p/trace(M)` → `p/trace(M⁻¹)`，效率指标不再反向 |
| D-efficiency 阈值对齐 | 问卷配置 | 0.5 → 0.80，与设计文档一致 |
| `rhat_by_param` 数据污染修复 | 重要性看板 | API 诊断数据从"完全失实"恢复正确 |
| 偏见审计 REJECT+REGENERATE | 画像管理 | 失败画像自动重试，批量产量可控 |
| 偏见审计 CRITICAL 级别 | 画像管理 | 一票否决权机制恢复 |
| Celery 任务超时+消毒 | 分析任务状态 | Worker 安全防护就位 |
| 进程间存储隔离缓解 | 分析任务状态 | Celery AsyncResult 桥接 + 僵尸检测 |
| `life_stage` 字段+赋值修复 | 画像管理 | 数据断裂修复 |
| City 词汇表统一 | 画像管理 | Schema 校验不再误拒 |
| 倒排索引语义修复 | 画像管理 | 检索功能恢复正常 |
| 品类绑定解除（后端） | 问卷+市场模拟 | API 支持自定义属性、ProductScenario 动态化 |
| study_id sanitize 全线覆盖 | 多模块 | 输入安全防护 |
