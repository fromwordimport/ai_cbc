# 前后端 API 覆盖率缺口 — 审查报告

> **版本**：v1.0
> **日期**：2026-06-13
> **审查方法**：后端路由全量枚举 × 前端页面逐页功能对照
> **审查范围**：6 个后端路由文件 vs 13 个前端页面

---

## 执行摘要

对 AI_CBC 全部后端 API 端点与前端页面的调用关系进行了逐端点对照审计。

**整体评级：D（后端功能覆盖率 ~60%，关键生成/配置链路前端几乎不可控）**

| 模块 | 后端端点数 | 前端已使用 | 前端未暴露 | 覆盖率 |
|------|-----------|-----------|-----------|--------|
| 问卷配置 | 6 | 3 | 3 | 50% |
| 画像管理 | 6 | 3 | 3 | 50% |
| 行为模拟 | 3 | 1 | 2 | 33% |
| 作答模拟 | 3 | 2 | 1 | 67% |
| 分析管线 | 9 | 8 | 1 | 89% |
| 系统运维 | 4 | 2 | 2 | 50% |
| **合计** | **31** | **19** | **12** | **~61%** |

**核心结论**：分析管线前端覆盖最好（仅缺模型类型选择器），行为模拟最差（3 个端点只用了 1 个）。最大痛点不在"端点缺失"而在"参数僵化"——多个已使用的端点，前端将关键参数硬编码（`hb` / `stochastic` / 默认属性），导致后端灵活性被架空。

## 一、问卷配置（Questionnaire）

### 端点对照

| # | 后端端点 | 方法 | 前端状态 | 说明 |
|---|---------|------|---------|------|
| 1 | `/studies` | POST | ⚠️ 半覆盖 | `StudyCreate.tsx` 使用，但**不传 attributes/design_parameters**，始终使用服务端洗碗机默认值 |
| 2 | `/studies` | GET | ✅ 覆盖 | `QuestionnaireConfig.tsx` 列表 |
| 3 | `/studies/{id}` | GET | ❌ 未使用 | `api.ts` 中定义了 `getStudy` 但从未被任何页面导入 |
| 4 | `/studies/{id}` | PUT | ❌ 未使用 | 创建后无法编辑研究 |
| 5 | `/studies/{id}` | DELETE | ✅ 覆盖 | |
| 6 | `/studies/{id}/generate` | POST | ⚠️ 半覆盖 | 仅在 `StudyCreate.tsx` 创建时联动触发，`QuestionnaireConfig.tsx` 无独立触发按钮 |
| 7 | `/studies/{id}/questionnaire` | GET | ✅ 覆盖 | |

### 缺口详情

- **自定义属性/设计参数表单缺失**：后端 `CreateStudyRequest` 已支持 `attributes` 和 `design_parameters` 字段，前端表单只有 `study_id`/`product_category`/`research_goal`/`target_segments` 四个字段
- **编辑端点闲置**：`PUT /studies/{id}` 后端完整实现（含 `StudyUpdateRequest` schema），前端无编辑页面
- **独立生成按钮缺失**：已创建的 Study 在 `QuestionnaireConfig.tsx` 中只能查看已生成问卷，无法重新生成

## 参数僵化清单

除端点缺失外，以下"已使用端点"的关键参数被前端硬编码，用户无法控制：

| 页面 | 端点 | 硬编码参数 | 后端实际支持 |
|------|------|-----------|------------|
| `StudyCreate.tsx` | `POST /studies` | 不传 attributes/design_parameters | 完整自定义属性 + 设计参数 |
| `AnalysisStatus.tsx` | `POST .../analyze` | `model_type: 'hb'` | hb / mnl / latent_class |
| `ResponseSimulator.tsx` | `POST .../simulate-responses` | `mode: 'stochastic'` | stochastic / llm / deterministic |
| `SegmentComparison.tsx` | `GET .../segment-comparison` | 测试类型硬编码 | hotelling / welch / permutation |
| `InterviewLab.tsx` | `POST .../converse` | 只用单轮对话 | 支持多轮 interview + purchase-decision |

## 与前期审查的关联

本报告与前期六次审查（2026-06-12）构成互补——前期关注后端代码质量（P0 Bug、数据模型、算法正确性），本报告关注"后端已修复/已实现但前端不可见"的功能断层。最典型的例子：

- 六次审查中的 P0-API-001（硬编码洗碗机属性）已在后端修复，`CreateStudyRequest` 已支持自定义 attributes，但前端 `StudyCreate.tsx` 表单并未同步增加对应字段——后端灵活性被前端架空
- 同理，`PUT /studies/{id}` 在第一次审查时为 P1-MAJOR 缺失项，后端已补充实现，但前端至今未接入
