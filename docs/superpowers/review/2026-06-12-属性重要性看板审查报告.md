## 第一部分：各层关键发现

### Layer 1 — HB建模引擎层

**评级：C+**

#### P0-CRITICAL

- ✅ **已修复** **[P0-HB-001]** `tasks.py:178-189` — **`rhat_by_param`/`ess_by_param` 数据污染**。`ConvergenceDiagnostics` 的 `rhat_by_param` 被错误赋值为 `population_mu`（总体效用均值），`ess_by_param` 被错误赋值为 `population_sigma`（总体标准差）。API 返回的逐参数收敛诊断数据完全失实，下游任何依赖逐参数 R-hat/ESS 的系统都会得到虚假信息。

- ✅ **已修复** **[P0-HB-002]** `preprocessing.py:82-145` — **`validate_dataset()` 缺失 2 条设计规范校验规则**。设计文档定义的 5 条规则中：`attributes_complete`（ERROR 级别）和 `none_option_rate`（WARNING 级别）完全未实现。且 `one_choice_per_task` 使用了 ERROR 而非规范定义的 WARNING 级别。

#### P1-MAJOR（5项）

- 无 MCMC 采样失败时的自动重试/降级策略（HBConfig 定义了配置但 fit() 不检查收敛）
- `HBResult` 缺少模型拟合统计量（log_likelihood/AIC/BIC），无法与 MNL 进行模型比较
- API 层 `prior_config` 是死参数——被接受但静默丢弃
- 整个建模管线零缺失值处理逻辑
- `to_long_format()` 缺少规范要求的 `none_option` 列

### Layer 2 — 后处理计算层

**评级：D（存在阻断性计算错误）**

#### P0-CRITICAL（3项——互相关联的系统性错误链）

- ✅ **已修复** **[P0-PP-001]** `importance.py:41-47` — **价格重要性计算单位混用**。预处理层将价格做了 z-score 标准化（`(price-mean)/sigma`），但重要性计算用标准化系数直接乘原始价格范围。以洗碗机场景为例（sigma≈1118），价格重要性被系统性放大 ~1118 倍，归一化后价格通常占据 >95% 权重，其他所有属性的重要性被压缩至接近零。

- ✅ **已修复** **[P0-PP-002]** `wtp.py:25` — **WTP 计算默认 `price_std=1.0` 导致金额单位错误**。所有 WTP 值以标准化价格单位输出（而非 CNY）。实际 WTP 应为 800 CNY 的属性，输出约为 0.72（无意义的标准差单位）。

- ✅ **已修复** **[P0-PP-003]** `wtp.py:96-117` — **分类属性 WTP 计算存在数学概念错误**。代码将 effects coding 的单个参数 `beta_{attr.id}_{i}` 当作"从基线到水平 i 的效用差"来计算 WTP。但在 effects coding 下，单个 beta 是水平效用值本身，水平间差值 = `beta_j - beta_i`。计算出的 WTP 在数学上无正确经济学含义。

#### P1-MAJOR（6项）

- 连续属性硬编码范围 `10.0`（魔法数字，无文档依据）
- `_recover_level_utilities()` 缺失参数静默返回 0（可能导致错误的水平效用恢复）
- `total_range == 0` 时重要性设为全零（和为 0 而非 1，违反归一化约定）
- 分段比较的多重 t-test 未做 Bonferroni/Holm 校正（12 次独立检验的误报率约 46%）
- WTP 异常值修剪后均值估计有偏
- Hotelling's T² 协方差矩阵接近奇异时未检查条件数

## 第二部分：跨层断裂分析

最严重的系统性问题是**价格标准化量纲断裂**：

```
effects_encode_price()          importance.py              wtp.py
  z-score: (p-μ)/σ     →     abs(beta_std)*range     →    -beta_feat/beta_price*1.0
  sigma≈1118                   缺除以sigma                  缺除以sigma
                               (价格放大1118倍)              (WTP缩水1118倍)
```

**根因**：预处理层的设计决策（价格标准化利于MCMC收敛）未以数据契约形式传递到后处理层。`price_std` 未从 Attribute 自动计算并注入到 `compute_importance()` 和 `WTPCalculator`。

**影响链**：
1. 属性重要性排序 → 价格永远第一（>95%），其他属性排序无意义
2. WTP 数值 → 无法用于定价决策（不是 CNY 单位）
3. 看板展示 → 基于错误数据的柱状图和饼图
4. LLM 解读 → 基于错误数据的业务洞察
