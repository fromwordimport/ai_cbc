## 一、公平性红线

### 1.1 绝对禁止的关联

以下关联在虚拟消费者的任何生成、模拟、分析环节中**绝对禁止**：

| 受保护属性 | 禁止关联的偏好/行为 | 违规示例 |
|-----------|-------------------|---------|
| **性别** | 产品属性偏好、价格敏感度、决策风格 | "女性更看重外观""男性只看参数" |
| **民族** | 消费习惯、品牌偏好、价格敏感度 | "某民族更节俭" |
| **地域/城市层级** | 品牌偏好、渠道选择、品味判断 | "小镇青年只用拼多多" |
| **年龄** | 技术接受度、学习能力、消费档次 | "老年人不会用手机" |
| **职业** | 消费档次、品味判断、决策理性程度 | "蓝领只看价格" |
| **婚姻状况** | 消费动机、易受影响程度 | "宝妈容易被种草" |

### 1.2 原则声明

```
虚拟消费者的偏好只能由以下因素解释：
✅ 个人经历与历史
✅ 价值观与生活方式
✅ 具体情境与情绪状态
✅ 经济能力与预算约束

虚拟消费者的偏好不得由以下因素解释：
❌ 性别刻板印象
❌ 民族/地域标签
❌ 年龄偏见
❌ 职业歧视
❌ 任何群体层面的简化归因
```

## 二、六条公平性硬规则

### 规则1：性别-偏好解耦（强制执行）

**规则内容**：虚拟消费者的产品偏好、价格敏感度、决策风格不得与性别标签存在统计显著关联。

**执行方式**：
- Prompt注入：在消费者生成指令中附加性别解耦约束
- 自动检测：批次完成后运行卡方检验（性别 vs 各属性偏好）
- 通过标准：p值 ≥ 0.05

**违规处置**：
- 检测到违规画像 → 拒绝入库，重新生成
- 连续3个批次违规 → 暂停生成，审查Prompt模板

### 规则2：民族-行为中性（强制执行）

**规则内容**：民族标签不得作为消费行为、价格敏感度或品牌偏好的解释变量。

**执行方式**：
- Prompt注入：明确禁止将民族与消费行为关联
- 自动检测：卡方检验（民族 vs 消费行为分类）
- 通过标准：p值 ≥ 0.05

### 规则3：地域-品牌解耦（强制执行）

**规则内容**：现居城市层级不得系统性关联品牌偏好或购买渠道选择。

**执行方式**：
- Prompt注入：禁止地域刻板印象叙事
- 自动检测：KS检验（城市层级 vs 品牌偏好得分）
- 通过标准：p值 ≥ 0.05

### 规则4：年龄-技术中性（强制执行）

**规则内容**：年龄不得系统性关联技术产品的接受度或学习能力。

**执行方式**：
- Prompt注入：技术能力只能由个人兴趣和职业需求解释
- 自动检测：KS检验（年龄 vs 技术接受度评分）
- 通过标准：p值 ≥ 0.05

### 规则5：职业-消费档次解耦（建议执行）

**规则内容**：职业标签不得系统性关联消费档次或品味判断。

**执行方式**：
- Prompt注入：消费选择基于个人价值观和经济能力
- 自动检测：ANOVA（职业类别 vs 消费档次评分）
- 通过标准：p值 ≥ 0.05

### 规则6：多样性保障（批次级强制）

**规则内容**：每个批次的虚拟消费者必须在关键维度上保持多样性。

**执行标准**：

| 维度 | 最小覆盖 | 均衡要求 |
|------|---------|---------|
| 年龄 | ≥ 4个年龄段 | 熵值 ≥ 0.7 |
| 性别 | 男女均有 | 比例 40%-60% |
| 收入 | ≥ 3个层级 | 熵值 ≥ 0.6 |
| 城市 | ≥ 3个城市层级 | — |
| 职业 | ≥ 5个类别 | 熵值 ≥ 0.5 |
| 价值观 | ≥ 3种核心 | — |

**违规处置**：批次未通过多样性检查 → 对不足维度进行加权重采样

## 二之一、硬规则执行层级统一（v1.1 新增）

所有受保护属性的公平性规则统一按以下执行层级实施，消除规则间强度差异：

| 规则 | 执行动作 | 触发条件 | 责任角色 |
|------|---------|---------|---------|
| 规则1 性别-偏好解耦 | reject_and_regenerate | 卡方 p < 0.05 或 Cramér's V ≥ 0.2 | 小伦 |
| 规则2 民族-行为中性 | reject_and_regenerate | 卡方 p < 0.05 或 Cramér's V ≥ 0.2 | 小伦 |
| 规则3 地域-品牌解耦 | reject_and_regenerate | 卡方 p < 0.05 或 Cramér's V ≥ 0.2 | 小伦 |
| 规则4 年龄-技术中性 | reject_and_regenerate | 卡方 p < 0.05 或 Cramér's V ≥ 0.2 | 小伦 |
| 规则5 职业-消费档次解耦 | reject_and_regenerate | 卡方 p < 0.05 或 Cramér's V ≥ 0.2 | 小伦 |
| 规则6 多样性保障 | reject_and_regenerate | 任一维度未达最小覆盖或熵值 | 小伦 |

**执行层级说明**：
- `reject_and_regenerate`：问题画像拒绝入库，触发重新生成；同一批次累计 ≥3 个画像被拒或统计检验未通过时，整批次暂停并通知小示审查 Prompt
- 所有规则不再区分"强制执行"与"建议执行"，统一为硬规则
- 统计判断以 **效应量优先**：Cramér's V ≥ 0.2 即触发处置，无论 p 值是否显著；p < 0.05 但 Cramér's V < 0.2 时记为 MEDIUM 警告，不触发 reject

## 三、偏见审计检查项

### 3.1 每批次必做检查

```markdown
□ 检查1：刻板印象模式扫描
   方法：关键词匹配 + Embedding语义相似度
   阈值：相似度 < 0.7
   通过：无CRITICAL级别命中

□ 检查2：统计分布检验
   连续变量（年龄、收入）：KS检验 vs 真实人口基准，D统计量 < 0.15
   分类变量（性别、城市层级、职业、民族）：卡方检验 vs 真实人口基准
   小样本修正：期望频数 < 5 的单元格占比 > 20% 时，改用 Fisher 精确检验
   通过：p值 ≥ 0.05

□ 检查3：偏好-属性关联分析
   方法：计算Cramér's V
   通过：Cramér's V < 0.2（弱关联或无关联）
   注：v1.0 阈值 0.3 偏高，v1.1 收紧至 0.2

□ 检查4：叙事简化度检测
   方法：互信息计算
   通过：人物小传与标签描述互信息 < 0.3（自然语言字段）或 < 0.5（结构化标签）

□ 检查5：多样性指数
   方法：Shannon熵
   通过：各维度熵值 ≥ 最小阈值
```

### 3.2 审计频率

| 审计类型 | 频率 | 样本 | 自动化 |
|---------|------|------|--------|
| 批次级偏见扫描 | 每批次 | 全量 | ✅ 自动 |
| 日报汇总 | 每日 | 当日全部 | ⚠️ 自动+人工复核 |
| 周度深度审计 | 每周 | 随机20% | ⚠️ 自动+人工复核 |
| 月度合规报告 | 每月 | 全量 | ⚠️ 自动+人工复核 |
| 新Prompt上线前 | 每次 | 测试批次100人 | ⚠️ 自动+人工复核 |

## 四、偏见超标处置流程

```
偏见审计完成
   │
   ▼
评估风险等级
   │
   ├── LOW（p ≥ 0.1）
   │   → 记录偏差，下批次调整采样权重
   │   → 批次通过
   │
   ├── MEDIUM（0.05 ≤ p < 0.1）
   │   → 标记问题画像
   │   → 通知小示优化Prompt
   │   → 批次通过（带警告）
   │
   ├── HIGH（0.01 ≤ p < 0.05）
   │   → 标记问题批次
   │   → 问题画像需重新生成或修正
   │   → 小伦审核修正后结果
   │   → 通过后方可入库
   │
   └── CRITICAL（p < 0.01 或 刻板印象 > 5%）
       → 🛑 小伦行使一票否决权
       → 暂停消费者生成流程
       → 全面审查Prompt模板
       → 修复后重新审计
       → 通过后方可恢复
```

### A.3 基准数据使用规则

1. **目标人群切片优先**：洗碗机场景目标人群为 18-35 岁一线城市居民，基准应使用该切片而非全国人口
2. **性别偏差声明**：研究设计（如"宝妈"细分）可能导致性别比例偏离人口基准，需在审计报告中单独声明并记录业务理由
3. **收入归一化**：CFPS 收入数据需按研究定义的收入层级重新分箱，保持与研究设计一致
4. **基准版本冻结**：阶段2启动前锁定基准数据版本，阶段内不得更换，以保证审计可比性
5. **基准缺失处理**：若某维度无公开基准，采用"研究设计分布"作为替代基准，并在审计报告中标注为 `design_baseline`

## 附录B：刻板印象模式库（v1.1 扩展至24个模式）

模式库用于关键词匹配和语义相似度扫描。所有模式按受保护属性分类，风险等级分为 HIGH（需修正）和 CRITICAL（一票否决）。完整模式清单见 `docs/公平性Prompt注入与审计模板.md` 第1.4节，此处为规范级引用。

### B.2 使用规则

1. **关键词匹配**：命中任意关键词即触发初筛，需结合上下文语义判断
2. **语义相似度**：使用 Embedding cosine similarity，阈值 **0.7**，超过即标记
3. **风险等级处置**：
   - CRITICAL：单个画像命中即 `reject_and_regenerate`
   - HIGH：累计同一批次 ≥3 个画像命中同一模式时，整批次暂停
4. **模式更新**：小伦每月 review 一次，新增模式需经小P审批
5. **误报申诉**：小示可向小伦提交申诉，经人工复核后可加入白名单

### B.3 与Prompt注入的联动

刻板印象模式库必须同步嵌入：
- 系统指令层的"绝对禁止"示例
- 任务指令层的具体规则反例
- Few-Shot示例中的反面示例
- 消费者生成自检Prompt的检查清单

审计管线中的 `STEREOTYPE_PATTERNS` 常量应以本附录为权威来源。
