## 一、伦理原则与公平性框架

### 1.1 核心伦理原则

```
┌─────────────────────────────────────────────────────────────┐
│                    虚拟消费者伦理四原则                        │
├─────────────────────────────────────────────────────────────┤
│  1. 非歧视原则（Non-Discrimination）                         │
│     虚拟消费者的偏好结构不得与受保护属性系统性相关             │
│                                                             │
│  2. 代表性原则（Representativeness）                         │
│     合成样本应在关键维度上反映真实人口的分布特征               │
│                                                             │
│  3. 透明性原则（Transparency）                               │
│     每个虚拟消费者的生成逻辑、决策过程必须可追溯、可审计       │
│                                                             │
│  4. 无害原则（Do No Harm）                                   │
│     虚拟消费者不得被用于放大刻板印象、歧视性营销或有害决策      │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 受保护属性清单

以下属性的任何标签不得成为偏好的系统性预测因子：

| 类别 | 受保护属性 | 风险示例 |
|------|-----------|---------|
| 人口统计 | 性别、民族、籍贯 | "女性只看颜值"、"某民族节俭" |
| 社会经济 | 职业、教育程度 | "蓝领工人只看价格"、"高学历者理性" |
| 家庭结构 | 婚姻状况、子女数量 | "宝妈容易被种草"、"单身青年冲动消费" |
| 地域 | 现居城市层级 | "小镇青年土味"、"一线城市精英" |
| 年龄 | 年龄段 | "老年人不会用手机"、"Z世代月光" |

### 1.3 公平性定义（数学化）

**人口统计平等（Demographic Parity）**：
对于任意受保护属性 $A$ 和偏好属性 $Y$：

$$P(Y = y | A = a) \approx P(Y = y | A = a') \quad \forall a, a'$$

即：不同群体的偏好分布应相似。

**机会均等化（Equalized Opportunity）**：
对于虚拟消费者的决策结果 $\hat{Y}$ 和真实偏好 $Y$：

$$P(\hat{Y} = 1 | Y = 1, A = a) \approx P(\hat{Y} = 1 | Y = 1, A = a')$$

即：在具有相同真实偏好的情况下，不同群体的决策概率应相等。

### 3.2 统计检验方法详解

#### 3.2.1 Kolmogorov-Smirnov检验（KS检验）

**用途**：比较合成样本与真实基准在连续变量上的分布差异。

**应用场景**：
- 年龄分布是否偏离真实人口结构
- 收入分布是否过度集中或分散
- 消费频次分布是否异常

**判定规则**：
```python
def ks_test_audit(synthetic_data, benchmark_data, variable, threshold=0.05):
    from scipy import stats
    
    statistic, p_value = stats.ks_2samp(
        synthetic_data[variable], 
        benchmark_data[variable]
    )
    
    return {
        "variable": variable,
        "ks_statistic": statistic,
        "p_value": p_value,
        "passed": p_value >= threshold,
        "severity": "CRITICAL" if p_value < 0.01 else "HIGH" if p_value < 0.05 else "LOW"
    }
```

**告警阈值**：
| p值范围 | 风险等级 | 处置 |
|---------|---------|------|
| p < 0.01 | CRITICAL | 暂停生成，全面审查 |
| 0.01 ≤ p < 0.05 | HIGH | 标记问题批次，要求修正 |
| 0.05 ≤ p < 0.1 | MEDIUM | 记录偏差，下批次调整采样权重 |
| p ≥ 0.1 | LOW | 通过 |

#### 3.2.2 卡方独立性检验

**用途**：检验分类变量之间是否存在不应有的关联。

**应用场景**：
- 性别 vs 产品属性偏好
- 年龄组 vs 品牌选择
- 地域 vs 价格敏感度分类

**判定规则**：
```python
def chi_square_audit(data, var_a, var_b, threshold=0.05):
    from scipy import stats
    import pandas as pd
    
    contingency_table = pd.crosstab(data[var_a], data[var_b])
    chi2, p_value, dof, expected = stats.chi2_contingency(contingency_table)
    
    # Cramér's V 效应量
    n = contingency_table.sum().sum()
    cramers_v = np.sqrt(chi2 / (n * (min(contingency_table.shape) - 1)))
    
    return {
        "variables": f"{var_a} vs {var_b}",
        "chi2": chi2,
        "p_value": p_value,
        "cramers_v": cramers_v,  # 效应量，0-1，越大关联越强
        "passed": p_value >= threshold and cramers_v < 0.3,
        "severity": "CRITICAL" if cramers_v > 0.5 else "HIGH" if cramers_v > 0.3 else "LOW"
    }
```

#### 3.2.3 多样性熵指数

**用途**：量化批次样本在各维度上的多样性。

```python
def diversity_entropy(data, dimension):
    from scipy.stats import entropy
    
    # 计算各水平的频率
    frequencies = data[dimension].value_counts(normalize=True)
    
    # Shannon熵
    H = entropy(frequencies, base=2)
    
    # 最大可能熵（均匀分布时）
    max_H = np.log2(len(frequencies))
    
    # 归一化熵（0-1）
    normalized_H = H / max_H if max_H > 0 else 0
    
    return {
        "dimension": dimension,
        "entropy": H,
        "max_entropy": max_H,
        "normalized_entropy": normalized_H,
        "passed": normalized_H >= 0.6,
        "threshold": 0.6
    }
```

## 六、审计频率与触发条件

### 6.1 定期审计

| 审计类型 | 频率 | 样本量 | 负责人 |
|---------|------|--------|--------|
| 批次级偏见扫描 | 每批次 | 全量 | 自动化管线 |
| 日报汇总 | 每日 | 当日全部批次 | 小伦（复核） |
| 周度深度审计 | 每周 | 随机抽取20% | 小伦 |
| 月度合规报告 | 每月 | 全量 | 小伦 + 小验 |

### 6.2 触发审计的事件

- 新Prompt模板上线前
- 模型版本切换后
- 业务方反馈"结果不可理喻"时
- 成本熔断触发后（检查是否为异常行为）
- 任何团队成员提出伦理疑虑时

### 6.3 一票否决权行使条件

小伦在以下情况下有权暂停消费者生成流程：

1. **批次级CRITICAL风险**：刻板印象命中数 > 批次量的5%
2. **统计检验系统性失败**：同一维度连续3个批次未通过KS/卡方检验
3. **新偏见模式发现**：发现未被模式库覆盖的系统性偏见
4. **合规要求变更**：法律法规或公司政策更新，现有流程不满足新要求
