# AI_CBC 成本管控方案

> **版本**：v1.0
> **定位**：基于现有代码实现（ProfileGenerator、LLMChoiceSimulator、LLMClient），建立精确的单次CBC全流程成本模型与管控方案
> **负责人**：小控（成本与资源管控专员）
> **自动执行**：成本熔断触发后无需人工审批，自动降级或暂停
> **基准依据**：`src/aicbc/config/settings.py` CostFuseSettings、`_COST_PER_1K` 定价表

## 一、代码级成本模型

### 1.1 定价基准（来自 `src/aicbc/llm/client.py`）

| 提供商 | 模型 | Input ($/1K tokens) | Output ($/1K tokens) | 用途 |
|--------|------|---------------------|----------------------|------|
| Anthropic | claude-sonnet-4-6 | $3.00 | $15.00 | 画像生成、模拟决策（默认） |
| Anthropic | claude-haiku-4-5 | $0.25 | $1.25 | 审校、解读、降级备用 |
| Anthropic | claude-opus-4-6 | $15.00 | $75.00 | 紧急/关键任务（极少使用） |
| OpenAI | gpt-4o | $5.00 | $15.00 | 备用模型 |
| OpenAI | gpt-4o-mini | $0.15 | $0.60 | 轻量备用 |

汇率：1 USD = 7.2 CNY（代码中硬编码于 `llm_choice_simulator.py:33` 和 `profile_generator.py:243`）

### 1.2 ProfileGenerator 成本测算（`src/aicbc/generators/profile_generator.py`）

**调用结构**：
- Layer 1-4 生成：4 次独立 LLM 调用（逐层构建，每层接收前层上下文）
- Auxiliary 生成：1 次 LLM 调用（语言样本 + 洗碗机情境）
- **总计：5 次 LLM 调用 / 画像**

**Token 估算**（基于代码中 Prompt 结构分析）：

| 调用 | 输入 Token | 输出 Token | 模型 | 单次成本 |
|------|-----------|-----------|------|---------|
| Layer 1 | ~2,500（模板+种子+Schema） | ~800（JSON） | Sonnet 4.6 | ¥0.108 |
| Layer 2 | ~3,500（模板+种子+Layer1+Schema） | ~600（JSON） | Sonnet 4.6 | ¥0.122 |
| Layer 3 | ~4,500（模板+种子+Layer1-2+Schema） | ~1,200（JSON含叙事） | Sonnet 4.6 | ¥0.227 |
| Layer 4 | ~5,500（模板+种子+Layer1-3+Schema） | ~800（JSON） | Sonnet 4.6 | ¥0.206 |
| Auxiliary | ~6,000（完整画像摘要） | ~600（语言样本+情境） | Sonnet 4.6 | ¥0.195 |
| **单画像小计** | | | | **¥0.858** |

> 注：以上为 Sonnet 4.6 全链路成本。若启用 Haiku 降级（如审校环节），成本可降约 60%。

### 1.3 LLMChoiceSimulator 成本测算（`src/aicbc/core/simulation/llm_choice_simulator.py`）

**调用结构**：
- 1 次系统 Prompt 构建（无 LLM 调用，纯文本拼接）
- 每选择集 1 次 LLM 调用：`system_prompt + user_prompt`
- **总计：N 选择集 次 LLM 调用 / 消费者**

**Token 估算**（基于代码中 `_build_system_prompt` 和 `_build_choice_prompt`）：

| 组件 | Token 估算 | 说明 |
|------|-----------|------|
| System Prompt（画像注入） | ~1,200 | 包含 Layer1-4 全量信息 |
| User Prompt（单选择集） | ~400 | 3 个选项 × 6 属性 |
| LLM 输出（JSON） | ~200 | chosen_alt_index + reasoning + confidence |

| 模型 | 单次选择集成本 | 12 题成本 | 备注 |
|------|---------------|----------|------|
| Sonnet 4.6 | ¥0.048 | **¥0.576** | 默认模型 |
| Haiku 4.5 | ¥0.004 | **¥0.048** | 降级模型 |

### 1.4 单次完整 CBC 流程成本汇总

**标准配置**（来自 `settings.py` StudySettings）：
- 样本量：150 人
- 选择集：12 题
- 每集选项：3 个 + None

| 环节 | 调用次数 | 单次成本 | 小计 | 占比 |

## 二、成本 KPI 量化

### 2.1 核心成本指标

| KPI | 目标值 | 熔断阈值 | 测量方式 | 代码位置 |
|-----|--------|---------|---------|---------|
| **单次标准研究成本** | ≤ ¥500 | ≥ ¥800 告警，≥ ¥1,000 熔断 | 任务完成后统计 | `settings.py: CostFuseSettings.single_study_cny` |
| **消费者平均成本** | ≤ ¥4/人 | ≥ ¥6/人 告警 | 总费用 / 消费者数 | 计算指标 |
| **单次选择集成本** | ≤ ¥0.05 | ≥ ¥0.08 告警 | 模拟费用 / 选择集数 | 计算指标 |
| **预算偏差率** | ≤ 10% | ≥ 20% 审计 | \|实际-预算\| / 预算 | 计算指标 |
| **成本节约率** | ≥ 50%（vs 传统调研） | — | (传统成本 - AI_CBC成本) / 传统成本 | 季度对比 |

### 2.2 分级熔断阈值（与代码配置对齐）

```python
# src/aicbc/config/settings.py 中的配置
class CostFuseSettings(BaseSettings):
```

## 四、计费追踪嵌入

### 4.1 当前代码中的成本追踪

`LLMClient._estimate_cost()`（`client.py:44-50`）已实现：
```python
def _estimate_cost(provider, model, prompt_tokens, completion_tokens) -> float:
    pricing = _COST_PER_1K.get(provider.value, {}).get(model)
    input_price, output_price = pricing
    return (prompt_tokens * input_price + completion_tokens * output_price) / 1000.0
```

`LLMResponse` 包含 `estimated_cost_usd` 字段，每次调用自动计算。

`ProfileGenerator` 在 `generation_metadata.cost_cny` 中记录单画像成本。

`LLMChoiceSimulator` 在 `PersonaResponse.cost_cny` 中记录单消费者模拟成本。

### 4.2 建议增强：成本熔断检查点

在以下位置嵌入熔断检查：

```python
# 建议：在 LLMClient.generate() 中增加熔断检查
def generate(self, messages, model=None, ...):
    # 新增：调用前检查成本熔断状态
    fuse_status = cost_tracker.check_fuse_status()
    if fuse_status == "FUSE":
        raise CostFuseError("成本熔断已触发，暂停新调用")
    if fuse_status == "DEGRADE":
        model = settings.cost_fuse.degrade_model  # 强制降级

    # 原有逻辑...
    response = self._call_anthropic(...)

    # 新增：调用后记录并检查
    cost_tracker.record_call(response)
    cost_tracker.check_and_notify()

    return response
```
