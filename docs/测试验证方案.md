## 二、模型可恢复性测试

### 2.1 测试原理

```
设定已知效用参数 → 生成合成选择数据 → 用分析系统建模 → 对比估计值与真实值
```

### 2.2 测试流程

```python
class ParameterRecoveryTest:
    """参数恢复性测试"""
    
    def run_test(self, n_resp: int = 200, n_tasks: int = 12) -> RecoveryReport:
        """
        执行完整的参数恢复测试
        """
        
        # Step 1: 设定已知参数（"真实世界"）
        true_params = {
            "price": -0.003,           # 价格系数为负
            "capacity_6": 0.5,         # 6套基准
            "capacity_10": 1.2,        # 10套效用
            "capacity_13": 1.8,        # 13套效用
            "installation_台式": 0.3,
            "installation_嵌入式": 0.8,
            "features_基础": 0.0,
            "features_智能": 0.6,
            "features_全能": 1.5,
            "brand_美的": 0.2,
            "brand_西门子": 1.0,
            "brand_方太": 0.8,
            "brand_小米": 0.5,
            "energy_二级": 0.0,
            "energy_一级": 0.3,
            "energy_超一级": 0.5,
        }
        
        # Step 2: 生成合成选择数据
        synthetic_data = self._generate_synthetic_choices(
            true_params, n_resp, n_tasks
        )
        
        # Step 3: 用分析系统建模
        estimated_params = self._run_analysis_pipeline(synthetic_data)
        
        # Step 4: 计算恢复误差
        report = self._compute_recovery_error(true_params, estimated_params)
        
        # Step 5: 综合判定
        report["overall_passed"] = (
            report["mape"] <= 0.15 and
            report["ranking_accuracy"] >= 0.8 and
            report["price_sign_correct"] == 1.0
        )
        
        return report
```

### 2.3 通过标准

| 指标 | 目标值 | 可接受值 | 说明 |
|------|--------|---------|------|
| 效用参数MAPE | ≤ 10% | ≤ 15% | 估计值与真实值的平均绝对百分比误差 |
| 属性重要性排序一致性 | 100% | ≥ 80% | TOP3属性排序是否正确 |
| 价格系数方向正确率 | 100% | 100% | 价格系数必须为负 |
| 市场份额预测R² | ≥ 0.85 | ≥ 0.70 | 预测份额与真实选择的一致性 |

### 3.2 人工盲测流程

```markdown
# 盲测执行手册

## 准备阶段
1. 收集真实消费者画像（来自过往调研，n=20）
2. 生成虚拟消费者画像（n=20，与真实画像混排）
3. 匿名化处理（移除可识别信息，随机编号001-040）
4. 准备评分表（七维度×0-2分）

## 执行阶段
1. 招募5-8位消费者研究员作为评估者
2. 每位评估者独立评估全部40个画像
3. 按七维度逐一评分
4. 附加问题："你认为这是真实消费者还是AI生成的？"

## 分析阶段
1. 统计真实 vs 虚拟在七维度上的得分差异
2. 计算虚拟画像被误判为真实的比例
3. 分析哪些维度是"暴露"AI身份的关键
4. 输出盲测报告

## 通过标准
- 虚拟画像平均评分 ≥ 真实画像的75%
- 虚拟画像被误判率 ≥ 30%
- 七维度评分无显著差异（t检验p > 0.05）
```

        测试完整研究流程
        """
        # 1. 创建研究
        study = self.create_study(
            product="洗碗机",

        # 2. 生成画像

        # 3. 生成问卷

        # 7. 验证成本

## 五、Agent鲁棒性测试

### 5.1 异常输入测试

| # | 异常类型 | 测试输入 | 预期行为 |
|---|---------|---------|---------|
| 1 | 空输入 | {} | 返回错误，不崩溃 |
| 2 | 超大输入 | 10万字描述 | 截断或拒绝，不崩溃 |
| 3 | 特殊字符 | Emoji/控制字符 | 过滤或转义 |
| 4 | 格式错误 | 非法JSON | 返回格式错误 |
| 5 | 并发冲突 | 同一画像同时多次请求 | 串行处理或乐观锁 |
| 6 | 超时 | 模型响应超过120秒 | 返回超时错误，释放资源 |
| 7 | 模型不可用 | API 503 | 切换备用模型或排队 |

## 七、测试架构治理

详见 [`docs/superpowers/specs/2026-06-18-测试架构治理设计.md`](superpowers/specs/2026-06-18-测试架构治理设计.md)。

关键原则：

- 测试按 `unit` / `integration` / `e2e` / `redteam` / `performance` 分层组织。
- CI 执行 `unit + integration` 快速测试，以及 `security` 红队快速测试。
- Nightly 执行全量回归（除 performance 外），并单独运行性能基准。
- 每季度审查一次测试用例，防止"杀虫剂"效应。
- 新增、修改、删除测试时参考 [`tests/REVIEW_CHECKLIST.md`](../tests/REVIEW_CHECKLIST.md)。

## 七、测试执行计划

### 7.1 测试阶段与负责人

| 阶段 | 测试类型 | 负责人 | 执行时机 | 时长 |

| 集成时 | 集成测试 | 小测+小端 | 每次模块集成 | 4小时 |
