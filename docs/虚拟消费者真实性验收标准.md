### 1.2 评分标准

每个维度按 **0-2分** 评分：

| 分数 | 等级 | 描述 |
|------|------|------|
| **0分** | 不合格 | 明显违反该维度的核心原则，不可接受 |
| **1分** | 部分合格 | 基本符合，但存在可改进的瑕疵 |
| **2分** | 优秀 | 完全符合，甚至超出预期 |

**总分计算**：满分 14 分（7维度 × 2分）

| 总分范围 | 评级 | 处置 |
|---------|------|------|
| 12-14分 | 🟢 优秀 | 可直接入库使用 |
| 9-11分 | 🟡 合格 | 可通过，建议优化 |
| 7-8分 | 🟠 需审查 | 需人工复核，决定是否可用 |
| < 7分 | 🔴 不合格 | 拒绝入库，需重新生成或修正 |

## 二、各维度详细验收标准

### 2.1 维度1：内在一致性（自动检测）

**核心原则**：画像的内在逻辑必须自洽，矛盾组合必须有叙事解释。

**验收检查项**：

| # | 检查项 | 通过标准 | 检测方式 |
|---|--------|---------|---------|
| IC-001 | 矛盾标签解释 | 所有 tension_combination 必须附 ≥30字解释 | 规则引擎 |
| IC-002 | 收入-消费匹配 | 低收入高消费必须有不寻常的合理解释 | 规则引擎 |
| IC-003 | 人生阶段-焦虑匹配 | 退休+育儿焦虑需特殊说明 | 规则引擎 |
| IC-004 | 品牌忠诚度一致性 | 深度忠诚者不应有品牌转换原因 | 规则引擎 |
| IC-005 | 时间线因果链 | present行为至少有一条可由past解释 | LLM辅助 |

**自动评分规则**：
```python
def score_internal_consistency(persona: dict) -> int:
    checks = [
        check_tension_explanation(persona),      # IC-001
        check_income_spending_match(persona),    # IC-002
        check_life_stage_anxiety(persona),       # IC-003
        check_brand_loyalty(persona),            # IC-004
        check_timeline_causality(persona),       # IC-005
    ]
    
    passed = sum(checks)
    
    if passed == 5: return 2      # 全部通过 → 2分
    elif passed >= 3: return 1    # 大部分通过 → 1分
    else: return 0                 # 多个失败 → 0分
```

### 2.2 维度2：情境敏感性（LLM辅助评分）

**核心原则**：同一消费者在不同情境下应有差异化的、非模板化的反应。

**验收检查项**：

| # | 检查项 | 通过标准 | 检测方式 |
|---|--------|---------|---------|
| CS-001 | 场景反应差异化 | 至少5个场景反应不重复 | 规则引擎 |
| CS-002 | 情境驱动行为 | 情境变化导致行为变化 | LLM辅助 |
| CS-003 | 非模板化 | 场景反应不能直接从标签推导 | LLM辅助 |

**LLM评分Prompt模板**：
```markdown

{persona_content}

2. 场景反应是否与画像标签一致但非直接推导？
```

## 三、统计检验方法

### 3.1 样本覆盖度检验

**目的**：确保生成的消费者样本在关键维度上覆盖充分，无系统性缺失。

**检验维度**：

| 维度 | 最小覆盖要求 | 检验方法 |
|------|------------|---------|
| 年龄 | 至少覆盖4个年龄段 | 频次统计 |
| 性别 | 男女比例 40%-60% | 卡方检验 |
| 收入 | 至少覆盖3个收入层级 | 频次统计 |
| 城市 | 至少覆盖3个城市层级 | 频次统计 |
| 职业 | 至少覆盖5个职业类别 | 频次统计 |
| 价值观 | 至少覆盖3种核心价值观 | 频次统计 |

**通过标准**：
```python
def check_coverage(batch: list[Persona]) -> dict:
    results = {}
    
    # 年龄覆盖
    age_groups = Counter(p.age_group for p in batch)
    results["age"] = {
        "coverage": len(age_groups),
        "required": 4,
        "passed": len(age_groups) >= 4
    }
    
    # 性别均衡
    gender_dist = Counter(p.gender for p in batch)
    male_ratio = gender_dist.get("男", 0) / len(batch)
    results["gender"] = {
        "male_ratio": male_ratio,
        "passed": 0.4 <= male_ratio <= 0.6
    }
    
    # 收入覆盖
    income_groups = Counter(p.income_tier for p in batch)
    results["income"] = {
        "coverage": len(income_groups),
        "required": 3,
        "passed": len(income_groups) >= 3
    }
    
    # 综合判定
    all_passed = all(r["passed"] for r in results.values())
    
    return {
        "details": results,
        "overall_passed": all_passed
    }
```

### 3.2 分布检验（与真实基准对比）

**目的**：验证合成样本的分布是否与真实人口分布一致（或是否在可接受偏差范围内）。

**检验流程**：

```python
class DistributionValidator:
    """分布验证器"""
    
    def validate_batch(
        self,
        synthetic_batch: pd.DataFrame,
        benchmark: pd.DataFrame,
        dimensions: list[str]
    ) -> dict:
        """验证批次样本的分布"""
        
        results = {}
        
        for dim in dimensions:
            if dim in ["age", "income", "price_sensitivity_score"]:
                # 连续/有序变量：KS检验
                stat, p_value = stats.ks_2samp(
                    synthetic_batch[dim].dropna(),
                    benchmark[dim].dropna()
                )
                results[dim] = {
                    "test": "KS",
                    "statistic": stat,
                    "p_value": p_value,
                    "passed": p_value >= 0.05,
                    "severity": "HIGH" if p_value < 0.01 else "MEDIUM" if p_value < 0.05 else "LOW"
                }
            else:
                # 分类变量：卡方检验
                syn_counts = synthetic_batch[dim].value_counts()
                bench_counts = benchmark[dim].value_counts()
                
                # 对齐类别
                all_categories = set(syn_counts.index) | set(bench_counts.index)
                syn_aligned = [syn_counts.get(c, 0) for c in all_categories]
                bench_aligned = [bench_counts.get(c, 0) for c in all_categories]
                
                chi2, p_value = stats.chisquare(syn_aligned, bench_aligned)
                
                results[dim] = {
                    "test": "Chi-square",
                    "chi2": chi2,
                    "p_value": p_value,
                    "passed": p_value >= 0.05,
                    "severity": "HIGH" if p_value < 0.01 else "MEDIUM" if p_value < 0.05 else "LOW"
                }
        
        # 综合判定
        failed = [d for d, r in results.items() if not r["passed"]]
        critical = [d for d, r in results.items() if r.get("severity") == "HIGH"]
        
        return {
            "dimension_results": results,
            "failed_dimensions": failed,
            "critical_dimensions": critical,
            "overall_passed": len(critical) == 0 and len(failed) <= 1
        }
```

### 3.3 模型可恢复性测试

**目的**：验证统计模型能否从虚拟消费者的选择数据中准确恢复预设的偏好参数。

**测试流程**：

```
Step 1: 设定已知参数
  • 定义一组"真实"的效用参数（如：品牌偏好、价格敏感度）
  • 这些参数代表"真实消费者"的偏好结构
       ↓
Step 2: 生成合成选择数据
  • 使用已知参数，通过Logit模型生成选择数据
  • 模拟100-500个消费者的选择序列
       ↓
Step 3: 用分析系统建模
  • 将合成数据输入分析系统
  • 运行HB模型估计参数
       ↓
Step 4: 对比估计值与真实值
  • 计算参数恢复误差
  • 评估属性重要性排序一致性
       ↓
Step 5: 判定通过/不通过
```

**通过标准**：

| 指标 | 目标值 | 可接受范围 |
|------|--------|-----------|
| 效用参数恢复误差（MAPE） | ≤ 10% | ≤ 15% |
| 属性重要性排序一致性 | 100% | ≥ 80% |
| 价格系数方向正确率 | 100% | 100% |
| 市场份额预测R² | ≥ 0.85 | ≥ 0.70 |

```python
class ParameterRecoveryTest:
    """参数恢复性测试"""
    
    def run_test(
        self,
        true_params: dict,
        n_resp: int = 200,
        n_tasks: int = 12,
        n_alts: int = 3
    ) -> dict:
        """
        执行参数恢复测试
        
        Parameters:
            true_params: 真实效用参数
            n_resp: 消费者数量
            n_tasks: 每人选择集数
            n_alts: 每集选项数
        """
        
        # 1. 生成合成选择数据
        synthetic_data = self._generate_choices(true_params, n_resp, n_tasks, n_alts)
        
        # 2. 用分析系统建模
        estimated_params = self._run_hb_model(synthetic_data)
        
        # 3. 计算恢复误差
        errors = {}
        for attr, true_val in true_params.items():
            est_val = estimated_params.get(attr, 0)
            mape = abs(est_val - true_val) / abs(true_val) * 100 if true_val != 0 else 0
            errors[attr] = {
                "true": true_val,
                "estimated": est_val,
                "absolute_error": abs(est_val - true_val),
                "mape": mape,
                "passed": mape <= 15
            }
        
        # 4. 重要性排序一致性
        true_importance = self._compute_importance(true_params)
        est_importance = self._compute_importance(estimated_params)
        
        true_ranking = sorted(true_importance.keys(), key=lambda x: true_importance[x], reverse=True)
        est_ranking = sorted(est_importance.keys(), key=lambda x: est_importance[x], reverse=True)
        
        ranking_match = sum(1 for i, attr in enumerate(true_ranking) if est_ranking[i] == attr)
        ranking_accuracy = ranking_match / len(true_ranking)
        
        # 5. 综合判定
        all_passed = all(e["passed"] for e in errors.values()) and ranking_accuracy >= 0.8
        
        return {
            "parameter_recovery": errors,
            "ranking_accuracy": ranking_accuracy,
            "overall_passed": all_passed,
            "report": self._generate_report(errors, ranking_accuracy)
        }
```

## 四、人工盲测（金标准）

### 4.1 盲测设计

**目的**：通过人类专家的判断，验证虚拟消费者的真实感。

**设计原则**：
- 双盲：评估者不知道哪些是虚拟的，哪些是真实的
- 对照：混合真实消费者画像（来自过往调研）和虚拟消费者画像
- 多维度：按七维度模型逐一评估
- 统计显著性：样本量足够进行显著性检验

### 4.2 盲测流程

```
准备阶段
  │
  ├── 收集真实消费者画像（来自过往调研，n=20）
  ├── 生成虚拟消费者画像（n=20，与真实画像混排）
  ├── 匿名化处理（移除可识别信息）
  └── 随机编号（001-040）
       ↓
评估阶段
  │
  ├── 招募评估者（5-8位消费者研究员）
  ├── 每位评估者独立评估全部40个画像
  ├── 按七维度逐一评分（0-2分）
  └── 附加问题："你认为这是真实消费者还是AI生成的？"
       ↓
分析阶段
  │
  ├── 统计真实 vs 虚拟在七维度上的得分差异
  ├── 计算虚拟画像被误判为真实的比例
  ├── 分析哪些维度是"暴露"AI身份的关键
  └── 输出盲测报告
```

### 4.3 盲测通过标准

| 指标 | 目标值 | 可接受值 |
|------|--------|---------|
| 虚拟画像平均真实性评分 | ≥ 真实画像的90% | ≥ 真实画像的80% |
| 虚拟画像被误判为真实的比例 | ≥ 40% | ≥ 30% |
| 七维度评分无显著差异（t检验） | p > 0.05 | p > 0.01 |
| 无单一维度明显暴露AI身份 | 所有维度 p > 0.05 | — |
