# AI_CBC 数据字典

> **版本**：v1.0  
> **定位**：统一全链路数据实体、字段、类型与约束的权威参考  
> **维护者**：小端（后端/集成工程师）+ 小数（数据/建模科学家）  
> **适用范围**：消费者模拟系统、CBC问卷系统、数据分析系统、全链路集成接口

## 二、消费者画像 `PersonaProfile`

由 GEN Agent 生成，经 REV Agent 审校后的完整四层画像。

| 字段名 | 类型 | 必填 | 描述 | 示例 | 约束 |
|---|---|---|---|---|---|
| `persona_id` | string | 是 | 全局唯一画像标识 | `"persona-dw-001"` | 前缀 +  study_id + 序号 |
| `segment` | string | 是 | 所属细分群体 | `"精致白领"` | 必须在 `SegmentProfile` 中预定义 |
| `layer1_demographics` | object | 是 | 人口统计层 | 见下表 | 受保护属性仅作描述，不得作为默认偏好解释 |
| `layer2_behavior` | object | 是 | 消费行为层 | 见下表 | 必须与 Layer1 存在逻辑关联 |
| `layer3_psychology` | object | 是 | 心理动机层 | 见下表 | `tension_combination.narrative_explanation` ≥ 50 字 |
| `layer4_scenarios` | object | 是 | 情境叙事层 | 见下表 | 必须覆盖洗碗机购买场景 |
| `language_samples` | string[] | 是 | 代表性发言（3 条） | `["洗碗机...", "...", "..."]` | 单条 20–60 字 |
| `dishwasher_context` | object | 是 | 洗碗机购买情境 | 见下表 | 与画像整体逻辑一致 |
| `authenticity_score` | number | 否 | 真实性评分（0–14） | `11` | ≥ 9 为及格，≥ 12 为优秀 |
| `bias_audit_status` | string | 否 | 偏见审计状态 | `"PASSED"` / `"FAILED"` / `"PENDING"` | 默认为 `PENDING` |
| `generation_metadata` | object | 否 | 生成元信息 | `{model, version, seed, cost_cny}` | 用于追溯 |
| `created_at` | datetime | 是 | 创建时间 | `"2026-06-09T14:32:00Z"` | UTC |

### 2.1 `layer1_demographics` 子字段

| 字段名 | 类型 | 描述 | 示例 |
|---|---|---|---|
| `age` | string | 年龄段 | `"28岁"` |
| `gender` | string | 性别 | `"女"` |
| `city` | string | 城市层级 | `"新一线城市"` |
| `income` | string | 收入档位 | `"月收入15K-25K"` |
| `occupation` | string | 职业 | `"互联网运营"` |
| `education` | string | 学历 | `"本科"` |
| `marital_status` | string | 婚姻状态 | `"已婚无孩"` |
| `living_type` | string | 居住形态 | `"70㎡两居室，租房"` |

### 2.2 `layer2_behavior` 子字段

| 字段名 | 类型 | 描述 | 示例 |
|---|---|---|---|
| `price_sensitivity` | string | 价格敏感度 | `"中高敏感"` |
| `purchase_channels` | string[] | 常用购买渠道 | `["京东", "天猫", "线下苏宁"]` |
| `decision_style` | string | 决策风格 | `"理性比较型"` |
| `brand_loyalty` | string | 品牌忠诚度 | `"中等，重性价比"` |
| `information_source` | string[] | 信息来源 | `["小红书", "知乎", "朋友推荐"]` |

### 2.3 `layer3_psychology` 子字段

| 字段名 | 类型 | 描述 | 示例 |
|---|---|---|---|
| `core_values` | string[] | 核心价值观 | `["效率", "品质生活"]` |
| `core_anxieties` | string[] | 核心焦虑 | `["时间不够用", "家务分工矛盾"]` |
| `tension_combination` | object | 张力组合与解释 | `{labels, narrative_explanation}` |
| `secret_motivation` | string | 隐藏动机 | `"用科技产品证明自己的生活品味"` |
| `defense_mechanism` | string | 心理防御机制 | `"合理化：说服自己这是长期投资"` |

### 2.4 `layer4_scenarios` 子字段

| 字段名 | 类型 | 描述 | 示例 |
|---|---|---|---|
| `daily_routine` | string | 日常生活轨迹 | `"早9晚7，周末打扫..."` |
| `purchase_trigger` | string | 购买触发事件 | `"看到同事晒洗碗机，心动"` |
| `stress_response` | string | 压力下的反应 | `"焦虑时更容易冲动消费"` |
| `social_behavior` | string | 社交行为特征 | `"爱在社交媒体分享好物"` |

### 2.5 `dishwasher_context` 子字段

| 字段名 | 类型 | 描述 | 示例 |

## 三、研究定义 `CBCStudy`

一次完整的 CBC 研究配置，作为全链路编排的顶层聚合根。

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `study_id` | string | 是 | 研究唯一标识 | `"dishwasher-202506"` |
| `product_category` | string | 是 | 产品类别 | `"洗碗机"` |
| `research_goal` | string | 是 | 研究目标 | `"评估价格敏感度与品牌偏好"` |
| `target_segments` | string[] | 是 | 目标细分群体 | `["精致白领", "新手宝妈", "Z世代租客"]` |
| `attributes` | object[] | 是 | 属性/水平定义 | 见 `洗碗机CBC实验设计方案.md` |
| `design_parameters` | object | 是 | 实验设计参数 | `{n_choice_sets, n_alternatives, n_blocks}` |
| `sample_size` | integer | 是 | 计划样本量 | `150` |
| `cost_budget_cny` | number | 是 | 成本预算（元） | `500` |
| `status` | string | 是 | 研究状态 | `"INIT"` → `"COMPLETED"` |
| `created_at` | datetime | 是 | 创建时间 | `"2026-06-09T10:00:00Z"` |

## 四、CBC问卷 `CBCQuestionnaire`

由实验设计引擎生成的选择集集合。

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `questionnaire_id` | string | 是 | 问卷唯一标识 | `"q-dishwasher-202506"` |
| `study_id` | string | 是 | 所属研究 | `"dishwasher-202506"` |
| `attributes` | object[] | 是 | 属性定义（同 Study） | — |
| `choice_sets` | object[] | 是 | 选择集列表 | 见下 |
| `design_parameters` | object | 是 | D-efficient 设计参数 | `{algorithm, seed, n_trials}` |
| `d_efficiency` | number | 是 | D 效率值 | `0.92` | 目标 ≥ 0.85 |
| `prohibited_pairs` | object[] | 否 | 禁止组合 | `[{"price":"2999","brand":"西门子"}]` |
| `created_at` | datetime | 是 | 创建时间 | `"2026-06-09T11:00:00Z"` |

| 字段名 | 类型 | 描述 | 示例 |
|---|---|---|---|
| `choice_set_id` | integer | 题号 | `1` |
| `alternatives` | object[] | 选项列表 | 见下 |

| 字段名 | 类型 | 描述 | 示例 |

| `attributes` | object | 属性水平组合 | `{"price":3999, "capacity":"6套", ...}` |

## 五、标准交换数据集 `CBCRawDataset`

问卷系统输出、分析系统输入的统一格式。包含元数据与选择记录两部分。

### 5.1 `metadata` 字段

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `study_id` | string | 是 | 研究标识 | `"dishwasher-202506"` |
| `n_respondents` | integer | 是 | 受访者数量 | `150` |
| `n_choice_sets` | integer | 是 | 每人选择集数量 | `12` |
| `n_alternatives` | integer | 是 | 每集选项数（不含 None） | `3` |
| `attributes` | object[] | 是 | 属性定义 | 同 Study |

### 5.2 `choice_records` 元素字段

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `respondent_id` | string | 是 | 受访者/画像 ID | `"persona-dw-001"` |
| `respondent_index` | integer | 是 | 受访者索引（0-based） | `0` |
| `segment` | string | 是 | 所属细分群体 | `"精致白领"` |
| `choice_set_id` | integer | 是 | 选择集 ID | `1` |
| `choice_set_index` | integer | 是 | 选择集索引（0-based） | `0` |
| `alternatives` | object[] | 是 | 选项记录 | 见下 |
| `none_chosen` | boolean | 是 | 是否选择"都不选" | `false` |

### 5.3 `alternatives`（选择记录内）字段

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `alt_index` | integer | 是 | 选项索引 | `1` |
| `chosen` | boolean | 是 | 是否被选中 | `true` |
| `attributes` | object | 是 | 属性原始值 | `{"price":4999, "capacity":"10套", ...}` |

## 七、分析结果 `AnalysisResult`

分析引擎拟合模型后输出的标准化结果。

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `result_id` | string | 是 | 结果唯一标识 | `"ar-dishwasher-202506"` |
| `study_id` | string | 是 | 所属研究 | `"dishwasher-202506"` |
| `model_type` | string | 是 | 模型类型 | `"hb"` / `"mnl"` / `"latent_class"` |
| `convergence` | object | 条件 | 收敛诊断 | `{"rhat_max":1.04,"ess_min":1200,"converged":true}` |
| `individual_utilities` | object | 是 | 个体系数 | `{resp_id: {price:-0.003, capacity_6:0.5, ...}}` |
| `population_params` | object | 是 | 总体参数 | `{"mu":{...}, "sigma":{...}}` |
| `importance` | object | 是 | 属性重要性 | `{price:0.425, brand:0.238, ...}` |
| `wtp` | object | 是 | 支付意愿 | `{capacity:{...}, brand:{...}}` |
| `market_simulation` | object | 否 | 市场份额模拟 | `{scenarios:[...], shares:[...]}` |
| `segment_analysis` | object | 否 | 细分分析 | `{by_segment:{...}}` |
| `report_url` | string | 否 | 报告下载地址 | `"/reports/ar-dishwasher-202506.md"` |
| `created_at` | datetime | 是 | 生成时间 | `"2026-06-09T16:00:00Z"` |

### 7.1 `convergence` 子字段

| 字段名 | 类型 | 描述 | 阈值 |
|---|---|---|---|
| `rhat_max` | number | 最大 R-hat | 必须 < 1.1 |
| `ess_min` | number | 最小有效样本量 | 建议 ≥ 400 |
| `converged` | boolean | 是否收敛 | `rhat_max < 1.1` |

## 九、偏见审计记录 `BiasAuditRecord`

单次画像批次的公平性审计结果。

| 字段名 | 类型 | 必填 | 描述 | 示例 |
|---|---|---|---|---|
| `audit_id` | string | 是 | 审计记录 ID | `"audit-20250609-001"` |
| `batch_id` | string | 是 | 批次 ID | `"batch-dw-001"` |
| `sample_size` | integer | 是 | 样本量 | `150` |
| `tests` | object | 是 | 各项测试结果 | 见下 |
| `overall_status` | string | 是 | 总状态 | `"PASSED"` / `"FAILED"` |
| `created_at` | datetime | 是 | 审计时间 | `"2026-06-09T12:00:00Z"` |

### 9.1 `tests` 子字段

| 字段名 | 类型 | 描述 | 阈值 |
|---|---|---|---|
| `ks_test` | object | 连续属性分布 KS 检验 | `p > 0.05` |
| `chi_square_test` | object | 分类属性卡方检验 | `p > 0.05` |
| `cramers_v` | object | Cramér's V 效应量 | `< 0.1` |
| `shannon_entropy` | object | 香农熵 | `normalized > 0.7` |

## 十、附录：Effects Coding 字段命名规范

分类属性在编码后的字段命名规则如下，确保分析系统、报告、看板使用同一套列名：

```
{attribute_id}_{level_index}
```

其中 `level_index` 为 `0` 到 `n_levels - 2`，最后一个水平由前 `n-1` 个参数取负和恢复。

### 10.1 洗碗机场景编码示例

| 属性 | 水平 | 编码字段 | effects coding 值 |
|---|---|---|---|
| `capacity` | 6套 | `capacity_0` | `1`（相对于基准） |
| `capacity` | 10套 | `capacity_1` | `1`（相对于基准） |
| `capacity` | 13套 | — | 由 `-capacity_0 - capacity_1` 恢复 |
| `installation` | 台式 | `installation_0` | `1` |
| `installation` | 嵌入式 | `installation_1` | `1` |
| `installation` | 水槽式 | — | 由负和恢复 |
| `features` | 基础 | `features_0` | `1` |
| `features` | 智能 | `features_1` | `1` |
| `features` | 全能 | — | 由负和恢复 |
| `brand` | 美的 | `brand_0` | `1` |
| `brand` | 西门子 | `brand_1` | `1` |
| `brand` | 方太 | `brand_2` | `1` |
| `brand` | 小米 | — | 由负和恢复 |
| `energy` | 二级 | `energy_0` | `1` |
| `energy` | 一级 | `energy_1` | `1` |
| `energy` | 超一级 | — | 由负和恢复 |
| `price` | 连续 | `price` | 原始价格值 |

编码后总参数数 = 1（price）+ 2 + 2 + 2 + 3 + 2 = **12**。
