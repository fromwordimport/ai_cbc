# AI_CBC 隐私政策

> **版本**：v1.0  
> **定位**：合规文档 / 数据主体权利告知  
> **维护人**：小伦（伦理/合规负责人）  
> **关联文档**：[数据主体权利接口](../../src/aicbc/api/routes/personas.py)、[数据字典](../../docs/数据字典.md)、[数据处理器协议](./data_processing_agreement.md)

## 1. 适用范围

本隐私政策适用于 AI_CBC 虚拟消费者研究平台（以下简称“本平台”）在提供 Choice-Based Conjoint（CBC）市场研究服务过程中收集、存储、处理和传输的所有数据。

## 2. 数据控制者与联系方式

- **数据控制者**：AI_CBC Platform
- **联系邮箱**：privacy@aicbc.example.com（示例，生产环境需替换）

## 3. 我们收集哪些数据

本平台处理的个人数据分为以下类别：

| 数据类别 | 示例 | 来源 |
|----------|------|------|
| 账户与角色信息 | 用户名、角色（admin/researcher/viewer） | 用户注册/SSO |
| 研究配置数据 | 研究 ID、产品属性、问卷设计 | 研究员上传/配置 |
| 虚拟消费者画像 | 四层画像（人口统计、行为、心理、情境） | LLM 生成 |
| 模拟作答数据 | 选择记录、完成状态 | 模拟器生成 |
| 审计日志 | 操作时间、IP、动作、资源 | 系统自动记录 |
| 成本与用量数据 | Token 消耗、模型调用次数 | 系统自动记录 |

**重要说明**：本平台生成的“虚拟消费者画像”为合成数据，不对应任何真实自然人。但在 GDPR/PIPL 框架下，我们仍将其视为需要保护的数据主体权益对象，并赋予相应的导出、删除与可携带权利接口。

## 4. 数据使用目的

1. 生成虚拟消费者画像并模拟其在 CBC 实验中的选择行为。
2. 运行统计分析（HB/MNL/LCM）并生成研究报告。
3. 审计系统操作，满足安全合规与可追溯要求。
4. 监控系统健康状态与成本消耗。

## 5. 数据存储与保留

- **存储位置**：MongoDB 数据库，默认启用持久化卷。
- **保留期限**：
  - 研究数据：自研究创建之日起 36 个月，到期后自动清理。
  - 审计日志：自记录之日起 24 个月。
  - 成本数据：自记录之日起 12 个月。
- **加密**：
  - 传输层：TLS 1.2+
  - 存储层：API Key 等敏感配置使用 AES-256-GCM 加密。

## 6. 数据主体权利

根据 GDPR/PIPL，数据主体享有以下权利，本平台已通过 API 提供对应能力：

| 权利 | 接口 | 说明 |
|------|------|------|
| 访问权 | `GET /api/v1/personas/{persona_id}/export` | 导出单个画像完整数据 |
| 访问权 | `GET /api/v1/studies/{study_id}/export` | 导出研究及其全部衍生数据 |
| 删除权 | `DELETE /api/v1/personas/{persona_id}` | 删除画像及其关联作答 |
| 删除权 | `DELETE /api/v1/studies/{study_id}` | 级联删除研究、画像、作答、分析结果 |
| 可携带权 | 导出接口返回 JSON | 支持结构化、通用格式 |

## 7. 数据共享与传输

- 本平台不会将数据出售给第三方。
- 与 LLM 提供商（Anthropic/OpenAI）的交互仅包含生成/模拟所需的必要上下文，不含真实个人数据。
- 跨境传输需通过数据处理器协议（DPA）明确约定。

## 8. 数据安全

- 输入清洗：对 narrative、language_samples 等字段进行注入与同形异义字检测。
- 访问控制：基于角色的权限控制（RBAC），admin/researcher/viewer 分级授权。
- 审计与监控：所有写操作记录审计日志。
- 成本熔断：防止异常调用导致数据与成本失控。

## 9. 隐私影响评估

平台已针对以下风险完成隐私影响评估（PIA）：

1. 生成数据可能被误用为真实消费者画像。
2. LLM 提示中意外泄露研究敏感属性。
3. 删除操作未级联清理关联数据。

缓解措施包括：偏见审计、Prompt 输出过滤、级联删除、数据导出水印等。

## 10. 政策更新

本政策可能随法规或业务变化而更新，更新后将在平台内公告并同步修订版本号。
