# AI_CBC 数据处理协议（DPA）

> **版本**：v1.0  
> **定位**：合规文档 / 数据处理器与数据控制者之间的协议模板  
> **维护人**：小伦（伦理/合规负责人）  
> **关联文档**：[隐私政策](./privacy_policy.md)、[Agent 安全架构纲要](../Agent安全架构纲要.md)、[数据字典](../数据字典.md)

## 1. 协议双方

- **数据控制者（Controller）**：使用 AI_CBC 平台开展市场研究的机构或团队。
- **数据处理器（Processor）**：AI_CBC Platform 运营方。

## 2. 处理目的与范围

数据处理器仅可按照数据控制者的 documented instructions 处理数据，处理目的限于：

1. 生成虚拟消费者画像；
2. 设计并执行 CBC 问卷实验；
3. 模拟消费者选择行为；
4. 运行统计分析与生成报告；
5. 满足安全、审计与合规要求。

## 3. 数据类别

处理的数据类别详见《隐私政策》第 3 节，包括但不限于：

- 研究配置与问卷设计
- 虚拟消费者画像（四层模型）
- 模拟作答与原始数据集
- 分析结果与衍生指标
- 审计日志与操作记录

## 4. 分包处理商（Sub-processors）

| 分包商 | 服务 | 数据类型 | 处理方式 |
|--------|------|----------|----------|
| Anthropic | LLM 模型调用 | 画像生成提示、模拟选择提示 | API 调用，不用于模型训练 |
| OpenAI | LLM 模型调用（降级/备选） | 同上 | API 调用，不用于模型训练 |
| MongoDB Inc. / 自托管 MongoDB | 数据持久化 | 全部业务数据 | 存储于控制者指定区域 |

新增分包商须提前 30 日通知控制者，并提供退出机制。

## 5. 数据安全义务

数据处理器承诺：

1. 实施技术与组织措施（TOMs）保障数据机密性、完整性与可用性。
2. 对 API Key、密钥等敏感配置进行加密存储。
3. 对所有写操作生成不可篡改的审计日志。
4. 对输入数据进行注入、同形异义字与偏见攻击检测。
5. 定期进行安全扫描、漏洞修复与红队测试（目标防御成功率 ≥95%）。

## 6. 数据主体权利支持

数据处理器应提供技术接口，协助数据控制者响应数据主体请求：

- `GET /api/v1/personas/{persona_id}/export`
- `GET /api/v1/studies/{study_id}/export`
- `DELETE /api/v1/personas/{persona_id}`
- `DELETE /api/v1/studies/{study_id}`

删除操作应确保无残留，包括数据库记录、缓存、衍生分析结果与审计追踪中的可识别内容。

## 7. 数据泄露通知

发生个人数据泄露、未经授权访问或数据丢失事件时，数据处理器应在知悉后 72 小时内通知数据控制者，并配合开展影响评估与补救。

## 8. 数据删除与返还

协议终止或服务到期后：

1. 数据处理器应根据控制者要求删除或返还全部数据。
2. 删除应覆盖数据库、备份（超出法定保留期的部分）及日志中的可识别内容。
3. 如需保留匿名化统计摘要，须事先获得控制者书面同意。

## 9. 审计与合规检查

数据控制者有权在合理通知后，对数据处理器的安全措施、日志与合规文档进行审计。数据处理器应提供必要协助。

## 10. 适用法律与争议解决

本协议适用中华人民共和国法律（如数据控制者位于欧盟，则补充适用 GDPR）。因本协议产生的争议应首先通过友好协商解决，协商不成的提交数据控制者所在地有管辖权的法院诉讼解决。
