## 三、P0级风险专项应对计划

### R01：LLM输出格式不稳定（T01）

| 项目 | 内容 |
|------|------|
| **风险描述** | LLM生成的画像或模拟回答格式不符合预期，导致解析失败 |
| **触发条件** | ①模型版本更新②Prompt长度超限③temperature设置不当④边缘Case |
| **应对策略** | 减轻 |
| **具体措施** | 1. Prompt中嵌入严格JSON Schema示例<br>2. temperature固定为0.3~0.5<br>3. 3次重试（指数退避）<br>4. 输出经过SchemaValidator校验<br>5. 失败时降级为更稳定的模型 |
| **应急方案** | 启用备用模型（GPT-4o），调整Prompt模板 |
| **责任人** | 小应 |
| **监控指标** | 解析成功率、重试次数、模型切换频率 |

### R02：业务方不信任虚拟消费者（B01）

| 项目 | 内容 |
|------|------|
| **风险描述** | 业务方认为虚拟消费者不可靠，拒绝采用分析结果 |
| **触发条件** | ①首次输出与业务直觉冲突②盲测未达标③缺乏成功案例④解释不透明 |
| **应对策略** | 减轻+转移 |
| **具体措施** | 1. 每次输出附验证逻辑说明<br>2. 盲测报告作为附件<br>3. 首批结果与业务专家共同解读<br>4. 明确标注"辅助决策，非唯一依据"<br>5. 收集正面反馈包装案例 |
| **应急方案** | 引入真实消费者数据做对比验证，调整虚拟消费者参数 |
| **责任人** | 小培+小验 |
| **监控指标** | UAT满意度、业务方采纳率、反馈负面率 |

### R03：成本超预算（C01+C02）

| 项目 | 内容 |
|------|------|
| **风险描述** | 单次研究或开发阶段LLM调用费用超出预算 |
| **触发条件** | ①批量生成规模超预期②重试次数过多③未触发熔断④模型选择不当 |
| **应对策略** | 减轻 |
| **具体措施** | 1. 三层熔断：API级（单次调用上限）/服务级（单研究上限）/账户级（日/周上限）<br>2. 自动降级：超80%预算→切换Haiku<br>3. 实时成本看板<br>4. 开发环境强制Mock模式<br>5. 批量调用优化（每API调用处理多题） |
| **应急方案** | 暂停非紧急任务，启用本地模型（Ollama）或降低样本量 |
| **责任人** | 小控 |
| **监控指标** | 单次研究成本、日均开发成本、熔断触发次数 |

### 5.2 风险升级规则

```
P3风险 → 概率/影响上升 → P2风险（通知小P）
P2风险 → 概率/影响上升 → P1风险（小P介入，制定应对方案）
```
