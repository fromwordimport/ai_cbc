# 合成结果与先验认知对比验证方案

> **版本**：v1.0
> **定位**：Phase 5 验证——确保虚拟消费者CBC选择数据经HB/MNL建模后，输出的效用值与市场先验认知一致
> **负责人**：小联（联合分析领域专家）
> **前置文档**：《洗碗机CBC实验设计方案》、《项目成功标准书》、《03-模型实现指南》
> **首次验证场景**：洗碗机产品定价策略研究

---

## 一、验证目标

本方案的核心问题是：**虚拟消费者在CBC选择任务中的行为，经统计建模后，是否能够复现真实市场研究中的先验认知？**

具体而言：
1. 每个属性的效用方向是否符合经济学直觉？
2. 属性重要性排序是否与市场常识一致？
3. 不同细分群体的偏好差异是否可以被先验假设解释？
4. 如果出现偏离，偏离的程度是否在可接受范围内？

## 二、先验假设清单

### 2.1 属性方向性假设（Sign Hypotheses）

这些是最强约束——如果违反，说明数据或模型存在严重问题。

| 编号 | 假设 | 预期符号/方向 | 约束强度 | 违反后果 |
|------|------|-------------|---------|---------|
| H1 | 价格系数为负 | `β_price < 0` | **硬约束** | 一票否决——模型或数据有根本性问题 |
| H2 | 全能 > 基础（功能递增） | `U(全能) > U(基础)` | **软约束** | 需检查样本构成或功能标签描述 |
| H3 | 智能 > 基础（功能递增） | `U(智能) > U(基础)` | **软约束** | 同上 |
| H4 | 一级 > 二级（能效递增） | `U(一级) > U(二级)` | **软约束** | 检查能效信息是否充分传达 |
| H5 | 超一级 > 一级（能效递增） | `U(超一级) > U(一级)` | **软约束** | 检查溢价感知是否过高 |
| H6 | 10套 > 6套（容量递增，总体均值） | `U(10套) > U(6套)` | **软约束** | 混合样本下可能因独居者占比而逆转 |
| H7 | 嵌入式 > 台式（总体均值） | `U(嵌入式) > U(台式)` | **软约束** | 租房群体占比高时可能逆转 |

### 2.2 属性重要性排序假设（Rank Hypotheses）

基于市场常识的预期排序：

| 排名 | 属性 | 预期重要性范围 | 理由 |
|------|------|--------------|------|
| 1 | **价格** | 35%-45% | 洗碗机作为大家电，价格是首要决策因素 |
| 2 | **容量** | 15%-25% | 家庭规模直接决定容量需求 |
| 3 | **品牌** | 12%-18% | 家电品牌信任度高，但低于价格和容量 |
| 4 | **核心功能** | 10%-16% | 功能差异可感知，但非门槛因素 |
| 5 | **安装方式** | 8%-12% | 由住房条件决定，主动选择空间较小 |
| 6 | **能耗等级** | 5%-10% | 长期成本考虑，但年差异仅约¥100 |

**可接受偏离**：相邻排名可交换（如品牌与功能互换），但价格必须保持第一，能耗必须保持末位。

### 2.3 细分群体差异化假设（Segment Hypotheses）

| 编号 | 群体 | 关键差异化预期 | 验证方法 |
|------|------|--------------|---------|
| S1 | **精致白领** vs **Z世代租客** | 白领价格敏感性 **显著低于** Z世代 | 价格系数绝对值比较；t检验 |
| S2 | **新手宝妈** vs **Z世代租客** | 宝妈容量偏好 **显著高于** Z世代 | 大容量效用值比较 |
| S3 | **新手宝妈** vs **精致白领** | 宝妈功能"全能"效用 **显著更高** | "全能"水平效用值比较 |
| S4 | **Z世代租客** vs 其他两组 | 台式安装偏好 **显著更高** | "台式"水平效用值比较 |
| S5 | **精致白领** vs **Z世代租客** | 白领品牌重要性 **显著更高** | 品牌属性重要性比较 |
| S6 | **Z世代租客** vs 其他两组 | 能耗重要性 **显著更低** | 能耗属性重要性比较 |

### 2.4 品牌效用排序假设（Brand Hypotheses）

| 编号 | 全样本预期排序 | 精致白领预期 | 新手宝妈预期 | Z世代租客预期 |
|------|--------------|------------|------------|-------------|
| B1 | 西门子 > 方太 > 美的 > 小米 | 西门子 > 小米 > 方太 > 美的 | 方太 > 西门子 > 美的 > 小米 | 小米 > 美的 > 方太 > 西门子 |

**理由**：
- 西门子作为德系高端品牌，在全样本中应领先
- 精致白领受智能生态吸引，小米排名较高
- 宝妈信任厨电专业品牌，方太排名最高
- Z世代租客预算敏感，小米/美的性价比品牌领先

## 三、验证方法

### 3.1 符号检查（Sign Check）

**方法**：遍历所有属性水平（效果编码参数化下），检查：
- 价格系数符号（连续变量，必须为负）
- 每个分类属性中，最低水平（基准水平，效果编码下为负和）的相对方向

**判定规则**：
- 价格系数为正 → **FAIL，一票否决**
- 其他属性方向与预期相反 → **WARNING，需人工审查**

**实现伪代码**：
```python
def sign_check(model_results: dict, hypotheses: list[SignHypothesis]) -> dict:
    results = {"passed": [], "warnings": [], "failed": []}
    for h in hypotheses:
        actual_sign = np.sign(model_results[h.param_name])
        if h.constraint == "hard" and actual_sign != h.expected_sign:
            results["failed"].append(h)
        elif actual_sign != h.expected_sign:
            results["warnings"].append(h)
        else:
            results["passed"].append(h)
    return results
```

### 3.2 排序一致性检查（Rank Consistency）

**方法**：计算先验排序与模型输出排序的Kendall tau相关系数。

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 全样本属性重要性排序 | Kendall tau | τ ≥ 0.73 (允许1对反序) |
| 品牌效用排序（全样本） | Kendall tau | τ ≥ 0.67 |
| 各群体品牌效用排序 | 分别计算Kendall tau | ≥ 2/3群体通过 |

**Kendall tau 参考**：
- 6个属性，1对反序 → τ ≈ 0.73（可接受）
- 6个属性，2对反序 → τ ≈ 0.47（需审查）
- 4个品牌，1对反序 → τ ≈ 0.67（可接受）

### 3.3 效应大小比较（Effect Size）

**方法**：对每个属性，将效用值归一化到 [0, 1] 区间后计算：

| 指标 | 公式 | 含义 |
|------|------|------|
| 属性重要性 | `range(utility_levels) / Σ range(all_attrs)` | 该属性在决策中的相对权重 |
| 效用跨度比 | `range(attr_i) / range(attr_j)` | 两个属性重要性的倍数关系 |

**通过标准**：
- 价格效用跨度 ≥ 任何其他属性的1.5倍
- 能耗效用跨度 ≤ 任何其他属性的0.75倍

### 3.4 群体差异显著性检验

**方法**：
1. 如果有HB个体效用，对每个属性水平做两组独立样本t检验
2. 如果没有个体效用（仅MNL），使用似然比检验（LRT）比较含交互项和不含交互项的模型

**检验清单**：

| 检验 | 对比组 | 关键参数 | 预期方向 |
|------|--------|---------|---------|
| T1 | 精致白领 vs Z世代 | β_price | 白领 > Z世代（白领的负系数更小，即更不敏感） |
| T2 | 宝妈 vs Z世代 | U(13套) | 宝妈 > Z世代 |
| T3 | 宝妈 vs 精致白领 | U(全能) | 宝妈 > 白领 |
| T4 | Z世代 vs 宝妈 | U(台式) | Z世代 > 宝妈 |
| T5 | 精致白领 vs Z世代 | 品牌重要性 | 白领 > Z世代 |

**通过标准**：≥ 4/5 检验在预期方向上显著（p < 0.05，或贝叶斯框架下95% HDI不含零点）。

### 3.5 WTP合理性检查

**方法**：
```
WTP(level_a → level_b) = -(U_b - U_a) / β_price
```

其中 `β_price` 是价格系数（负值），所以WTP始终为正。

**检查清单**：
- 所有WTP值必须 > 0（升级属性应产生正WTP）
- WTP(6套→10套) ≈ ¥400-¥1,200
- WTP(10套→13套) < WTP(6套→10套)（边际递减）
- WTP(二级→一级) < 年电费节省 × 5年 ≈ ¥300

## 四、可接受偏差范围

| 验证维度 | 目标值 | 可接受上限 | 超出后果 |
|---------|--------|-----------|---------|
| 符号检查通过率 | 100% | 硬约束100%，软约束≥80% | 硬约束失败一票否决 |
| 排序一致性（Kendall τ） | ≥ 0.73 | ≥ 0.47 | < 0.47需排查样本偏差 |
| 属性重要性偏差 | ≤ 10% | ≤ 15% | > 15%需审查属性定义 |
| 群体差异t检验 | p < 0.05 | p < 0.10（方向正确） | 不显著需增大样本量 |
| WTP偏差（vs先验中线） | ≤ 25% | ≤ 30% | > 30%需审查价格敏感度校准 |
| 品牌效用排序一致性 | ≥ 0.67 | ≥ 0.33 | < 0.33需排查品牌标签设计 |
| 边际递减（容量WTP） | 成立 | 不违反 | 违反需审查非线性 |

## 五、偏离排查流程

当验证发现先验认知与模型输出不一致时，按以下流程逐层排查：

```
Level 0: 数据完整性检查
  ├── 样本量是否达标？（每组 ≥ 80）
  ├── 选择集设计是否满足D-efficiency ≥ 0.80？
  ├── 是否有大量缺失值或异常选择？
  └── None选项选择率是否在10%-50%的正常范围？
      │
      ▼
Level 1: 模型拟合检查
  ├── HB模型：R-hat < 1.1？ESS > 100？
  ├── MNL模型：对数似然收敛？标准误是否合理？
  ├── 是否有参数估计的极端值（|β| > 5）？
  └── 价格系数是否在合理范围（通常 -0.0003 到 -0.0010 之间）？
      │
      ▼
Level 2: 样本构成检查
  ├── 各群体比例是否与设计一致？（白领40%、宝妈35%、Z世代25%）
  ├── 是否有单个群体占比过高导致全样本结果偏差？
  ├── 群体内画像多样性是否足够？（张力检查）
  └── 是否有画像生成偏差（如生成的全是高收入群体）？
      │
      ▼
Level 3: 属性设计检查
  ├── 属性水平间距是否合理？（价格步长¥1,000是否太密/太稀？）
  ├── 标签描述是否充分传达了差异？
  ├── 禁止组合是否过度限制了设计空间？
  └── 某些水平是否在设计中占比过低？
      │
      ▼
Level 4: 先验假设修正
  ├── 该市场的先验认知本身是否准确？（查行业报告验证）
  ├── 是否发现了"反直觉但合理"的洞察？
  ├── 虚拟消费者是否揭示了我们忽略的市场细分？
  └── 如果是，记录为"反直觉洞察"并更新先验认知库
```

### 5.1 常见偏离场景与应对

| 偏离现象 | 可能原因 | 排查方向 | 解决建议 |
|---------|---------|---------|---------|
| 价格系数为正 | 编码错误（价格未取负或符号反了） | Level 0 数据检查 | 检查数据预处理中价格编码 |
| 能耗重要性过高（>25%） | 能效标签过度强调环保属性 | Level 3 属性设计 | 检查标签描述是否客观 |
| 品牌排序完全颠倒 | 虚拟消费者未获得品牌信息 | Level 3 属性设计 | 检查prompt中品牌信息传递 |
| 三个群体无显著差异 | 画像多样性不足 | Level 2 样本构成 | 提高张力要求，增加异常值比例 |
| 某属性所有水平效用相同 | 该属性在设计中无差异（如所有选择集中该属性恒定） | Level 0 数据检查 | 检查实验设计的水平均衡性 |
| WTP极高（>¥3,000） | 价格系数绝对值过小 | Level 1 模型检查 | 检查价格步长是否合理 |
| None选项选择率 > 50% | 所有选项吸引力不足 | Level 0 数据检查 | 检查属性水平设计是否极端 |

## 六、最小验证样本量

### 6.1 统计效力计算

对于群体差异t检验，所需最小样本量：

```
效应量 d = (μ₁ - μ₂) / σ_pooled

对于中等等效量 d = 0.5：
  - 每组需 64 人（α=0.05, power=0.80, 双尾）
  - 三组共需 192 人

对于小等效量 d = 0.35：
  - 每组需 129 人（α=0.05, power=0.80, 双尾）
  - 三组共需 387 人
```

### 6.2 推荐样本量

| 验证目的 | 最小样本量（每组） | 推荐样本量（每组） | 理由 |
|---------|------------------|------------------|------|
| 符号检查 + 排序一致性 | 30 | 50 | 聚合层面，MNL即可满足 |
| 群体差异显著性（大效应 d≥0.5） | 64 | 80 | 当前设计（3×80=240）满足 |
| 群体差异显著性（中效应 d≥0.35） | 129 | 150 | 当前设计偏紧，可增量 |
| 品牌效用排序（全样本） | 50 | 80 | 聚合层面 |
| WTP估算（±20%精度） | 100 | 150 | 需要稳定的价格系数 |
| 群体×属性交互效应 | 80 | 120 | 多参数模型需要更大样本 |

### 6.3 当前设计的样本量评估

当前实验设计：3群体 × 80人 = 240人，12选择集/人 = 2,880个选择观察。

- **优势**：总选择观察数充足（12参数 × 5倍 = 60，实际有2880观察，远超最小要求）
- **劣势**：群体级80人对于中等以下效应量（d < 0.5）的群体差异检验可能效力不足
- **建议**：
  - 验证阶段可以使用当前240人设计
  - 如果群体差异检验中某项不显著但方向正确，考虑补充样本至100人/组
  - 重点检验假设（S1-S4）应优先分配统计效力

## 七、验证执行checklist

### 7.1 数据准备阶段

- [ ] 确认原始数据格式符合《数据字典》`CBCRawDataset` 规范
