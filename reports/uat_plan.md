### 1.2 标准可测量性分析

经逐条审查，发现以下指标存在模糊性或矛盾：

| 编号 | 指标 | 出处 | 问题类型 | 具体问题 |
|------|------|------|---------|---------|
| **F-01** | "TOP3属性符合业务直觉" | 项目成功标准书 4.1 | **主观指标** | "业务直觉"依赖个体经验，未定义：(1) 专家需几位？(2) 意见分歧时如何裁决？(3) 专家的资质标准是什么？ |
| **F-02** | "至少1个反直觉洞察" | 系统成功标准书 5.2 | **定义模糊** | "反直觉"无法量化。谁来判定"反直觉"？如果所有结果都在预期内（系统可信但没有惊喜），算通过还是失败？ |
| **F-03** | "消费者真实性评分" 尺度不一致 | 多文档交叉 | **数值矛盾** | `业务验收标准与KPI框架.md` 使用 **10分制**（目标 >= 8.5/10）；`虚拟消费者真实性验收标准.md` 使用 **14分制**（合格 >= 9/14）；`系统成功标准书.md` v1.2 已统一为14分制。但 KPI 框架文档未同步更新。 |
| **F-04** | "报告可直接用于决策会议" | 系统成功标准书 5.2 | **主观指标** | "可直接使用"无客观判定标准。需定义：(1) 格式要求？(2) 内容完整度？(3) 谁判定？ |
| **F-05** | "方向正确" (市场份额) | 系统成功标准书 2.1 | **不够精确** | 只要降价份额>=涨价份额即为"方向正确"，但这是极低门槛。R^2 >= 0.5 是量化标准但未核验与"方向正确"的关系。 |
| **F-06** | "专家评审" 无方法论规范 | 多文档多处 | **流程缺失** | 未定义：(1) 最少专家人数；(2) 评分者间信度 (IRR) 是否检验；(3) 专家评审结果与定量指标冲突时以谁为准。 |
| **F-07** | WTP偏差标准不统一 | 多文档交叉 | **数值矛盾** | `项目成功标准书.md` WTP偏差 <= 25%（目标）/ <= 30%（可接受）；`系统成功标准书.md` 同为 <= 25% / <= 30%。但 `业务验收标准与KPI框架.md` 模板中写的是 <= 40%。v2.0 UAT 以 v1.2 为准。 |
| **F-08** | "业务用户满意度" 采样偏差风险 | 系统成功标准书 3.3 | **方法缺陷** | 满意度评分仅来自10位UAT参与者，样本量太小（n=10），无法代表全部业务用户。需在后续阶段扩大采样。 |
| **F-09** | "替代调研比例" 缺乏基线 | 系统成功标准书 3.1 | **基线缺失** | 目标">= 30%"是绝对比例，但当前业务方使用传统调研的频次未知。应先统计基线再设定目标。 |
| **F-10** | "属性重要性数值偏差 <= 10%" | 系统成功标准书 2.1 | **参照基准缺失** | 偏差与什么对比？历史真实调研数据？专家预判？如果两者冲突，以哪个为准？ |

### 1.3 文档间矛盾汇总

| 矛盾编号 | 涉及文档 | 矛盾内容 | 建议解决方案 | 优先级 |
|---------|---------|---------|-------------|--------|
| C-01 | `业务验收标准与KPI框架.md` vs `虚拟消费者真实性验收标准.md` | 真实性评分量表：10分制 vs 14分制 | 统一为14分制（七维度x2分），更新 KPI 框架文档中的 8.5/10 -> 9/14 | **P0** |
| C-02 | `项目成功标准书.md` vs `系统成功标准书.md` | 前者 Must Have 清单未包含 "公平性硬规则嵌入Prompt"（v1.2新增） | 更新 `项目成功标准书.md` 引用 v1.2 为权威来源 | **P0** |
| C-03 | `业务验收标准与KPI框架.md` vs `系统成功标准书.md` | WTP可接受偏差：40% vs 30% | 统一为30%（v1.2 为准） | P1 |
| C-04 | 三文档间 | 阶段门禁条件不完全一致 | 以 `系统成功标准书.md` v1.2 第四节为准 | P1 |

### 1.4 改进建议

针对 F-01 至 F-10，小验提出以下标准化建议：

1. **"业务直觉"操作化**：定义专家组（>=3人，含小联+2位外部领域专家），采用多数表决制（>=2/3认可即通过），分歧时记录异议并标注在报告中。
2. **"反直觉洞察"操作化**：定义为"专家预判中未提及、且事后被认可为有价值的发现"。专家组在测试前先书面记录预期发现，测试后对比。
3. **"报告可直接使用"操作化**：定义检查清单——(a) 含执行摘要（<=1页）；(b) 含5个关键业务问题回答；(c) 图表标注完整；(d) 含使用建议和免责声明。
4. **专家评审规范化**：每次UAT至少3位专家独立评审，计算 Fleiss' Kappa 检验评分者间信度；Kappa < 0.4 需增加专家或重新校准评审标准。

### UAT-01：创建研究项目并配置CBC实验

**验证的业务能力**: 系统是否能正确接收业务需求并将其转化为有效的实验设计？

| 项目 | 内容 |
|------|------|
| **用例编号** | UAT-01 |
| **优先级** | P0 |
| **测试场景** | 产品经理创建一个新洗碗机定价研究项目，配置产品属性、水平、目标人群 |
| **前置条件** | 系统已部署并处于可用状态；用户已登录 |
| **测试数据** | 洗碗机产品，6个属性（价格、容量、安装方式、核心功能、品牌、能耗等级），3个目标细分群体（精致白领 80人、新手宝妈 80人、Z世代租客 80人） |

#### 操作步骤

| 步骤 | 操作描述 | 验证点 |
|------|---------|--------|
| 1 | 进入系统首页，点击"新建研究项目" | 入口清晰可见 |
| 2 | 输入项目名称："洗碗机新品定价与配置研究" | 支持中文输入 |
| 3 | 选择产品类别："大家电 > 洗碗机" | 类别选择器可用 |
| 4 | 逐条添加6个属性及其水平（含3属性3水平，3属性4水平） | 支持动态增删属性；水平数量可不同 |
| 5 | 设定研究参数：样本量240人、选择集12题、每题3选项+None | 参数配置界面完整 |

4. 三个目标细分群体被正确关联到研究，人群约束参数（年龄、城市、收入等）生效
5. 系统自动生成实验设计摘要（含正交设计/D-efficiency预估）

#### 验收标准

| 检查项 | 标准 | 通过条件 | 测量方法 |

| 水平配置完整性 | 每个属性的所有水平均被保存（3+3+3+4+4+3=20水平） | 20/20水平正确 | API返回验证 |

| 操作流畅性 | 从开始到提交完成 | <= 10分钟 | 计时 |
| 无阻断性错误 | 过程中无报错或崩溃 | 0次异常 | 日志检查 |

### UAT-02：生成虚拟消费者画像并检查真实性

**验证的业务能力**: 生成的虚拟消费者是否"像真人"，能否让业务用户产生代入感？

| 项目 | 内容 |
|------|------|
| **用例编号** | UAT-02 |
| **优先级** | P0 |
| **测试场景** | 在已创建的研究项目中触发240个虚拟消费者画生成，抽样检查画像的真实性、立体感和群体区分度 |
| **前置条件** | UAT-01已通过（研究项目已创建） |
| **测试数据** | 240个画像，分属3个细分群体，每个群体80人 |

#### 操作步骤

| 步骤 | 操作描述 | 验证点 |
|------|---------|--------|
| 1 | 在项目页面点击"生成消费者画像" | 触发按钮可见 |
| 2 | 确认生成参数：每群体80人/共240人/启用偏见审计 | 参数确认界面 |
| 3 | 启动生成，观察进度条/状态更新 | 进度信息可见、状态正确轮转 |
| 4 | 生成完成后，进入画像浏览页面 | 画像列表可浏览、可按群体筛选 |
| 5 | 随机抽取9个画像（每群体3个），逐层阅读四层信息 | 四层信息完整、无空字段 |
| 6 | 检查画像的"张力"特征（内部矛盾 + 叙事解释 >= 50字） | 张力组合存在且有足够长度的叙事解释 |
| 7 | 检查画像"发言样本"是否符合该群体特征和语言自然度 | 语言风格与画像一致、无禁止术语 |
| 8 | 查看画像的authenticity_score分布 | 评分可见，均值 >= 9（14分制） |
| 9 | 检查偏见审计结果 | 无CRITICAL级别偏见 |

#### 预期输出

1. 240个画像全部生成成功，生成时间在可接受范围内（目标 <= 30分钟）
2. 每个画像包含四层完整信息（Layer1-4 + 发言样本 + 洗碗机购买情境）
3. 画像内部存在"张力"，且有心理叙事解释（>= 50字）
4. 不同细分群体间的画像存在显著差异（群体可被正确分类）
5. authenticity_score 均值 >= 9（满分14，等价于7维度总分 >= 9）
6. 偏见审计未发现 CRITICAL 级别问题；HIGH 级别 <= 2个且已标注修复计划

#### 验收标准

| 检查项 | 标准 | 通过条件 | 测量方法 |
|--------|------|---------|---------|
| 生成成功率 | 240个画像全部生成 | 100%（无失败画像） | 系统统计 |
| 四层信息完整性 | 每画像均含 Layer1-4 + 发言样本 + 洗碗机情境 | 抽样检查100%完整 | 人工抽查9个 |
| 张力特征存在 | 矛盾标签 + 叙事解释 >= 50字 | >= 80%的抽查画像满足 | 人工抽查9个 |
| 群体区分度 | 不同群体的画像可被业务用户区分 | 盲测分类正确率 >= 70% | 盲测 |
| 真实性评分 | authenticity_score 均值（14分制） | >= 9.0 | 系统统计 |
| 偏见审计 | 无CRITICAL级别偏见 | 100%通过 | 自动化审计 |

**通过判定**: 上述6项全部满足方为通过。群体区分度若 < 70% 但 >= 60%，标记为 HIGH 问题。

| None选项存在 | 每题都有"都不选" | 12/12 | 系统验证 |
