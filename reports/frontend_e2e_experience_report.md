### Step 5 — 模拟作答（Response Simulation）

**状态**：✅ PASS

- 操作流程：作答模拟 → 选择消费者（2个）→ 使用随机模式(stochastic) → 开始模拟
- **结果**：**「模拟完成 — 成功模拟 2 个消费者作答」**
- 耗时约 10 秒（2 个消费者 × 12 个选择集）
- **数据验证**：后端导出数据确认有 3 个受访者的 36 条记录（含之前测试的数据），每条记录包含：
  - `record_id`、`respondent_id`、`set_id`、`chosen`、`response_time_ms`
- **模式说明**：
  - stochastic（随机模式）：基于随机效用理论模拟选择行为
  - deterministic（确定性模式）：基于效用最大化
  - llm（LLM 模式）：由大语言模型驱动，模拟真实消费者的思考过程 —— **本项目核心卖点**
- **建议**：LLM 模式是核心卖点，但默认选择随机模式，且 LLM 模式在 UI 中切换的交互需要更明确的引导

## 三、关键发现与问题

### 🔴 高优先级问题

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| 1 | **画像生成数量与请求不符**：请求生成 5 个，实际只生成 2 个 | 中等 | 后端检查生成逻辑，前端增加数量不匹配提示 |
| 2 | **数据导出路径不一致**：前端调用 `/responses/export`，API 文档需要统一 | 低 | 统一 API 文档和前端调用路径 |
| 3 | **中文编码问题**：后端返回的 Unicode 转义序列在终端显示乱码 | 低 | 后端设置 `ensure_ascii=False` |

### 🟡 中优先级问题

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| 4 | **属性重要性看板空状态**：已选中研究项目但仍显示空状态，需手动点击「运行 HB 分析」 | 用户体验 | 当研究已选中时，空状态文案应改为「点击上方按钮运行 HB 分析」，并预估耗时 |
| 5 | **LLM 模式切换交互**：Ant Design Select 的选项在 DOM 中动态渲染，Playwright 自动化难以精准操作 | 自动化测试 | 为 Select 组件添加 `data-testid` 或稳定的选择器 |
| 6 | **Dashboard 空状态引导**：首次使用无引导文案 | 新用户体验 | 增加「快速开始」引导卡片 |

### 🟢 亮点与优势

| # | 亮点 | 说明 |
|---|------|------|
| 1 | **一键创建 + 自动问卷设计** | 从研究概念到可执行问卷仅需 30 秒，D-efficiency 达 0.93 |
| 2 | **虚拟消费者画像质量高** | 真实性评分 8.0/10，包含细分群体、城市层级、收入、人生阶段等多维度 |
| 3 | **三种模拟模式覆盖全场景** | stochastic（快速测试）/ deterministic（基准验证）/ llm（核心卖点） |
| 4 | **问卷设计可视化** | 12 个选择集全部可预览，属性水平组合清晰 |

## 五、体验总结与建议

### 整体评价

AI_CBC 作为 **LLM 驱动的虚拟消费者 CBC 联合分析平台**，其核心创新点——**用大语言模型替代真实消费者完成调研**——在系统架构层面已经完整落地。从创建研究到生成虚拟消费者、模拟作答、再到分析可视化，整个链路已经打通。

### 给开发团队的优化建议（按优先级排序）

1. **修复画像生成数量不一致问题**：这是数据完整性的基础问题，直接影响分析结果的可靠性
2. **优化空状态用户体验**：在属性重要性看板、分析任务状态等页面增加引导式空状态文案
3. **增加「快速开始」引导流程**：首次进入 Dashboard 时显示 3 步引导（创建研究 → 生成画像 → 运行分析）
4. **统一 API 响应编码**：后端 JSON 序列化时设置 `ensure_ascii=False`，避免中文 Unicode 转义
5. **为 E2E 测试增加稳定选择器**：关键交互元素添加 `data-testid` 属性，便于自动化测试
6. **LLM 模式作为默认选项**：在作答模拟页面，将 LLM 模式设为默认（或至少更突出），体现项目核心卖点

### 下一步验证建议

- 手动触发 HB 分析并验证结果可视化（part-worth 图、属性重要性雷达图）
- 使用 LLM 模式进行模拟作答，对比随机模式与 LLM 模式的选择分布差异
- 测试大规模画像生成（50+ 消费者）的性能和稳定性
- 测试细分市场比较功能在多画像场景下的表现
