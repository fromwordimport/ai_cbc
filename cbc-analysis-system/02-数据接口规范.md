## 一、接口概述

### 1.1 数据流

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│   问卷系统       │ ──→  │   标准交换接口   │ ──→  │   分析系统       │
│                 │ EXPORT│                │ IMPORT│                │
│ • 原始回答       │      │ • Schema 校验    │      │ • 长格式转换     │
│ • 属性定义       │      │ • 编码标准化     │      │ • 模型拟合       │
│ • 设计参数       │      │ • 增量同步       │      │ • 结果输出       │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 1.2 接口原则

1. **松耦合**：分析系统不关心问卷是如何设计的，只关心标准交换格式
2. **自描述**：数据携带完整的元信息，分析系统无需外部配置即可解析
3. **可追溯**：保留原始标识符，支持结果反向关联到问卷和受访者
4. **可增量**：支持增量导出，分析系统可基于新增数据更新结果

## 二、输入数据格式

### 2.1 标准交换 Schema

分析系统接收的数据必须遵循以下顶层结构：

```typescript
interface CBCRawDataset {
  metadata: DatasetMetadata;           // 问卷元数据
  choice_records: ChoiceRecord[];      // 选择记录（核心）
  respondent_attributes?: RespondentMap; // 受访者画像属性（可选）
}
```

### 2.2 元数据（DatasetMetadata）

```typescript
interface DatasetMetadata {
  attributes: AttributeMeta[];         // 属性定义列表
  design_parameters: DesignParameters; // 实验设计参数
  questionnaire_info?: QuestionnaireInfo; // 问卷基本信息
}

interface AttributeMeta {
  id: string;                          // 属性标识（如 "brand"）
  name: string;                        // 属性显示名称（如 "品牌"）
  type: "categorical" | "continuous" | "ordinal" | "price";
  levels?: LevelMeta[];                // 分类/有序属性的水平定义
  unit?: string;                       // 连续变量的单位（如 "mAh", "W"）
  is_none?: boolean;                   // 是否为"都不选"选项的虚拟属性
}

interface LevelMeta {
  value: any;                          // 水平值（如 "huawei" 或 3999）
  label: string;                       // 水平显示标签（如 "华为"）
  index: number;                       // 水平索引（0-based，用于编码）
}

interface DesignParameters {
  num_sets: number;                    // 选择集数量
  alts_per_set: number;                // 每集选项数
  none_option: boolean;                // 是否包含"都不选"
  randomization?: boolean;             // 是否随机化顺序
}

interface QuestionnaireInfo {
  questionnaire_id: string;
  version: string;
  created_at: string;                  // ISO 8601
  product_category?: string;
}
```

### 2.3 选择记录（ChoiceRecord）

```typescript
interface ChoiceRecord {
  respondent_id: string;               // 受访者唯一标识
  respondent_index: number;            // 受访者索引（0-based）
  segment_id?: string;                 // 所属细分群体（可选）
  choice_set_id: number;               // 选择集ID（问卷内唯一）
  choice_set_index: number;            // 选择集索引（0-based）
  alternatives: AlternativeRecord[];   // 该选择集中的所有选项
  none_chosen: boolean;                // 是否选择了"都不选"
}

interface AlternativeRecord {
  alt_index: number;                   // 选项在选择集中的索引（0-based）
  chosen: boolean;                     // 是否被选中
  attribute_values: Record<string, any>; // 原始属性值 {attr_id: value}
  encoded_vector: number[];            // 编码后的设计向量
}
```

### 2.4 受访者画像属性（RespondentMap）

```typescript
interface RespondentMap {
  [respondent_id: string]: RespondentAttributes;
}

interface RespondentAttributes {
  age_group?: string;
  gender?: string;
  income_tier?: string;
  city_tier?: string;
  price_sensitivity?: string;
  brand_loyalty?: string;
  decision_style?: string;
  [key: string]: any;                  // 允许扩展
}
```

## 三、编码规范

### 3.1 编码向量（encoded_vector）构建规则

编码向量的维度必须一致，且顺序与 `metadata.attributes` 严格对应。

#### 3.1.1 Effects Coding（默认推荐）

适用于分类变量，保证各水平参数之和为0。

```
假设属性顺序为：[brand(3水平), price(连续), storage(2水平)]

编码向量长度 = (3-1) + 1 + (2-1) = 4

brand="huawei" (index 0)  → [1, 0]
brand="xiaomi" (index 1)  → [0, 1]
brand="apple"  (index 2)  → [-1, -1]

price=3999 → [3999]

storage="128GB" (index 0) → [1]
storage="256GB" (index 1) → [-1]

最终 encoded_vector = [1, 0, 3999, 1]  // 当 brand=huawei, storage=128GB
最终 encoded_vector = [-1, -1, 3999, -1]  // 当 brand=apple, storage=256GB
```

#### 3.1.2 Dummy Coding

以参考水平为基准（通常为最后一个水平）。

```
brand="huawei" (index 0)  → [1, 0]
brand="xiaomi" (index 1)  → [0, 1]
brand="apple"  (index 2)  → [0, 0]   // 参考水平
```

#### 3.1.3 Continuous / Price

直接使用原始值，可选标准化。

```
price=3999 → [3999]
# 或标准化后
price=3999 → [0.5]  // (3999 - min) / (max - min)
```

### 3.2 编码配置

```python
class EncodingConfig:
    """编码配置"""
    
    SUPPORTED_SCHEMES = ["effects", "dummy", "orthogonal", "raw"]
    
    DEFAULTS = {
        "categorical": "effects",
        "ordinal": "orthogonal",
        "continuous": "raw",
        "price": "raw"
    }
    
    @classmethod
    def get_encoder(cls, attr_type: str, scheme: str = None):
        """获取对应类型的编码器"""
        scheme = scheme or cls.DEFAULTS[attr_type]
        
        encoders = {
            ("categorical", "effects"): EffectsEncoder(),
            ("categorical", "dummy"): DummyEncoder(),
            ("continuous", "raw"): RawEncoder(),
            ("price", "raw"): RawEncoder(),
        }
        
        return encoders.get((attr_type, scheme))
```

## 四、CSV 长格式规范

### 4.1 标准列定义

当导出格式为 CSV 时，使用长格式，每行代表一个"受访者-选择集-选项"组合：

```csv
respondent_id,respondent_index,segment_id,choice_set_id,choice_set_index,alt_index,chosen,brand,price,storage,battery,charging,none_chosen
persona-001,0,seg-001,1,0,0,0,huawei,3999,256gb,5000,66,0
persona-001,0,seg-001,1,0,1,1,xiaomi,2999,128gb,5500,120,0
persona-001,0,seg-001,1,0,2,0,apple,5999,256gb,4500,33,0
```

### 4.2 必填列

| 列名 | 类型 | 说明 |
|------|------|------|
| `respondent_id` | string | 受访者唯一标识 |
| `respondent_index` | int | 受访者索引（0-based） |
| `choice_set_id` | int | 选择集ID |
| `choice_set_index` | int | 选择集索引（0-based） |
| `alt_index` | int | 选项索引（0-based） |
| `chosen` | int (0/1) | 是否被选中 |
| `none_chosen` | int (0/1) | 是否选择"都不选" |

### 4.3 属性列

属性列根据 `metadata.attributes` 动态生成，列名使用 `attr_id`。

## 五、数据校验规则

### 5.1 Schema 级校验

```python
class SchemaValidator:
    """Schema 校验器"""
    
    RULES = [
        {
            "id": "ATTR-001",
            "name": "属性定义完整性",
            "check": lambda data: len(data["metadata"]["attributes"]) > 0,
            "severity": "ERROR",
            "message": "metadata.attributes 不能为空"
        },
        {
            "id": "ATTR-002", 
            "name": "属性ID唯一性",
            "check": lambda data: len(set(a["id"] for a in data["metadata"]["attributes"])) 
                == len(data["metadata"]["attributes"]),
            "severity": "ERROR",
            "message": "属性ID存在重复"
        },
        {
            "id": "LEVEL-001",
            "name": "水平索引连续性",
            "check": lambda data: all(
                sorted([l["index"] for l in attr.get("levels", [])]) == list(range(len(attr.get("levels", []))))
                for attr in data["metadata"]["attributes"] if attr.get("levels")
            ),
            "severity": "ERROR",
            "message": "水平索引必须从0开始连续递增"
        },
        {
            "id": "ENC-001",
            "name": "编码向量维度一致性",
            "check": lambda data: len(set(
                len(alt["encoded_vector"])
                for record in data["choice_records"]
                for alt in record["alternatives"]
            )) == 1,
            "severity": "ERROR",
            "message": "所有 encoded_vector 的维度必须一致"
        },
        {
            "id": "CHOICE-001",
            "name": "每题必有一选",
            "check": lambda data: all(
                sum(1 for alt in record["alternatives"] if alt["chosen"]) == 1
                or record["none_chosen"]
                for record in data["choice_records"]
            ),
            "severity": "WARNING",
            "message": "存在未选择或多选的题目"
        },
        {
            "id": "SAMPLE-001",
            "name": "样本量充足性",
            "check": lambda data: SchemaValidator._check_sample_size(data),
            "severity": "WARNING",
            "message": "样本量可能不足以支持精确估计"
        }
    ]
    
    @staticmethod
    def _check_sample_size(data):
        n_records = len(data["choice_records"])
        n_attrs = len(data["metadata"]["attributes"])
        # 粗略估计：每个属性至少5个观测
        return n_records >= n_attrs * 5
```

### 5.2 校验结果格式

```json

  "valid": false,
  "errors": [
    {
      "rule_id": "ENC-001",
      "severity": "ERROR",

      "location": "choice_records[15].alternatives[2]",
      "expected": 6,

    }

    {
```

### 6.3 分析系统增量处理

```python
class IncrementalProcessor:
    """增量数据处理器"""
    
    def __init__(self):
        self.accumulated_data = None
        self.last_sync = None
    
    def process_increment(self, new_data: dict) -> dict:
        """
        处理增量数据
        
        策略：
        1. 如果是首次加载，直接存储
        2. 如果是增量更新，合并数据后重新拟合模型
        3. 保留之前的结果用于热启动（HB模型的初始值）
        """
        if self.accumulated_data is None:
            self.accumulated_data = new_data
            return {"action": "full_load", "n_records": len(new_data["choice_records"])}
        
        # 合并数据
        self.accumulated_data["choice_records"].extend(new_data["choice_records"])
        
        # 更新受访者索引（避免冲突）
        self._reindex_respondents()
        
        return {
            "action": "incremental_merge",
            "total_records": len(self.accumulated_data["choice_records"]),
            "new_records": len(new_data["choice_records"])
        }
```

## 七、Python 数据加载器实现

```python
import json
import pandas as pd
from pathlib import Path
from typing import Union

class CBCDatasetLoader:
    """
    CBC 标准数据集加载器
    
    支持 JSON / CSV / Parquet 格式
    统一转换为内部长格式 DataFrame
    """
    
    def __init__(self):
        self.raw_data = None
        self.metadata = None
        self.df = None
    
    def load(self, path: Union[str, Path]) -> "CBCDatasetLoader":
        """加载数据文件"""
        path = Path(path)
        
        if path.suffix == ".json":
            self._load_json(path)
        elif path.suffix == ".csv":
            self._load_csv(path)
        elif path.suffix == ".parquet":
            self._load_parquet(path)
        else:
            raise ValueError(f"不支持的文件格式: {path.suffix}")
        
        return self
    
    def _load_json(self, path: Path):
        """加载 JSON 格式"""
        with open(path, "r", encoding="utf-8") as f:
            self.raw_data = json.load(f)
        
        self.metadata = self.raw_data["metadata"]
        self._build_dataframe()
    
    def _build_dataframe(self):
        """将原始数据转换为长格式 DataFrame"""
        rows = []
        
        for record in self.raw_data["choice_records"]:
            resp_id = record["respondent_id"]
            resp_idx = record["respondent_index"]
            seg_id = record.get("segment_id")
            set_id = record["choice_set_id"]
            set_idx = record["choice_set_index"]
            none_chosen = record["none_chosen"]
            
            for alt in record["alternatives"]:
                row = {
                    "resp_id": resp_id,
                    "resp_index": resp_idx,
                    "segment_id": seg_id,
                    "task_id": set_id,
                    "task_index": set_idx,
                    "alt_id": alt["alt_index"],
                    "choice": 1 if alt["chosen"] else 0,
                    "none_chosen": 1 if none_chosen else 0,
                }
                
                # 添加属性值
                for attr_id, value in alt["attribute_values"].items():
                    row[attr_id] = value
                
                # 添加编码向量（展平为列）
                for i, val in enumerate(alt["encoded_vector"]):
                    row[f"enc_{i}"] = val
                
                rows.append(row)
        
        self.df = pd.DataFrame(rows)
    
    def get_attribute_cols(self) -> list[str]:
        """获取属性列名"""
        return [attr["id"] for attr in self.metadata["attributes"]]
    
    def get_encoded_cols(self) -> list[str]:
        """获取编码向量列名"""
        return [col for col in self.df.columns if col.startswith("enc_")]
    
    def get_summary(self) -> dict:
        """获取数据摘要"""
        return {
            "n_respondents": self.df["resp_id"].nunique(),
            "n_choice_sets": self.df.groupby("resp_id")["task_id"].nunique().iloc[0],
            "n_records": len(self.df),
            "n_attributes": len(self.metadata["attributes"]),
            "attributes": [
                {"id": a["id"], "type": a["type"], "n_levels": len(a.get("levels", []))}
                for a in self.metadata["attributes"]
            ]
        }
    
    def to_long_format(self, use_encoded: bool = True) -> pd.DataFrame:
        """
        获取长格式数据
        
        Parameters
        ----------
        use_encoded : bool
            是否使用编码后的向量作为特征列
        """
        if use_encoded:
            # 使用编码向量列
            feature_cols = self.get_encoded_cols()
            df = self.df[["resp_id", "resp_index", "task_id", "task_index", "alt_id", "choice"] + feature_cols].copy()
        else:
            # 使用原始属性列
            feature_cols = self.get_attribute_cols()
            df = self.df[["resp_id", "resp_index", "task_id", "task_index", "alt_id", "choice"] + feature_cols].copy()
        
        return df
```

## 八、错误码定义

| 错误码 | 级别 | 说明 | 处理建议 |
|--------|------|------|---------|
| `E001` | ERROR | 属性定义为空 | 检查上游问卷系统导出配置 |
| `E002` | ERROR | 编码向量维度不一致 | 检查编码逻辑，确保所有记录使用相同的编码方案 |
| `E003` | ERROR | 水平索引不连续 | 检查 `metadata.attributes[i].levels` 的 index 字段 |
| `E004` | ERROR | 文件格式不支持 | 使用 json / csv / parquet 格式 |
| `W001` | WARNING | 样本量不足 | 增加模拟消费者数量或选择集数量 |
| `W002` | WARNING | 存在未做选择的题目 | 检查模拟消费者 Agent 的决策逻辑 |
| `W003` | WARNING | "都不选"比例过高 (>50%) | 检查属性水平设置是否合理 |
| `W004` | WARNING | 某水平出现频率过低 (<10%) | 调整实验设计或增加样本量 |
| `I001` | INFO | 增量数据已合并 | 无需处理 |
