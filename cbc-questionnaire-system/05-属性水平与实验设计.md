## 一、最终属性/水平表

### 1.1 属性筛选结果

基于 `docs/洗碗机CBC实验设计方案.md` v1.0 的初稿，经与目标人群角色卡对齐后，最终确定以下6个属性、21个水平：

| 属性 | 类型 | 水平数 | 水平定义 | 参数数 |
|------|------|--------|---------|--------|
| **价格** | 连续（对数） | 4 | ¥2,999 / ¥3,999 / ¥4,999 / ¥5,999 | 1 |
| **容量** | 分类 | 3 | 6套(1-2人) / 10套(3-4人) / 13套(5人+) | 2 |
| **安装方式** | 分类 | 3 | 台式(免安装) / 嵌入式 / 水槽式 | 2 |
| **核心功能** | 分类 | 3 | 基础洗+烘干 / 智能洗+除菌 / 全能洗+AI+UV | 2 |
| **品牌** | 分类 | 4 | 美的 / 西门子 / 方太 / 小米 | 3 |
| **能耗等级** | 分类 | 3 | 二级 / 一级 / 超一级 | 2 |

**总参数数**：1 + 2 + 2 + 2 + 3 + 2 = **12个参数**
（价格作为连续变量处理，其余采用Effects Coding）

### 1.2 水平与角色卡对齐说明

| 群体 | 关键需求 | 对应水平侧重 |
|------|---------|-------------|
| **精致白领** | 品质+空间紧凑+智能 | 10套、台式/水槽式、智能/全能、小米/西门子 |
| **新手宝妈** | 除菌+大容量+专业品牌 | 10-13套、嵌入式、全能、方太/西门子 |
| **Z世代租客** | 低价+免安装+够用 | 6套、台式、基础、小米/美的 |

### 1.3 水平合理性校验

| 校验项 | 标准 | 结果 |
|--------|------|------|
| 价格间隔 | ≥ ¥1,000 | 通过（间隔¥1,000） |
| 容量间隔 | ≥ 4套 | 通过（间隔4套/3套） |
| 品牌覆盖度 | 涵盖不同定位 | 通过（国民/德系/专业/智能生态） |
| 功能递进 | 可感知差异 | 通过（基础→智能→全能） |
| 能耗递进 | 可量化差异 | 通过（年电费¥180→¥120→¥80） |

## 二、实验设计参数

### 2.1 核心参数

```yaml
experimental_design:
  method: "D-efficient设计"
  
  parameters:
    n_choice_sets: 12          # 每人12道选择题
    n_alternatives_per_set: 3   # 每题3个产品选项
    include_none_option: true   # 包含"都不选"选项
    n_respondents: 240         # 总样本量
    
  design_constraints:
    min_d_efficiency: 0.80     # D-efficiency最低要求
    target_d_efficiency: 0.85  # 目标D-efficiency
    level_balance: true        # 各水平出现次数均衡
    positional_balance: true   # 各位置（A/B/C）均衡
    no_repeat_alternatives: true  # 同一选择集中不出现重复配置
    
  prohibitions:
    - "西门子 + ¥2,999"        # 品牌定位冲突
    - "13套 + 台式"            # 物理空间限制
    - "全能 + ¥2,999"          # 成本不支持
    - "超一级 + ¥2,999"        # 技术成本不支持
```

### 2.2 D-efficiency目标说明

| 场景 | 预期D-efficiency | 说明 |
|------|-----------------|------|
| 无禁止组合 | ≥ 0.90 | 理论最优 |
| 有4条禁止组合 | ≥ 0.85 | 本方案目标 |
| 保守估计 | ≥ 0.80 | 最低可接受线 |

**选择0.80作为最低线的原因**：
- 4条禁止组合对设计空间有约束
- 240人样本量充足，可补偿略低的D-efficiency
- 与 `03-CBC实验设计算法说明.md` 的分级标准一致

### 2.3 样本量计算

```
总参数数 = 12个（含价格连续变量）

最小样本量公式：
n_respondents × n_choice_sets ≥ n_params × 5
n × 12 ≥ 12 × 5
n ≥ 5人（理论最小）

实际推荐：
- 基础估计：n ≥ 100
- 细分分析（群体级HB）：每群体 n ≥ 80
- 本方案：3群体 × 80人 = 240人
- 总选择观察数：240 × 12 = 2,880个选择
```

## 三、禁止组合列表

### 3.1 最终禁止组合（4条）

| 规则ID | 规则名称 | 组合 | 禁止原因 | 类型 |
|--------|---------|------|---------|------|
| PROH-001 | 高端品牌+最低价格 | 西门子 × ¥2,999 | 西门子定位高端，2999不符合品牌定位 | 品牌-价格冲突 |
| PROH-002 | 大容量+台式 | 13套 × 台式 | 台式物理空间有限，无法容纳13套 | 物理约束 |
| PROH-003 | 旗舰功能+最低价格 | 全能 × ¥2,999 | 成本不支持 | 功能-价格冲突 |
| PROH-004 | 超一级能效+最低价格 | 超一级 × ¥2,999 | 技术成本不支持 | 技术-价格冲突 |

### 3.2 禁止组合对设计的影响

| 指标 | 无禁止组合 | 有4条禁止组合 | 影响 |
|------|-----------|--------------|------|
| 全因子配置数 | 4×3×3×3×4×3 = 1,296 | 约1,260（去除约36种） | 减少2.8% |
| 预期D-efficiency | ~0.92 | ~0.85 | 下降约7% |
| 是否可接受 | — | 是 | ≥0.80达标 |

### 3.3 已评估但未纳入的禁止组合

以下组合经评估后**不纳入**禁止列表：

| 组合 | 不纳入原因 |
|------|-----------|
| 美的/小米 + ¥5,999 | 品牌向上延伸是可能的，保留以测试品牌溢价边界 |
| 6套 + 嵌入式 | 小户型也可能选择嵌入式，保留 |
| 基础功能 + ¥5,999 | 虽不合理，但可测试"功能-价格错配"的拒绝行为 |

## 四、细分群体设计方案

### 4.1 群体分配

| 群体 | 人数 | 占比 | 专属情境 | 实验设计调整 |
|------|------|------|---------|-------------|
| **精致白领** | 80人 | 33.3% | 忙碌工作日晚餐后的场景 | 标准设计 |
| **新手宝妈** | 80人 | 33.3% | 宝宝辅食后清洁场景 | 标准设计 |
| **Z世代租客** | 80人 | 33.3% | 合租小厨房场景 | 标准设计 |

### 4.2 设计策略：统一设计 vs 分组设计

**决策：采用统一实验设计**

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| **统一设计** | 群体间可比、设计效率高、实现简单 | 可能包含某群体不关注的水平 | **选定** |
| 分组设计 | 更贴合各群体需求 | 设计工作量大、群体间不可比 | 放弃 |

**理由**：
1. 三个群体的属性空间有重叠（都关注价格、容量、品牌）
2. 统一设计可通过情境注入实现个性化（见4.3节）
3. 分组设计会导致样本量分割，降低统计精度

### 4.3 情境注入方案

虽然实验设计统一，但每个群体在作答前接收**不同的情境描述**：

```yaml
scenario_injection:
  精致白领:
    context: |
      你刚下班到家，做了顿简单的晚餐。外面在下雨，你不想出门。
      厨房不大，但你希望生活品质不打折。你正在看洗碗机...
    emphasis: ["省时", "品质", "空间紧凑", "设计感"]
    
  新手宝妈:
    context: |
      宝宝的辅食碗、奶瓶、餐具堆了一水槽。你担心手洗洗不干净，有细菌残留。
      家里正准备装修厨房，可以预留嵌入式空间...
    emphasis: ["除菌", "安全", "大容量", "专业品牌"]
    
  Z世代租客:
    context: |
      你和室友合租，厨房很小。你们经常做饭，但谁都不想洗碗。
      房东不让改造厨房，不能装嵌入式...
    emphasis: ["免安装", "低价", "够用", "搬家可带走"]
```

### 4.4 群体-画像映射

详见 `docs/洗碗机CBC实验设计方案.md` 第3.1节，此处引用关键约束：

| 群体 | 年龄 | 收入 | 居住 | 核心焦虑 |
|------|------|------|------|---------|
| 精致白领 | 25-30 | 15-50万 | 租房/小公寓 | 同辈压力、FOMO |

## 五、编码方案

### 5.1 Effects Coding 定义

```yaml
encoding:
  method: "effects_coding"
  
  attributes:
    price:
      type: "continuous"
      levels: [2999, 3999, 4999, 5999]
      n_params: 1
      note: "取对数或标准化后进入模型"
      
    capacity:
      type: "categorical"
      levels: ["6套", "10套", "13套"]
      n_params: 2
      reference: "13套为参照（-(capacity_0 + capacity_1)）"
      
    installation:
      type: "categorical"
      levels: ["台式", "嵌入式", "水槽式"]
      n_params: 2
      reference: "水槽式为参照"
      
    features:
      type: "categorical"
      levels: ["基础", "智能", "全能"]
      n_params: 2
      reference: "全能为参照"
      
    brand:
      type: "categorical"
      levels: ["美的", "西门子", "方太", "小米"]
      n_params: 3
      reference: "小米为参照"
      
    energy:
      type: "categorical"
      levels: ["二级", "一级", "超一级"]
      n_params: 2
      reference: "超一级为参照"
      
  total_parameters: 12
```

### 5.2 参数命名规范

| 属性 | 参数名 | 含义 |
|------|--------|------|
| 价格 | `price` | 价格系数（必须为负） |
| 容量 | `capacity_0` | 6套 vs 参照(13套) |
| 容量 | `capacity_1` | 10套 vs 参照(13套) |
| 安装 | `install_0` | 台式 vs 参照(水槽式) |
| 安装 | `install_1` | 嵌入式 vs 参照(水槽式) |
| 功能 | `feature_0` | 基础 vs 参照(全能) |
| 功能 | `feature_1` | 智能 vs 参照(全能) |
| 品牌 | `brand_0` | 美的 vs 参照(小米) |
| 品牌 | `brand_1` | 西门子 vs 参照(小米) |
| 品牌 | `brand_2` | 方太 vs 参照(小米) |
| 能耗 | `energy_0` | 二级 vs 参照(超一级) |
| 能耗 | `energy_1` | 一级 vs 参照(超一级) |
