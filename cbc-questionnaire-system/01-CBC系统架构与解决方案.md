# AI生成CBC问卷系统 — 架构与解决方案

> **版本**：v1.0  
> **定位**：与消费者模拟Agent配套使用的CBC问卷自动生成系统  
> **配套系统**：consumer-simulation（消费者模拟Agent）

---

## 一、什么是CBC问卷

**CBC（Choice-Based Conjoint，基于选择的联合分析）** 是一种经典的市场调研方法：

- 将产品拆解为若干**属性**（如价格、品牌、功能）
- 每个属性设定若干**水平**（如价格：99元/199元/299元）
- 生成多个**选择集**，每个集合展示2-4个由不同水平组合而成的产品配置
- 受访者在每个选择集中选出最偏好的选项
- 通过统计模型反推各属性的**相对重要性**和**各水平的效用值**

### CBC问卷的典型呈现形式

┌─────────────────────────────────────────────────────────────┐

│                                                             │

│  │   选项 A      │  │   选项 B      │  │   选项 C      │      │
│  │              │  │              │  │              │      │
│  │ 品牌：华为    │  │ 品牌：小米    │  │ 品牌：苹果    │      │
│  │ 价格：¥3999   │  │ 价格：¥2999   │  │ 价格：¥5999   │      │
│  │ 内存：256GB   │  │ 内存：128GB   │  │ 内存：256GB   │      │
│  │ 电池：5000mAh │  │ 电池：4500mAh │  │ 电池：4800mAh │      │
│  │ 充电：66W     │  │ 充电：33W     │  │ 充电：20W     │      │

### 2.2 核心设计原则

1. **智能实验设计**：AI自动生成统计效率最优的选择集，避免人工设计中的正交表陷阱
2. **与画像联动**：利用消费者画像中的属性偏好，生成更符合目标人群的属性/水平设定
3. **动态自适应**：可根据前期模拟结果，动态调整后续问卷的属性组合
4. **质量可校验**：内置D-efficiency、A-efficiency等实验设计效率指标

## 五、必填信息详解（无则无法生成有效问卷）

### 5.1 产品属性列表（Attributes）

**定义**：产品/服务的各个维度，受访者需要在其中做权衡取舍。

**示例**：
```json
{
  "attributes": [
    {"id": "brand", "name": "品牌"},
    {"id": "price", "name": "价格"},
    {"id": "storage", "name": "存储容量"},
    {"id": "battery", "name": "电池容量"},
    {"id": "charging", "name": "充电功率"}
  ]
}
```

**不填的后果**：❌ **无法生成问卷**。属性是CBC问卷的骨架，没有属性就没有选择的基础。

**设计原则**：
- 属性数量建议 **3-6个**（过多导致认知负荷过高，过少信息量不足）
- 每个属性应该是受访者**真正关心的决策维度**
- 属性之间应具有一定的**独立性**（避免高度相关属性如"屏幕尺寸"和"机身重量"）

---

### 5.2 属性水平（Levels）

**定义**：每个属性的具体取值或档位。

**示例**：
```json
{
  "levels": {
    "brand": ["华为", "小米", "苹果"],
    "price": [2999, 3999, 5999],
    "storage": ["128GB", "256GB", "512GB"],
    "battery": ["4500mAh", "5000mAh", "5500mAh"],
    "charging": ["33W", "66W", "120W"]
  }
}
```

**不填的后果**：❌ **无法生成问卷**。没有水平就无法构成具体的产品配置选项。

**设计原则**：
- 每个属性的水平数建议 **2-5个**
- 水平之间应有**可感知的差异**
- 水平应覆盖**市场上真实存在的范围**
- **价格属性强烈建议包含**（用于计算货币化效用）

---

### 5.3 研究目的（Research Objective）

**定义**：本次CBC研究要回答的核心商业问题。

**示例**：
```markdown
研究目的：
1. 了解消费者对智能手机各属性的相对重视程度
2. 确定最优产品配置组合
3. 评估价格敏感度，确定最优定价区间
4. 预测不同配置的市场份额
```

**不填的后果**：⚠️ **问卷可以生成，但实验设计可能次优**。没有明确的研究目的：
- 无法确定哪些属性需要更精细的水平划分
- 无法确定选择集数量（精度需求不同）
- 可能遗漏关键属性或包含无关属性
- 后续数据分析缺乏方向

---

### 5.4 关键研究问题（Key Questions）

**定义**：研究目的的具体化，指导问卷设计的优先级。

**示例**：
```markdown
关键问题：
1. 在3000-4000元价位段，消费者最看重什么？
2. 120W快充是否足以成为溢价理由？
3. 品牌溢价空间有多大？
4. 存储容量升级意愿的价格弹性如何？
```

**不填的后果**：⚠️ **问卷可生成，但属性/水平设定可能不够精准**。例如：
- 如果不明确"价格敏感度是核心"，价格水平的跨度可能设置不合理
- 如果不明确"品牌对比是重点"，品牌属性可能只放1-2个水平

## 六、可选信息详解与影响分析

### 6.1 目标消费者画像（Consumer Personas）

**定义**：使用现有消费者模拟系统生成的画像，用于个性化问卷设计。

**示例**：
```json
{
  "target_personas": ["persona-001", "persona-003", "persona-005"],
  "adaptation_strategy": "per_segment"
}
```

**不填的影响**：
- ✅ **问卷仍可生成**，但使用"通用人群"假设
- ⚠️ 属性水平范围可能偏离目标人群的实际考虑范围（如给高收入人群设置了过低的价格水平）
- ⚠️ 可能包含目标人群不关心的属性（如给老年人设置"游戏性能"属性）
- ⚠️ 模拟消费者回答时缺乏画像背景，可能产生不真实的偏好表达

**使用方式**：
```
有画像 → 系统从画像中提取：
  - 价格敏感度 → 调整价格水平的跨度
  - 品牌偏好 → 确定品牌属性的候选集
  - 功能需求 → 筛选相关属性
  - 决策风格 → 调整选择集复杂度

无画像 → 使用默认通用设定
```

---

### 6.2 属性类型标注（Attribute Types）

**定义**：标注每个属性是分类变量、连续变量还是价格变量。

**示例**：
```json
{
  "attribute_types": {
    "brand": "categorical",
    "price": "price",
    "storage": "categorical",
    "battery": "continuous",
    "charging": "continuous"
  }
}
```

**不填的影响**：
- ✅ **问卷外观不受影响**
- ⚠️ 系统默认所有属性为分类变量
- ⚠️ 连续变量（如电池容量）被当作离散水平处理，丢失序数信息
- ⚠️ 价格属性未被特殊处理，无法自动计算WTP（支付意愿）
- ⚠️ 统计分析时无法使用更高效的模型（如线性参数模型 vs 全交模型）

---

### 6.3 实验设计参数（Design Parameters）

#### 6.3.1 选择集数量（Number of Choice Sets）

**定义**：问卷中包含多少个选择任务。

**不填的默认**：系统根据属性数量和水平数自动计算（通常为 8-16 个）

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 如果选择集过少 → 参数估计精度不足，置信区间过宽
- ⚠️ 如果选择集过多 → 受访者疲劳，后期回答质量下降
- ⚠️ 默认数量可能不匹配你的样本量计划

**推荐公式**：
```
最小选择集数 ≥ 总参数数 × 5 / 样本量
典型范围：8-20个（移动端建议≤12个）
```

#### 6.3.2 每集选项数（Alternatives per Set）

**定义**：每个选择集中展示几个产品选项。

**不填的默认**：3个选项 + 1个"都不选"

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 选项过多 → 认知负荷高，模拟消费者可能随机选择
- ⚠️ 选项过少 → 信息提取效率低，需要更多选择集来补偿

#### 6.3.3 "都不选"选项（None Option）

**定义**：是否允许受访者表示"以上都不选"。

**不填的默认**：包含"都不选"

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 不包含"都不选" → 强制选择可能高估偏好强度
- ⚠️ 包含"都不选" → 可以估计"退出市场"的概率，但增加模型复杂度

**建议**：
- 市场份额预测 → **建议包含**
- 产品优化研究 → **可选**
- 强制选择可获得更清晰的偏好排序

---

### 6.4 固定产品/基准选项（Fixed/Benchmark Alternative）

**定义**：在每个选择集中固定出现的一个选项（通常是现有产品或竞品）。

**示例**：
```json
{
  "fixed_alternative": {
    "enabled": true,
    "profile": {
      "brand": "当前品牌",
      "price": 3999,
      "storage": "256GB",
      "battery": "5000mAh",
      "charging": "66W"
    },
    "label": "您目前使用的产品"
  }
}
```

**不填的影响**：
- ✅ 问卷可生成，所有选项都随机生成
- ⚠️ 无法直接对比"新产品 vs 现有产品"
- ⚠️ 效用值是相对的，缺乏锚定点
- ⚠️ 市场份额预测需要额外假设

**使用场景**：
- 新产品概念测试 → **强烈建议**
- 竞品分析 → **强烈建议**
- 纯属性重要性研究 → **可选**

---

### 6.5 禁止组合（Prohibited Combinations）

**定义**：现实中不可能或不合理的产品配置组合。

**示例**：
```json
{
  "prohibited_combinations": [
    {"brand": "苹果", "price": 2999},
    {"brand": "小米", "charging": "120W", "battery": "4500mAh"}
  ]
}
```

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 可能出现现实中不存在的产品组合（如"苹果 + 2999元"）
- ⚠️ 模拟消费者面对不合理组合时可能困惑或给出不真实回答
- ⚠️ 降低实验设计的统计效率（因为设计空间被污染）

---

### 6.6 必须包含组合（Required Combinations）

**定义**：必须出现在问卷中的特定配置（如正在考虑的新品配置）。

**示例**：
```json
{
  "required_combinations": [
    {
      "profile": {"brand": "华为", "price": 4999, "storage": "512GB", "battery": "5500mAh", "charging": "120W"},
      "min_appearances": 3
    }
  ]
}
```

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 特定关心的产品配置可能只出现1次或根本不出现
- ⚠️ 无法对该配置获得足够的偏好数据
- ⚠️ 新品概念可能没有被充分测试

---

### 6.7 决策场景描述（Decision Context）

**定义**：为模拟消费者设定的购买情境描述。

**示例**：
```markdown
场景描述：
"你现在的手机已经用了3年，电池续航明显下降，
存储空间也经常不足。你打算在618期间换一部新手机，
预算在3000-5000元之间。你在逛京东时看到了以下选项..."
```

**不填的影响**：
- ✅ 问卷可生成，使用默认场景
- ⚠️ 模拟消费者缺乏情境代入感，可能输出"基线理性人"的偏好
- ⚠️ 无法测试情境对偏好的影响（如促销情境 vs 日常情境）
- ⚠️ 与消费者模拟系统的Layer 4场景叙事无法联动

**最佳实践**：
- 结合消费者画像的**场景反应库**自动生成情境
- 不同画像使用不同的情境描述

---

### 6.8 问卷呈现参数（Presentation Parameters）

#### 6.8.1 媒介形式

- **移动端** → 单列布局，每集2-3个选项
- **PC端** → 多列并排，可展示更多选项
- **对话式** → 模拟消费者Agent以对话方式逐题回答

**不填的默认**：PC端标准表格布局

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 模拟消费者Agent的交互方式可能与实际不匹配
- ⚠️ 移动端选择集过于复杂可能导致模拟失真

#### 6.8.2 品牌展示策略

- **显示品牌名** → 品牌作为显性属性
- **隐藏品牌（盲测）** → 测试纯产品属性偏好
- **品牌logo+名称** → 更接近真实购买场景

**不填的默认**：显示品牌名

**不填的影响**：
- ✅ 问卷可生成
- ⚠️ 无法区分"品牌偏好"和"产品属性偏好"
- ⚠️ 如果研究目的是测试产品概念（不含品牌），默认设置会引入混淆

---

### 6.9 交互式自适应参数（Adaptive Parameters）

**定义**：根据前期回答动态调整后续选择集的参数。

**示例**：
```json
{
  "adaptive": {
    "enabled": true,
    "adapt_after": 4,
    "method": "prior_update",
    "focus_attributes": ["price", "brand"]
  }
}
```

**不填的默认**：非自适应设计（所有受访者看到相同的固定问卷）

**不填的影响**：
- ✅ 问卷可生成，使用标准固定设计
- ⚠️ 无法根据模拟消费者的早期回答聚焦关键属性
- ⚠️ 问卷长度固定，无法对"已明确偏好"的受访者提前结束
- ⚠️ 统计效率低于自适应设计（需要更多选择集达到同等精度）

## 七、信息完整度与输出质量关系

| 完整度等级 | 包含信息 | 输出质量预期 | 适用场景 |
|-----------|---------|------------|---------|
| **最小可行** | 属性+水平+研究目的 | 可生成基础CBC问卷，能用于简单偏好排序 | 快速概念验证、内部讨论 |
| **标准** | 最小可行+关键问题+设计参数 | 统计有效的标准CBC问卷，可用于正式研究 | 常规市场调研 |
| **完整** | 标准+画像+场景+约束条件 | 与消费者模拟系统联动的个性化问卷 | AI模拟消费者研究 |
| **极致** | 完整+自适应参数+品牌策略+多版本 | 研究级自适应CBC问卷，支持动态优化 | 高精度市场需求预测 |

### 8.2 画像驱动的属性发现

```python
# 伪代码：从画像自动提取CBC属性建议
def extract_cbc_attributes_from_persona(persona: Profile) -> AttributeSuggestion:
    """
    基于消费者画像的特征，推荐CBC问卷的属性和水平
    """
    suggestions = AttributeSuggestion()
    
    # 从Layer 2行为签名提取
    if persona.behavior.price_sensitivity == "理性比价/精明型":
        suggestions.attributes.append("价格")
        suggestions.levels["价格"] = ["低价", "中价", "高价"]  # 相对化
        suggestions.priority["价格"] = "high"
    
    # 从Layer 3心理引擎提取
    if "品牌忠诚度" in persona.behavior.brand_loyalty:
        suggestions.attributes.append("品牌")
        suggestions.levels["品牌"] = persona.brand_preferences[:3]
    
    # 从场景反应提取情境
    if persona.scene_reactions:
        suggestions.decision_context = generate_context_from_scenes(
            persona.scene_reactions
        )
    
    return suggestions
```

## 十、常见错误与规避

| 错误类型 | 错误示例 | 后果 | 修正 |
|---------|---------|------|------|
| 属性过多 | 8个属性 × 5个水平 | 认知超载，模拟消费者随机选 | 精简至3-6个核心属性 |
| 水平不可区分 | 价格：2999 vs 3049 | 受访者感知不到差异 | 水平间距应≥最小可觉差 |
| 遗漏价格 | 没有价格属性 | 无法计算WTP和市场份额 | 始终包含价格属性 |
| 忽略禁止组合 | 苹果+2999 | 不合理选项导致回答失真 | 设置 prohibited_combinations |
| 选择集过少 | 4个选择集 | 参数估计不收敛 | 按公式计算最小数量 |
| 情境缺失 | 无场景描述 | 模拟消费者缺乏代入感 | 提供决策场景 |
| 属性不独立 | 屏幕尺寸+机身重量 | 多重共线性 | 合并或删除相关属性 |
