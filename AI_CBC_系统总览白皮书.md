## 一、系统愿景与核心价值

### 1.1 我们要解决什么问题

传统联合分析（Conjoint Analysis / CBC）面临三大痛点：

| 痛点 | 传统方式 | 我们的方案 |
|------|---------|-----------|
| **受访者招募难** | 需要招募200-500名真实消费者，成本高、周期长 | 用LLM生成"虚拟消费者"，按需批量生产，分钟级就绪 |
| **回答质量不可控** | 真实受访者可能敷衍作答、中途退出、理解偏差 | 虚拟消费者可被精确控制，每次回答都附带完整决策理由 |
| **分析门槛高** | 需要专业统计软件（Sawtooth/Lighthouse Studio），学习曲线陡峭 | Agent驱动的自然语言交互，自动建模、自动解读、自动生成报告 |

### 1.2 核心命题

> **基于LLM的Agent能否生成足够"像真人"的虚拟消费者，使其填写的CBC问卷数据在统计建模后产生与真实消费者研究等价甚至更优的业务洞察？**

这不是要"替代"真实消费者研究，而是要：
- **在真实调研前快速验证假设**（前置筛选，降低真实调研风险）
- **在无法进行真实调研时提供决策依据**（如概念测试阶段、预算受限时）
- **作为真实调研的补充，探索更多细分场景**（生成真实世界中难以招募的小众人群）

### 1.3 系统核心价值主张

```
┌─────────────────────────────────────────────────────────────┐
│                    AI_CBC 核心价值                            │
├─────────────────────────────────────────────────────────────┤
│  🚀 速度：从业务问题到洞察报告，从数周压缩到数小时              │
│  🎯 精度：分层贝叶斯 + 个体效用估计，统计严谨性不妥协          │
│  💰 成本：单次全流程成本可控，规模化后边际成本趋近于零         │
│  🔒 可控：每个虚拟消费者都有完整画像可追溯，决策过程透明        │
│  🧪 迭代：Prompt级调优即可改变消费者行为模式，实验成本低       │
│  📊 深度：交叉分析画像特征与偏好结构，发现传统调研遗漏的细分    │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 端到端标准数据交换格式

三个子系统之间通过标准格式交换数据：

| 数据流 | 从 → 到 | 格式 | 规范文档 |
|--------|---------|------|---------|
| 画像输出 | 消费者模拟 → 问卷系统 | `PersonaProfile` JSON | `consumer-simulation/09-Agent输出信息规范.md` |
| 问卷输入 | 问卷系统 ← 业务输入 | `CBCQuestionnaireConfig` JSON | `cbc-questionnaire-system/02-CBC问卷生成输入规范.md` |
| 标准交换数据 | 问卷系统 → 分析系统 | `CBCRawDataset` JSON/CSV | `cbc-questionnaire-system/04-CBC与模拟消费者集成方案.md` §3.3.b |
| 分析报告 | 分析系统 → 用户 | Markdown / HTML / PDF | `cbc-analysis-system/01-CBC数据分析系统架构.md` §8 |

## 五、关键设计原则

### 5.1 虚拟消费者"像不像真人"的设计哲学

```
真实消费者 ≠ 完美理性人
真实消费者 = 内在矛盾性 + 情境驱动性 + 认知有限性 + 叙事一致性
```

| 原则 | 含义 | 反例 | 实现方式 |
|------|------|------|---------|
| **张力优先** | 主动组合矛盾标签，制造内在冲突 | "退休+节俭+被保健品骗"的刻板印象 | Seed Generator内置张力检测 |
| **动机可溯** | 每个异常行为都有心理层面的解释 | 高收入者极端节俭却无历史成因 | Logic Validator RULE-001 |
| **认知有限** | 会疲劳、会敷衍、会走捷径 | 无论多累都保持完美理性 | 情境注入认知负荷参数 |
| **情境敏感** | 同一人在不同场景下行为不同 | 情绪高涨和焦虑时决策模式相同 | Layer 4 场景叙事层 |
| **时间延续** | 有记忆、有惯性、有状态转移 | 每次交互都是独立随机事件 | 模拟历史上下文传递 |

### 5.2 统计严谨性原则

1. **模型选择有据**：根据样本量和属性数自动推荐模型（MNL/MXL/HB）
2. **收敛必检**：HB模型必须报告R-hat和ESS，R-hat > 1.1视为未收敛
3. **基准验证**：使用已知参数的合成数据验证模型参数恢复能力
4. **编码规范**：统一使用Effects Coding，参数和为0，便于解释
5. **WTP计算有前提**：价格系数必须为负，否则标记警告

### 5.3 安全与伦理原则

1. **偏见零容忍**：虚拟消费者的偏好不得与受保护属性（性别、种族等）系统性相关
2. **透明可审计**：每个虚拟消费者的生成过程、决策过程、修改历史全量记录
3. **最小权限**：Agent的工具调用权限按需分配，禁止越权操作
4. **输入不可信**：所有外部输入（包括LLM输出）必须经过校验
5. **成本可见**：每次LLM调用的token数和费用全量记录，实时可见

## 九、附录

### 附录A：术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| 联合分析 | Conjoint Analysis | 通过让受访者在多个产品配置中做选择，反推各属性偏好的统计方法 |
| CBC | Choice-Based Conjoint | 基于选择的联合分析，每题展示2-4个选项让受访者选择 |
| 虚拟消费者 | Synthetic Consumer | 由LLM生成的、具有完整画像和偏好的模拟消费者 |
| 部分效用值 | Part-Worth Utility | 某属性水平对总体偏好的贡献值，可为正或负 |
| 属性重要性 | Attribute Importance | 某属性在决策中的相对权重，所有属性之和为100% |
| WTP | Willingness to Pay | 支付意愿，消费者愿意为某项属性改进支付的金额 |
| HB | Hierarchical Bayes | 分层贝叶斯模型，可估计个体层面的效用分布 |
| D-efficiency | Design Efficiency | 实验设计的统计效率指标，越接近1越好 |
| 选择集 | Choice Set | CBC问卷中的每道题目，包含若干个产品选项 |
| 效应编码 | Effects Coding | 一种分类变量编码方式，参数和为0，便于解释 |

### 附录B：外部参考资源

- **Sawtooth Software**: CBC/HB行业标准软件，用于基准验证

- **biogeme**: Python离散选择模型库，用于MNL/MXL基线
- **PyMC**: Python概率编程库，用于HB模型的MCMC采样
- **Lighthouse Studio**: Sawtooth的问卷设计平台，用于实验设计对比
